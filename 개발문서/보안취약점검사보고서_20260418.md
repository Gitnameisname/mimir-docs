#### [High] SEC-BE-001: OAuth 토큰 평문 저장 (Fail-Open) ✅ 수정완료

- **파일**: `backend/app/api/auth/encryption.py:42-60`
- **OWASP**: A02:2021 – Cryptographic Failures
- **설명**: `OAUTH_TOKEN_ENCRYPTION_KEY` 환경변수 미설정 시 `encrypt_token()`이 경고 로그만 출력하고 **평문 토큰을 반환**하여 DB에 암호화 없이 저장됨. 키 미설정이 운영 환경에서 발생하면 DB 유출 시 OAuth Access Token 노출.
- **재현 시나리오**: 신규 배포 환경에서 `OAUTH_TOKEN_ENCRYPTION_KEY` 환경변수 누락 → GitLab OAuth 연동 → DB `oauth_tokens` 컬럼에 평문 토큰 저장 → DB 덤프 유출 시 토큰 즉시 탈취.
- **수정**: `encrypt_token()` 키 미설정 시 `RuntimeError` raise (Fail-Secure). `decrypt_token()` 키 미설정 시 평문 반환 대신 `None` 반환.

```python
# 수정 전
if key is None:
    logger.warning("storing plaintext")
    return plaintext  # 평문 저장

# 수정 후
if key is None:
    raise RuntimeError("OAUTH_TOKEN_ENCRYPTION_KEY 미설정 — 평문 저장 불가")
```

#### [Medium] SEC-BE-002: 스택 트레이스 서버 로그 기록 ✅ 수정완료

- **파일**: `backend/app/api/errors/handlers.py:260-270`
- **OWASP**: A05:2021 – Security Misconfiguration
- **설명**: 처리되지 않은 예외 발생 시 `logger.exception()` + `traceback.format_exc()`로 전체 스택 트레이스를 서버 로그에 기록. 로그 수집 시스템 접근 가능한 내부자 또는 로그 인젝션 공격 시 내부 파일 경로·모듈 구조 노출.
- **재현 시나리오**: 공격자가 로그 수집 시스템(ELK, CloudWatch 등)에 접근 시 내부 구조 파악 가능.
- **위험도 평가**: 외부 클라이언트에는 노출되지 않으나 내부 보안 수준에 영향.
- **수정**: `error_ref = uuid.uuid4().hex[:12]`로 에러 참조 ID 생성. `logger.error`에는 `request_id + error_ref`만 기록. 스택 트레이스는 `logger.debug`로 분리하여 프로덕션 로그 레벨에서 자동 제외.

```python
# 수정 후
error_ref = uuid.uuid4().hex[:12]
logger.error("Unhandled exception | request_id=%s error_ref=%s", meta.request_id, error_ref)
logger.debug("Stack trace for error_ref=%s:\n%s", error_ref, traceback.format_exc())
```

#### [Medium] SEC-BE-003: Webhook 타임아웃 단일 설정 (Slowloris 부분 노출) ✅ 수정완료

- **파일**: `backend/app/services/alert_notifier.py`
- **OWASP**: A05:2021 – Security Misconfiguration
- **설명**: Webhook 전송에 `urllib.request.urlopen(timeout=10)` 사용. `timeout`은 소켓 수준(읽기+쓰기)에 적용되어 기본 Slowloris는 차단되지만, TLS 핸드셰이크 타임아웃이 별도 제어되지 않음.
- **재현 시나리오**: TLS 핸드셰이크를 지연시키는 서버로 Webhook URL 설정 시 최대 10초 지연 가능.
- **위험도**: Low-Medium — Private IP 차단 + 스킴 제한으로 내부망 공격은 차단됨.
- **수정**: `urllib.request` → `httpx.Client` 마이그레이션. `connect_timeout=5`, `read_timeout=10`, `write_timeout=5`, `pool_timeout=2` 분리 설정. `verify=True`로 TLS 검증 명시.

```python
# 수정 후
timeout = httpx.Timeout(connect=5, read=10, write=5.0, pool=2.0)
with httpx.Client(timeout=timeout, verify=True) as client:
    resp = client.post(url, json=payload, headers={"User-Agent": "Mimir-Alert/1.0"})
    return 200 <= resp.status_code < 300
```

#### [High] SEC-FE-001: Open Redirect (`?redirect=` 파라미터 미검증) ✅ 수정완료

- **파일**: `frontend/src/app/login/page.tsx:35-36, 64-65`
- **OWASP**: A01:2021 – Broken Access Control
- **설명**: 로그인 후 `?redirect=` 파라미터를 `decodeURIComponent()` 후 `router.push()`에 직접 전달. 외부 도메인 URL 허용으로 오픈 리다이렉트 가능.
- **재현 시나리오**: 피싱 이메일에 `/login?redirect=https://evil.com` 링크 삽입 → 사용자가 정상 Mimir 도메인 로그인 후 외부 사이트로 리다이렉트 → 자격 증명 탈취.
- **CVSS 3.1**: 6.1 (Medium) — AV:N/AC:L/PR:N/UI:R/S:C/C:L/I:L/A:N
- **수정**: `/`로 시작하는 경로만 허용 (내부 경로 whitelist).

```typescript
// 수정 전
router.push(redirect ? decodeURIComponent(redirect) : "/");

// 수정 후
const safeRedirect = raw && decodeURIComponent(raw).startsWith("/") ? decodeURIComponent(raw) : "/";
router.push(safeRedirect);
```

#### [High] SEC-FE-002: CSP `unsafe-eval` 프로덕션 노출 ✅ 수정완료

- **파일**: `frontend/next.config.ts:33`
- **OWASP**: A05:2021 – Security Misconfiguration
- **설명**: Content-Security-Policy에 `'unsafe-eval'`이 환경 구분 없이 설정됨. 개발 환경 HMR을 위한 설정이 프로덕션 빌드에도 적용되어 XSS 공격 시 `eval()` 실행 가능.
- **재현 시나리오**: XSS 취약점 발견 시 `eval(atob('maliciousPayload'))` 실행 가능. CSP가 마지막 방어선이므로 XSS와 결합 시 피해 증대.
- **수정**: `process.env.NODE_ENV === "development"` 기준으로 CSP `script-src` 환경 분리.

```typescript
// 수정 후
const scriptSrc = IS_DEV
  ? "script-src 'self' 'unsafe-inline' 'unsafe-eval'"
  : "script-src 'self' 'unsafe-inline'";
```

---

#### [High] SEC-FE-003: `dangerouslySetInnerHTML` — DOMPurify 도입 ✅ 수정완료

- **파일**: `frontend/src/features/search/SearchPage.tsx`
- **OWASP**: A03:2021 – Injection
- **설명**: 검색 하이라이트 렌더링에 `dangerouslySetInnerHTML` 사용. 1차 검수에서 3단계 방어 구현됨. 백엔드 `ts_headline()` 출력 형식 변경 시 수동 방어 로직이 깨질 위험 잔존.
- **수정**: `escapeHtmlExceptHighlight()` 수동 방어 → `DOMPurify.sanitize()` 라이브러리 방어로 교체. `dompurify@^3.4.0` + `@types/dompurify@^3.0.5` 설치. `ALLOWED_TAGS: ["mark"]`, `ALLOWED_ATTR: ["class"]`로 허용 범위 최소화.

```typescript
// 수정 후
function sanitizeSnippet(raw: string): string {
  const withMark = raw.replace(/<b>/gi, '<mark class="...">').replace(/<\/b>/gi, "</mark>");
  return DOMPurify.sanitize(withMark, { ALLOWED_TAGS: ["mark"], ALLOWED_ATTR: ["class"] });
}
```

#### [Medium] SEC-FE-004: 토큰 갱신 실패 시 대기 요청 미처리 ✅ 수정완료

- **파일**: `frontend/src/lib/api/client.ts`
- **OWASP**: A07:2021 – Identification and Authentication Failures
- **설명**: 401 응답 시 silent refresh 1회 시도 후 실패하면 로그인 리다이렉트. 동시에 여러 요청이 401로 실패 시 중복 리다이렉트 발생 및 대기 요청 미해결 상태 잔류.
- **재현 시나리오**: 여러 컴포넌트가 동시에 API 요청 → 토큰 만료 → 갱신 실패 → 복수의 `window.location.href` 재할당 경쟁.
- **수정**: 모듈 수준 `_isRedirecting` 플래그 추가. 갱신 실패 후 첫 번째 요청만 리다이렉트 수행; `_waitQueue`에 적체된 나머지 요청은 일괄 실패 처리. `AuthContext.tsx`의 `_refreshPromise` 패턴으로 갱신 자체는 이미 1회만 실행됨.

---

#### [Medium] SEC-FE-005: `window.__mimir_at` 레거시 노출 ✅ 수정완료

- **파일**: `frontend/src/contexts/AuthContext.tsx`, `frontend/src/types/global.d.ts`
- **OWASP**: A02:2021 – Cryptographic Failures
- **설명**: Access Token이 메모리에 저장되는 것이 원칙이나, 레거시 호환성을 위해 `window.__mimir_at`에도 저장. 브라우저 개발자 도구 콘솔에서 `window.__mimir_at`로 토큰 확인 가능.
- **재현 시나리오**: 악의적 브라우저 확장 프로그램 또는 XSS → `window.__mimir_at` 접근 → Access Token 탈취.
- **수정**: `setAccessToken()`의 `window.__mimir_at` 할당/삭제 블록 제거. `global.d.ts`의 `Window` 인터페이스 확장 제거. `getAccessToken()`은 이미 모듈 수준 `_accessToken` 변수를 읽어 `window.__mimir_at`를 사용하지 않음이 확인됨 — 안전하게 제거 가능.
