## 2. 발견 항목 및 조치 결과

### SEC5-BE-001 · Medium · [완료] 파일 업로드 MIME 타입 미검증

| 항목 | 내용 |
|------|------|
| **위치** | `backend/app/api/v1/golden_sets.py` — `import_golden_set()` (line 509) |
| **취약점** | `UploadFile` 수신 시 `content_type` 및 파일 확장자 미검증. 임의 파일 타입 허용 |
| **OWASP** | A04:2021 – Insecure Design (Unrestricted File Upload) |
| **심각도** | Medium |
| **분석** | `file.file.read()` 후 바로 `json.loads()`를 시도하므로 실제 코드 실행으로 이어지지는 않음. 그러나 `.exe`, `.sh` 등 비 JSON 파일이 업로드 허용되면 향후 로직 변경 시 위험도 상승. 방어 심층화(Defense in Depth) 원칙 위반 |
| **조치** | `content_type` 화이트리스트 검증(`application/json`, `text/json`, `text/plain`) + 파일명 확장자 `.json` 강제. MIME 불일치 또는 확장자 불일치 시 400 반환 |
| **상태** | ✅ 완료 |

### SEC5-BE-002 · Low · [완료] OAuth 콜백 엔드포인트 Rate Limit 부재

| 항목 | 내용 |
|------|------|
| **위치** | `backend/app/api/v1/auth_router.py` — `oauth_gitlab_callback()` (line 516) |
| **취약점** | `GET /auth/oauth/gitlab/callback` 엔드포인트에 `@limiter.limit()` 데코레이터 미적용. 로그인(`POST /auth/login`)은 Valkey 기반 이메일별 잠금 + slowapi 제한이 모두 적용되어 있으나, OAuth 콜백은 slowapi 제한만 미적용 상태 |
| **OWASP** | A04:2021 – Insecure Design |
| **심각도** | Low |
| **분석** | GitLab state/code 쌍은 단회성이며 만료 시간이 짧아 실질적 brute-force 가능성은 낮음. 그러나 DDoS 관점에서 rate limit 부재는 방어 심층화 부족 |
| **조치** | `@limiter.limit("20/minute")` 데코레이터 추가 및 `from app.api.rate_limit import limiter` 임포트 추가 |
| **상태** | ✅ 완료 |

## 3. 허위 양성(False Positive) 목록

| 스캔 보고 ID | 위치 | 판정 사유 |
|-------------|------|----------|
| rate_limit 타입 강제 변환 | `auth/rate_limit.py` | `decode_responses=True`로 Valkey 응답이 항상 str 반환. `int("5")` 정상 동작. `except Exception`으로 ValueError도 포착 |
| JWT 알고리즘 혼용 | `auth/tokens.py` | PyJWT 2.x는 `algorithms=["HS256"]` 지정 시 `alg: "none"` 자동 거부. 추가 검증 불필요 |
| OAuth state 데이터 길이 | `oauth_service.py` | state 값(Value)은 서버가 생성·저장. 공격자는 state 키(Key)만 제공 가능하며, 키 불일치 시 Valkey 조회 결과 None |
| Cookie Secure 플래그 | `auth_router.py` | 개발 환경 전용 의도적 설정. 스테이징/프로덕션에서 `_RT_COOKIE_SECURE = True` 적용 |
| CSP 헤더 미적용 | `next.config.ts` | 이미 VULN-010/SEC-007 대응으로 완전한 CSP 헤더 구성 중 (`default-src 'self'`, `frame-ancestors 'none'` 등) |

| Rate Limit UX 미흡 | `api/client.ts` | UX 개선 사항이며 보안 취약점 아님 |
| CSV 인젝션 | export 엔드포인트 | 프로젝트에 CSV/스프레드시트 내보내기 기능 없음. StreamingResponse는 SSE(text/event-stream)에만 사용 |

## 6. 보안 강점 (4차 확인 사항)

4차 검사에서 다음 항목이 올바르게 구현됨을 재확인:

- **세션 관리**: AT 메모리 저장, RT HttpOnly Cookie, Valkey 블랙리스트 (jti 기반)
- **SQL 인젝션**: 전체 쿼리 파라미터 바인딩 (`%s`) 사용, f-string은 하드코딩 컬럼명만
- **XSS 방어**: SearchPage DOMPurify, CSP 헤더, X-XSS-Protection
- **SSRF 방어**: webhook/avatar URL에 `ipaddress` 모듈 기반 사설 IP 차단
- **인증 강화**: bcrypt(cost 12), 이메일별 로그인 시도 제한, JWT blacklist
- **보안 헤더**: X-Content-Type-Options, X-Frame-Options(SAMEORIGIN), frame-ancestors, Referrer-Policy, Permissions-Policy, CSP 전체 적용
