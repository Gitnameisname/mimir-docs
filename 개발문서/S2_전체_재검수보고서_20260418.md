## 0. 임원 요약 (TL;DR)

`S2_종결보고서.md`는 "잔존 결함 0건, 테스트 940건 100% 통과, AI 품질 기준 충족 ✅"으로 Season 2를 종결 선언하였다. 그러나 본 재검수 결과 **실측 데이터로 뒷받침되지 않는 선언 3건, 개발 규약(S1/S2/글로벌) 위반 6건, 프론트엔드 품질 이슈 17건(에러 10 + 경고 15에서 집계) 등 총 9건의 상위 지적 사항**이 확인되었다. 종결 선언 그 자체는 유지하되, 아래 P0·P1 시정 조치가 완료되기 전까지는 **실질 완료(substantive done)** 가 아닌 **선언적 완료(declared done)** 로 분류함이 적절하다.

가장 중요한 3건:

1. 테스트 커버리지가 `pyproject.toml`의 `--cov-fail-under=80` 기준에 **크게 미달** (실측 line-rate 35.44%).
2. "AI 품질 기준 충족 ✅"은 **골든셋 실측 없이** 선언됨. Phase 3의 AI 품질평가보고서조차 실측값 N/A · 조건부 승인이고, 저장소에 평가 결과 메타데이터가 존재하지 않음.
3. 프론트엔드는 `tsc --noEmit` 7건 · `eslint` 10건의 에러가 남아 있음 — 빌드 파이프라인 게이트가 실제로 막지 못한 것으로 보임.

어느 경우든 **종결 선언의 품질 근거가 결락**되어 있다.

**권고**: 주요 서비스 레이어(`services/llm`, `services/retrieval`, `services/extraction`, `repositories`)에 테스트를 보강해 80% 통과 → `coverage.xml` 과 종결보고서 수치가 일치하도록 재생성.

### 3.2 🔴 F-02 — AI 품질 기준 "달성"은 실측 근거 없음

`CLAUDE.md`:

> AI 기능 포함 FG 추가 의무: AI품질평가보고서 (Faithfulness ≥0.80, Citation-present ≥0.90 등)

`S2_종결보고서.md` §2:

> ✅ Faithfulness ≥0.80, Citation-present ≥0.90, Precision ≥0.85

**재검수 결과**:

1. 존재하는 AI 품질평가보고서는 `docs/개발문서/S2/phase3/산출물/AI품질평가보고서_FG3.md` **한 건뿐**이며, 본문에 다음과 같이 명시:
   - §4 평가 결과: Faithfulness 실측값 **N/A**, Answer Relevance **N/A**, Citation-present **N/A**
   - §1 최종 판정: **조건부 승인**
   - §3 평가 대체 근거: "골든셋 미비로 실측 지표 측정 불가"
2. 저장소 내 골든셋 JSON 파일(`*golden*.json`) **존재하지 않음**. 모델/리포지토리 코드만 존재.
3. 평가 run 결과 메타데이터(`evaluation_metadata*.json`, `evaluation_reports/`)도 **존재하지 않음**.
4. Phase 7 의 CI Gate(`backend/config/evaluation_thresholds.yaml`)는 임계값 정의만 완료. 실행 로그 없음.

→ "AI 품질 기준 충족 ✅"은 **인프라 구축 완료**와 **실측 달성**을 혼동한 선언으로 판단된다. 이는 S1 원칙 ⑤("개발 후에는 검수 수행 및 검수 결과 보고서 작성")·S2 규약("AI품질평가보고서 의무")를 실질적으로 충족시키지 못한다.

**권고**:

- 최소 50건 규모의 골든셋 커밋 + `scripts/evaluate_golden_set.py` 실행 메타데이터 1회 이상 커밋.
- 실측치가 임계값 미달이면 **달성 선언을 철회**하거나, 임계값을 현재 모델·폐쇄망 수준에 맞춰 조정 후 재선언.

### 3.7 🟡 F-07 — 감사 `actor_type` 비필수 처리

`backend/app/audit/emitter.py`:

```python
def emit(
    ...
    actor_type: Optional[str] = None,  # S2 원칙 ⑥: 필수 — 사람(user)과 에이전트(agent)
    ...
):
    ...
    if actor_type:
        log_event["actor_type"] = actor_type
    ...
```

- 시그니처가 `Optional[str] = None` 이고 `if actor_type:` 가드가 걸려 있어, 호출부가 값을 주지 않으면 **감사 로그에서 actor_type 이 통째로 누락**된다.
- S2 원칙 ⑥ "감사 로그에 `actor_type` 필드 필수 기록" 을 엄격 해석하면 위반.

**권고**: 시그니처를 `actor_type: str`(필수) 로 바꾸거나, 최소한 `actor_type or "unknown"` 으로 항상 기록. 엄격 모드에서는 누락 시 예외 발생시키는 것이 바람직.

### 3.8 🟡 F-08 — `llm_base_url` 설정이 실제로 작동하지 않을 가능성

`config.py` 는 폐쇄망 LLM 서버를 위한 `llm_base_url` 을 제공한다:

```python
llm_base_url: str = ""  # 폐쇄망 LLM 서버 (빈 문자열 = 공개 API 사용)
```

그러나 `backend/app/services/rag_service.py:284` `OpenAILLMProvider`:

```python
self._client = AsyncOpenAI(api_key=self._api_key)
# base_url=settings.llm_base_url 가 전달되지 않음
```

→ 폐쇄망에 vLLM / LiteLLM / Ollama OpenAI-호환 서버를 두고 `llm_base_url` 만 설정해도 **공개 api.openai.com 으로 요청이 나간다**. S2 원칙 ⑦("폐쇄망 환경 전 기능 동작") 에 대한 실제 보증이 약함.

보완 fallback 으로 `MockLLMProvider` 가 존재하지만 "API 키가 설정되지 않아…" 라는 에러 메시지 반환에 가까워 degrade 경로도 품질 보증과는 거리가 있다.

**권고**: `AsyncOpenAI(api_key=..., base_url=settings.llm_base_url or None)` 으로 전달. 테스트에서 `llm_base_url` 설정만으로도 폐쇄망 목업 서버와 통신되는지 확인.

### 3.9 🟡 F-09 — Deprecated API 잔존 (글로벌 규칙 #1 위반)

`CLAUDE.md` (글로벌): "deprecated 기능은 대체되는 기능이나 대안을 사용할 것".

- `backend/app/main.py:98,130` — `@app.on_event("startup" / "shutdown")` — FastAPI `lifespan` 컨텍스트로 이관 권고. 현재 코드는 계속 동작하지만 향후 버전에서 제거될 가능성.
- `backend/app/services/search_service.py:682` — `datetime.utcnow()` 호출. 다른 파일들은 이미 주석 처리된 채 대체 완료되었으나 이 파일만 남음. `datetime.now(timezone.utc)` 로 교체 필요.
