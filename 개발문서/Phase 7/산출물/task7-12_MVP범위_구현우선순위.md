## 5. MVP 완료 체크리스트

### 구조/공통

- [ ] Admin UI 공통 레이아웃이 구현되어 있다 (`AdminLayout`, `AdminHeader`, `AdminSidebar`)
- [ ] User UI(`/documents`, `/reviews`)와 Admin UI(`/admin`)의 라우팅이 완전히 분리되어 있다
- [ ] 미인증 또는 권한 없는 사용자가 `/admin`에 접근하면 리다이렉트된다
- [ ] SUPER_ADMIN과 ORG_ADMIN에 따라 메뉴가 다르게 노출된다

### 핵심 화면

- [ ] 시스템 상태 대시보드가 동작하며 핵심 지표 카드 6개를 표시한다
- [ ] 이상 상태 카드가 시각적으로 강조된다
- [ ] 사용자 목록/상세 화면이 동작한다
- [ ] 역할 목록/상세 및 연결 권한 조회가 가능하다
- [ ] 감사 로그 목록 및 기간/이벤트 타입 필터링이 동작한다
- [ ] 감사 이벤트 상세 패널이 동작한다 (변경 전후 표시 포함)
- [ ] DocumentType 목록/상세 조회가 동작한다 (메타데이터 스키마 표시 포함)
- [ ] 백그라운드 작업 상태 목록 조회가 동작한다

### 보안/감사

- [ ] 모든 Admin API 호출에 JWT 권한 검증이 적용된다
- [ ] SUPER_ADMIN 전용 기능이 ORG_ADMIN에게 노출되지 않는다
- [ ] 주요 관리 액션 (상태 변경, 정책 변경 등)이 감사 이벤트로 자동 기록된다
- [ ] DangerDialog가 고위험 액션에 적용된다
