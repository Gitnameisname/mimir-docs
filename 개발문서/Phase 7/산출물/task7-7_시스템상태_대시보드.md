## 2. 핵심 지표 카드 정의서

### 2-1. 카드 항목

| 카드 | 표시 내용 | 이상 조건 | 이동 대상 |
|------|----------|-----------|-----------|
| 총 사용자 | 전체 N명 / 활성 N명 | - | `/admin/users` |
| 활성 문서 | 전체 N건 | - | - (Phase 8 이후) |
| 진행 중 작업 | PENDING+RUNNING N건 | 30건 초과 시 경고 | `/admin/jobs?status=RUNNING` |
| 실패 작업 | FAILED N건 | 1건 이상 시 강조 | `/admin/jobs?status=FAILED` |
| 최근 감사 이벤트 | 24시간 내 N건 (CRITICAL N건) | CRITICAL 이벤트 존재 시 강조 | `/admin/audit-logs` |
| 인덱싱 실패 | FAILED N건 | 1건 이상 시 강조 | `/admin/indexing?status=FAILED` |

### 2-2. 카드 상태 디자인 원칙

| 상태 | 스타일 |
|------|--------|
| 정상 | 기본 카드 스타일 (흰 배경, 회색 테두리) |
| 경고 | 노란 테두리 + ⚠️ 아이콘 + 노란 배경 (옅게) |
| 오류/이상 | 빨간 테두리 + 🔴 아이콘 + 빨간 배경 (옅게) |

```
┌──────────────────────────┐
│  실패 작업               │
│                          │
│  🔴  7건                 │
│  ────────────────────    │
│  ↑3 (지난 1시간)         │
│                          │
│  [상세 보기 →]           │
└──────────────────────────┘
```

## 3. 시스템 컴포넌트 상태 표시 기준 문서

### 3-1. 컴포넌트 목록

| 컴포넌트 | 상태 판단 기준 | 확인 주기 |
|----------|--------------|-----------|
| API Server | `/api/health` 응답 여부 | 30초 |
| Database | 연결 풀 상태 | 30초 |
| Background Job Queue | 큐 대기 시간 기준 | 1분 |
| Indexing Pipeline | 파이프라인 마지막 활동 시각 | 1분 |
| (Phase 8+) Vector Store | 연결 상태 Reserved | - |

### 3-2. 상태 표시 방식

```
┌──────────────────────────────────────────┐
│  시스템 컴포넌트 상태                     │
│                                          │
│  🟢 API Server           정상            │
│  🟢 Database             정상            │
│  🟡 Background Job Queue 지연 (15초)     │
│  🟢 Indexing Pipeline    정상            │
│  ⚫ Vector Store          미구현          │
└──────────────────────────────────────────┘
```

| 아이콘 | 상태 | 설명 |
|--------|------|------|
| 🟢 | HEALTHY | 정상 동작 |
| 🟡 | DEGRADED | 성능 저하 또는 경고 |
| 🔴 | DOWN | 응답 없음 또는 오류 |
| ⚫ | UNKNOWN / NOT_IMPLEMENTED | 미구현 또는 확인 불가 |

## 4. 최근 오류/경고 요약 섹션

```
┌──────────────────────────────────────────────────────┐
│  최근 오류/경고                    [전체 작업 보기 →] │
```

## 6. 이상 상태 우선 가시화 원칙

1. **아무 이상 없을 때**: 모든 카드 기본 스타일, "✅ 전체 시스템 정상" 배너 표시
2. **경고 있을 때**: 해당 카드 노란 강조, 경고 카드를 시각적으로 더 크게
3. **오류 있을 때**: 빨간 카드를 시각적으로 최우선 배치 (상단 또는 눈에 띄는 위치)
4. **다중 오류**: 오류 수가 많을수록 더 강한 시각 표현

### 전체 정상 상태 배너

```
┌──────────────────────────────────────────────────────┐
│  ✅  전체 시스템 정상 — 마지막 확인: 30초 전          │
└──────────────────────────────────────────────────────┘
```

## 7. 대시보드 갱신 정책

| 옵션 | 설명 |
|------|------|
| 수동 새로고침 | 버튼 클릭 시 즉시 전체 데이터 재조회 |
| 자동 갱신 off | 기본값. 수동 새로고침만 |
| 자동 갱신 30초 | 30초마다 API 재호출 |
| 자동 갱신 1분 | 1분마다 API 재호출 |
| 자동 갱신 5분 | 5분마다 API 재호출 |

**구현 방식**: polling (WebSocket은 Phase 후속에서 구현)
