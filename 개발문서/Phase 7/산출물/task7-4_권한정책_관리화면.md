# Task 7-4 산출물: 권한 정책 관리 화면 설계서

---

## 1. 권한 정책 목록 화면 (`/admin/permissions`)

### 1-1. 화면 레이아웃

```
Admin > Permissions

⚠️  이 화면은 SUPER_ADMIN 전용입니다.

┌──────────────────────────────────────────────────────────────┐
│  [🔍 정책명 검색]  [역할 ▾]  [리소스 유형 ▾]  [상태 ▾]      │
├─────────────────────┬──────────────┬──────────────┬──────────┤
│  정책명             │ 리소스 유형  │ 연결 역할 수  │ 상태     │
├─────────────────────┼──────────────┼──────────────┼──────────┤
│ Document Write      │ Document     │ 3            │ 🟢 ACTIVE │
│ System Read         │ System       │ 1            │ 🟢 ACTIVE │
│ Admin Full Access   │ *            │ 1            │ 🟢 ACTIVE │
└─────────────────────┴──────────────┴──────────────┴──────────┘
```

### 1-2. 목록 테이블 컬럼

| 컬럼 | 데이터 | 정렬 |
|------|--------|------|
| 정책명 | `name` | ✅ |
| 설명 | `description` | - |
| 연결된 역할 수 | `role_count` | ✅ |
| 리소스 유형 | `resource_type` | - |
| 허용 액션 | `actions` (태그 형식) | - |
| 상태 | `status` | - |
| 최종 변경일 | `updated_at` | ✅ |

## 3. 역할별 권한 매핑 매트릭스 설계서

### 3-1. 매트릭스 뷰 (`/admin/permissions` 탭 전환)

```
┌──────────────────┬──────────┬──────────┬──────────┬──────────┐
│ 역할 \ 리소스    │ Document │ Workflow │ System   │ AuditLog │
├──────────────────┼──────────┼──────────┼──────────┼──────────┤
│ SUPER_ADMIN      │ ✅ FULL  │ ✅ FULL  │ ✅ FULL  │ ✅ READ  │
│ ORG_ADMIN        │ ✅ READ  │ ✅ READ  │ ❌       │ ✅ READ  │
│ AUTHOR           │ ✅ WRITE │ ✅ READ  │ ❌       │ ❌       │
│ REVIEWER         │ ✅ READ  │ ✅ READ  │ ❌       │ ❌       │
│ APPROVER         │ ✅ READ  │ ✅ WRITE │ ❌       │ ❌       │
│ VIEWER           │ ✅ READ  │ ❌       │ ❌       │ ❌       │
└──────────────────┴──────────┴──────────┴──────────┴──────────┘
```

- 셀 클릭 시 해당 역할-리소스 조합의 상세 정책 목록 표시
- 미정의(❌)는 회색, 허용(✅)은 녹색으로 표시

## 4. 정책 편집 UX 흐름 정의서

### 4-1. 편집 진입

1. 정책 상세에서 "편집" 버튼 클릭
2. 인라인 편집 폼 활성화 또는 편집 모달 표시

### 4-2. 편집 폼 구성

```
┌──────────────────────────────────────────────────────────────┐
│  정책 편집: Document Write Policy                             │
│                                                              │
│  설명: [_________________________________]                   │
│                                                              │
│  허용 액션 (선택):                                            │
│  ☑ CREATE  ☑ READ  ☑ UPDATE  ☐ DELETE  ☐ PUBLISH            │
│                                                              │
│  변경 내역 미리보기:                                          │
│  - 제거: DELETE (기존 포함 → 제거)                           │
│                                                              │
│  ⚠️  영향 받는 역할: 3개 (AUTHOR, EDITOR, SENIOR_AUTHOR)     │
│  ⚠️  영향 받는 사용자: 42명                                   │
│                                                              │
│                      [취소]  [변경 저장]                     │
└──────────────────────────────────────────────────────────────┘
```

### 4-3. 위험 변경 확인 절차

DELETE/PUBLISH 권한 추가, 정책 비활성화, 관리자 역할 정책 변경 시 DangerDialog:

```
┌──────────────────────────────────────────────────────────────┐
│  ⚠️  위험: 권한 정책 변경                                     │
│                                                              │
│  대상: Document Write Policy                                 │
│  변경: DELETE 액션 추가                                      │
│                                                              │
│  영향 받는 사용자: 42명                                       │
│  이 변경은 즉시 적용되어 모든 해당 사용자의                   │
│  문서 삭제 권한이 활성화됩니다.                              │
│                                                              │
│  변경 사유 (필수):                                           │
│  [__________________________________________________]        │
│                                                              │
│           [취소]  [⚠️ 권한 변경 적용 (위험)]                 │
└──────────────────────────────────────────────────────────────┘
```

## 5. 향후 ACL 확장 포인트 정의서

| 확장 포인트 | 현재 상태 | 향후 계획 |
|------------|-----------|-----------|
| 리소스 인스턴스 조건 | 미구현 (Reserved) | 특정 문서 ID 또는 조직 ID 조건 |
| 조건 표현식 (CEL) | Reserved | `resource.org_id == user.org_id` 형태 |
| 정책 상속/우선순위 | Reserved | 역할 계층 기반 정책 상속 |
| 시간 기반 조건 | Reserved | 특정 시간대만 허용 |

**정책 상세 화면에서 확장 포인트 영역은 "향후 구현 예정" 안내와 함께 표시.**
