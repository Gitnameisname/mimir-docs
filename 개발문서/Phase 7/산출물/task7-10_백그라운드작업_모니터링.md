### 1-2. 목록 테이블 컬럼

| 컬럼 | 데이터 | 정렬 |
|------|--------|------|
| 작업 ID | `job_id` | - |
| 작업 유형 | `job_type` | - |
| 대상 리소스 | `resource_type + resource_id + display_name` | - |
| 상태 | `status` | StatusBadge |
| 진행률 | `progress` (RUNNING 시 %) | - |
| 요청자 | `requester.display_name` | - |
| 시작 시각 | `started_at` | ✅ |
| 완료/실패 시각 | `ended_at` | ✅ |
| 소요 시간 | 계산값 | - |

## 2. 작업 상태 전이 표시 정의서

### 2-1. 상태 전이 흐름

```
PENDING → RUNNING → COMPLETED
                 ↘ FAILED
RUNNING → CANCELLED (수동 취소)
FAILED → PENDING (재시도 시)
```

### 2-2. 상태 배지 규칙

| 상태 | 색상 | 특이사항 |
|------|------|---------|
| PENDING | 회색 | - |
| RUNNING | 파란색 | 펄스 애니메이션 |
| COMPLETED | 녹색 | - |
| FAILED | 빨간색 | 행 강조 |
| CANCELLED | 회색 (취소선 텍스트) | - |

## 4. 재시도/취소/강제 종료 액션 UX 흐름 문서

### 4-1. 재시도 (FAILED 상태)

1. "재시도" 버튼 클릭
2. ConfirmDialog: "job-001 작업을 동일 파라미터로 재시도하시겠습니까?"
3. 확인 → 상태 PENDING 전환 → Toast "작업이 재시도 대기열에 추가되었습니다"
4. 원본 FAILED 작업과 재시도 작업 연관 표시
5. **감사 이벤트**: `JOB_RETRIED`

### 4-2. 취소 (PENDING/RUNNING 상태)

**PENDING 취소:**
1. "취소" 버튼 → ConfirmDialog: "대기 중인 작업을 취소하시겠습니까?"
2. 확인 → 상태 CANCELLED → Toast

**RUNNING 취소:**
1. "취소" 버튼 → ConfirmDialog:
   - ⚠️ "현재 처리 중인 작업입니다. 취소하면 처리가 중단됩니다."
2. 확인 → API 호출 → Toast

### 4-3. 강제 종료 (RUNNING 상태, 위험 액션)

1. "강제 종료" 버튼 클릭 (SUPER_ADMIN 전용)
2. DangerDialog:
   ```
   ⚠️  작업 강제 종료
   
   대상: INDEX_BUILD (job-003)
   
   강제 종료는 즉시 프로세스를 중단합니다.
   데이터 무결성이 손상될 수 있습니다.
   
   종료 사유 (필수):
   [_______________________________]
   
          [취소]  [⛔ 강제 종료 (위험)]
   ```
3. 확인 → API 호출 → 상태 CANCELLED → Toast
4. **감사 이벤트**: `JOB_FORCE_STOPPED` (CRITICAL)

## 5. 실패 분류 및 권장 조치 가이드

| 오류 유형 | 설명 | 권장 조치 |
|-----------|------|-----------|
| `RESOURCE_NOT_FOUND` | 대상 리소스 삭제됨 | 작업 정리 (재시도 불필요) |
| `PERMISSION_DENIED` | 권한 부족 | 권한 설정 확인 후 재시도 |
| `EXTERNAL_SERVICE_ERROR` | 외부 연계 오류 | 연계 상태 확인 후 재시도 |
| `TIMEOUT` | 처리 시간 초과 | 리소스 크기 확인 후 재시도 |
| `SYSTEM_ERROR` | 내부 시스템 오류 | 로그 확인 필요 (관리자 연락) |

## 6. 장기 실행 작업 모니터링 규칙

| 임계값 | 처리 |
|--------|------|
| RUNNING 30분 이상 | 작업 목록 해당 행에 ⚠️ 아이콘 + 경고 배지 |
| RUNNING 60분 이상 | 대시보드 "장기 실행 경고" 카운트 증가 |
| RUNNING 120분 이상 | "장기 실행 작업" 별도 배너로 상단 강조 |
