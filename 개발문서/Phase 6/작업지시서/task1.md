

# Task 6-1. User UI 정보 구조 및 화면 맵 설계

## 1. 작업 목적

사용자 UI(User UI)의 전체 구조를 정의하고, 주요 화면 간 이동 흐름을 설계한다.

이 작업의 목표는 다음과 같다.

- 사용자(작성자 / 검토자 / 열람자)가 어떻게 시스템을 탐색하는지 명확히 정의
- 문서 중심 업무 흐름(조회 → 상세 → 편집 → 검토 → 승인)을 UI 관점에서 정리
- 이후 모든 화면 설계의 기준이 되는 **정보 구조(IA, Information Architecture)** 확정

---

## 2. 작업 범위

### 포함 범위

- User UI 전체 메뉴 구조 정의
- 주요 페이지 라우팅 설계
- 문서 중심 사용자 흐름 정의
- 역할별 진입 시나리오 정의
- 공통 레이아웃 구조 초안

### 제외 범위

- 실제 UI 디자인 (스타일, 색상 등)
- 컴포넌트 상세 설계
- API 연동 구현

---

## 3. 선행 조건

- Phase 1: Document / Version / Node 구조 이해
- Phase 2: 역할(Role) 및 권한 모델 정의 완료
- Phase 4~5: 문서 상태 흐름(Draft / Review / Approved / Published) 정의 완료

---

## 4. 주요 설계 대상

### 4-1. 최상위 메뉴 구조

User UI에서 제공할 주요 메뉴를 정의한다.

예시:

- Dashboard (선택)
- Documents
- My Tasks (검토/승인 대상)
- Recent / Favorites (선택)

각 메뉴의 목적과 포함 기능을 명확히 정의한다.

---

### 4-2. 라우팅 구조 (Route Map)

Next.js 기준으로 URL 구조를 설계한다.

예시:

- `/documents`
- `/documents/{document_id}`
- `/documents/{document_id}/edit`
- `/documents/{document_id}/versions`
- `/reviews`

요구사항:

- RESTful하고 일관된 URL 구조
- 문서 중심으로 계층 구성
- 확장 가능한 구조

---

### 4-3. 주요 화면 정의

필수 화면을 식별하고 정의한다.

#### (1) 문서 목록 화면

- 문서 리스트
- 검색/필터
- 정렬

#### (2) 문서 상세 화면

- 문서 내용
- 메타데이터
- 상태 표시
- 액션 버튼 (편집, 검토 요청 등)

#### (3) 문서 편집 화면

- 구조 기반 편집
- 저장 / 임시 저장

#### (4) 버전 관리 화면

- 버전 목록
- 버전 상세 진입

#### (5) 검토/승인 화면

- Review 대상 목록
- 승인 / 반려 / 의견 입력

---

### 4-4. 사용자 흐름 정의 (User Flow)

다음 흐름을 반드시 포함한다.

#### 열람자 흐름

- 문서 목록 → 문서 상세 → 구조 탐색

#### 작성자 흐름

- 문서 생성 → 편집 → 저장 → 검토 요청

#### 검토자 흐름

- 검토 대상 확인 → 문서 상세 → 승인 / 반려

각 흐름은 단계별로 명확히 서술한다.

---

### 4-5. 공통 레이아웃 구조 초안

User UI의 기본 레이아웃을 정의한다.

구성 요소:

- Header
- Sidebar (Navigation)
- Main Content
- Optional Panel (우측 상세 / 트리 / RAG 영역)

레이아웃은 다음을 만족해야 한다.

- 문서 작업에 집중 가능한 구조
- 확장 가능한 패널 구조
- Admin UI와 명확히 분리

---

### 4-6. 역할 기반 진입점 정의

각 역할별 주요 진입 경로를 정의한다.

- 작성자 → Documents → 편집 중심
- 검토자 → My Tasks → 승인 중심
- 열람자 → Documents → 조회 중심

---

## 5. 산출물

다음 문서를 작성한다.

1. User UI 화면 맵 (페이지 구조 다이어그램)
2. 라우팅 구조 정의서
3. 사용자 흐름 정의서

---

## 6. 완료 기준

다음 조건을 만족하면 완료로 본다.

- 전체 User UI 페이지 구조가 명확히 정의되어 있다
- 모든 핵심 화면이 식별되어 있다
- 사용자 흐름(작성자/검토자/열람자)이 명확히 정리되어 있다
- 라우팅 구조가 일관되게 설계되어 있다
- 이후 Task(6-2~6-12)가 이 문서를 기준으로 설계 가능하다

---

## 7. Codex 작업 지침

- 실제 코드 구현은 하지 않는다
- 설계 문서 중심으로 작성한다
- 가능한 한 **구조화된 Markdown**으로 정리한다
- 표, 리스트, 계층 구조를 적극 활용한다
- 추상적 표현보다 **명확한 구조 정의**를 우선한다
