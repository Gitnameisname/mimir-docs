

# Task 6-10. RAG 질의 UI 연계 포인트 설계

## 1. 작업 목적

문서 기반 사용자 UI에 **RAG 질의 기능을 후속 Phase에서 자연스럽게 연결할 수 있도록 UI 연계 포인트를 설계**한다.

이 작업의 핵심 목표는 다음과 같다.

- 현재의 문서 열람 UI에 AI 질의 경험을 무리 없이 확장할 수 있도록 한다
- 문서 범위 질의와 전체 문서 집합 질의를 구분하여 설계한다
- 답변과 근거(citation)가 다시 문서 구조 탐색으로 이어지는 흐름을 정의한다

---

## 2. 작업 범위

### 포함 범위

- 문서 상세 화면 내 RAG 질의 UI 배치 검토
- 질의 입력 방식 정의
- 답변 표시 구조 정의
- citation 기반 문서 이동 UX 정의
- 후속 API / 상태 모델 연계 포인트 정리

### 제외 범위

- 실제 RAG 검색 / 생성 엔진 구현
- 벡터 인덱스 구축
- 답변 품질 평가 체계 구현

---

## 3. 선행 조건

- Task 6-4 (문서 상세 보기 및 구조 탐색 UI 설계)
- Task 6-9 (공통 피드백 및 상호작용 UX 정리)
- Phase 10~11의 검색 / RAG 연계 방향 초안 존재

---

## 4. 주요 설계 대상

### 4-1. RAG 질의 UI 배치 위치

문서 상세 화면에 RAG 질의 기능을 어디에 배치할지 정의한다.

#### 배치 후보

- 우측 사이드 패널
- 하단 보조 패널
- 별도 탭 또는 Drawer

#### 설계 요구사항

- 기존 읽기 흐름을 방해하지 않아야 한다
- 문서 구조 탐색 패널과 충돌하지 않아야 한다
- 필요 시 접기 / 펼치기 가능한 보조 영역이어야 한다

---

### 4-2. 질의 범위 정의

사용자가 어떤 범위에 대해 질문하는지 UI에서 명확히 구분할 수 있어야 한다.

#### 질의 범위 예시

- 현재 문서 기준 질문
- 현재 섹션 기준 질문
- 전체 문서 집합 기준 질문

#### 설계 요구사항

- 기본 범위를 무엇으로 둘지 정의
- 범위 선택 UI(토글, 셀렉트, 탭 등) 검토
- 범위가 답변 품질과 의미에 영향을 준다는 점을 사용자에게 명확히 전달

---

### 4-3. 질의 입력 UI

사용자가 자연어 질문을 입력할 수 있는 UI를 설계한다.

#### 기본 요소

- 질문 입력창
- 전송 버튼
- 범위 선택 UI

#### UX 요구사항

- 짧은 질문과 긴 질문 모두 입력 가능해야 함
- 질의 중 상태 표시 필요
- 이전 질문 재사용 또는 추천 질문 확장 가능성 고려

---

### 4-4. 답변 표시 구조

RAG 응답 결과를 어떻게 보여줄지 정의한다.

#### 기본 구성

- 답변 본문
- 근거 문서 / 섹션 목록
- 신뢰 참고용 citation 표시

#### 설계 요구사항

- 답변과 근거를 시각적으로 분리
- 답변이 길어질 경우 접기 / 펼치기 고려
- 잘못된 확신을 주지 않도록 근거 기반 표현 강조

---

### 4-5. Citation 기반 문서 이동 UX

답변에 포함된 근거(citation)를 클릭했을 때 사용자가 해당 문서 위치로 이동할 수 있어야 한다.

#### UX 흐름 예시

- 답변 내 citation 클릭
- 해당 문서 / 섹션으로 이동
- 문서 본문 내 관련 위치 하이라이트
- 구조 트리도 동기화 가능하면 반영

#### 설계 요구사항

- 같은 문서 내 이동과 다른 문서로 이동을 구분
- 이동 후 사용자가 맥락을 잃지 않도록 표시 필요

---

### 4-6. 상태별 UX 처리

#### Loading

- 질의 처리 중 메시지 표시
- 답변 생성 중 Skeleton 또는 Spinner 표시

#### Empty

- 아직 질문하지 않은 상태 안내
- 예시 질문 제공 가능

#### Error

- 답변 생성 실패 메시지
- 재시도 액션 제공

#### No Citation / Weak Result

- 충분한 근거를 찾지 못한 경우 명시적으로 안내

---

### 4-7. 기존 문서 UI와의 관계 정리

RAG 패널은 문서 UI의 주 기능이 아니라 보조 기능이므로, 우선순위와 상호작용 원칙을 정리한다.

#### 원칙

- 문서 읽기 경험이 항상 우선
- RAG 패널은 보조 도구로 배치
- 문서 편집 화면에서는 제한하거나 분리할지 검토
- 승인 / 검토 화면에서의 사용 허용 범위 검토

---

### 4-8. 후속 API / 상태 모델 연계 포인트

후속 Phase에서 실제 기능을 붙일 수 있도록 필요한 UI 연계 포인트를 정리한다.

#### 예시 항목

- 질의 요청 상태
- 답변 응답 모델
- citation 메타데이터 구조
- 현재 문서 ID / 섹션 ID 전달 방식
- 질의 히스토리 유지 여부
