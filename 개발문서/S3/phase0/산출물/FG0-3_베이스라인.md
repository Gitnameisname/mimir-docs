## 0. 이 보고서의 성격 (⚠️ 반드시 먼저 읽을 것)

본 파일의 아래 통계(§1~§4)는 **리포지토리에 커밋되어 있던 `backend/coverage.xml` (2026-04-18 작성) 스냅샷**에 기반한다. 이 스냅샷은 전체 테스트 스위트가 아닌 **부분 실행 결과**로 보이며, 전체 6.45% 는 "실제 커버 불가능" 이 아니라 **"해당 실행이 많은 모듈을 import 하지 않았다"** 는 뜻에 가깝다.

**정확한 베이스라인**:

- S2 종결 재검수 보고서(2026-04-18)에 기록된 수치는 **35.44%**. 이것을 본 FG 의 출발점으로 간주한다.
- §1~§4 의 표는 **갱신 전 임시값** 이며, 아래 §6 절차에 따라 CI 녹색 1회 후 실제 값으로 덮어쓴다.

## 6. 갱신 절차 (운영자/담당자용)

1. 로컬 혹은 CI 에서 전체 테스트 스위트를 실 DB 환경에서 실행:
   ```bash
   cd backend
   USE_EXISTING_DB=1 \
     pytest tests/ \
     --cov=app \
     --cov-append \
     --cov-report= \
     --cov-fail-under=0
   coverage xml -o coverage.xml
   coverage html -d htmlcov
   ```
   (또는 FG 0-1 의 `backend-integration.yml` CI 가 자동으로 생성 → artifact 다운로드)

2. 본 파일을 재생성:
   ```bash
   cd backend
   python scripts/coverage_baseline.py --xml coverage.xml --markdown \
     > ../docs/개발문서/S3/phase0/산출물/FG0-3_베이스라인.md
   ```

3. 임계값 검증:
   ```bash
   python scripts/coverage_baseline.py --xml coverage.xml --check
   # 모두 통과: exit 0
   # 미달: exit 1 + 상세 항목 stderr 로
   ```

4. 재생성 시 §0 은 스크립트가 덮어쓰지 않는다 — 별도 보존이 필요하면 사본을 두거나, 재생성 후 수동으로 §0 을 맨 위에 붙여넣는다.
