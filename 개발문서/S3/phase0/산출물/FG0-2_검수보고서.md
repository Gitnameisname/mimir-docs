## 1. 검수 범위

본 FG 는 S2 종결 시점 BUG-04 (EMBEDDING_DIM 설정 vs `document_chunks.embedding` 컬럼 차원 불일치) 의 재발을 **Alembic revision 시점** 과 **애플리케이션 기동 시점** 양측에서 단호하게 차단한다.

### 1.1 작업지시서 전제 vs 실제 코드 상태

착수 직후 확인된 갭:

| 항목 | 작업지시서 전제 | 실제 코드베이스 |
|------|---------------|----------------|
| 임베딩 저장 | `document_chunks.embedding VECTOR(<dim>)` (pgvector) | Milvus 중심 (`app/db/milvus.py`) |
| `document_chunks.embedding` 컬럼 | 존재 | **미존재** (init_db / Alembic 어디에도 생성 DDL 없음) |
| VectorRetriever SQL | — | `dc.embedding <=> %s::vector` — 컬럼 부재 상태라 현재 dead path |

**사용자 승인 결정**: "유연한 검증" 모드 (2026-04-23 세션 내). 즉:

- 컬럼 부재 → pass (현재 아키텍처 정합)
- 컬럼 존재 + 차원 일치 → pass
- 컬럼 존재 + 차원 불일치 → **RuntimeError → revision 실패**
- Milvus collection 차원도 best-effort 로 함께 확인

### 1.2 검수 대상 산출물

| 파일 | 역할 |
|------|------|
| `backend/app/db/embedding_dim_check.py` | 공용 검증 헬퍼 (Alembic / Healthcheck 양측 사용) |
| `backend/app/db/migrations/versions/20260423_1500_s3_p0_embedding_dim_check.py` | Alembic revision `s3_p0_embedding_dim_check` (검증 전용, 스키마 변경 없음) |
| `backend/app/api/v1/system.py` (수정) | `/system/health` 에 `embedding_dim` 서브체크 추가 |
| `backend/tests/unit/test_embedding_dim_check.py` | 유닛 테스트 (≥ 4건, 실제 17건) |
| `backend/tests/integration/test_it07_embedding_dim_consistency.py` (수정) | IT-07 skip 해제 + FG 0-1 CI 에 편입 |

## 4. 리스크와 완화

| 리스크 | 심각도 | 완화 |
|--------|-------|------|
| pgvector 컬럼이 추가된 미래 revision 이 FG 0-2 보다 앞에 배치되면 검증이 먼저 돌아 실패 | 중 | Alembic 은 revision 체인 순서대로 처리 — 새 alter revision 은 반드시 `down_revision = "s3_p0_embedding_dim_check"` 로 체이닝해야 함. 운영가이드 Step 2 에 명시. |
| `pg_attribute` 쿼리의 PostgreSQL 버전 호환성 | 낮 | `format_type()` 는 PG 9.4 이상 기본 함수 — pgvector 요구 버전 기준 광범위 호환 |
| Milvus best-effort 검증이 false negative | 낮 | `milvus_match=None` 은 "확인 불가" 로만 보고됨 — degrade 발동 안함 |
| Healthcheck 가 DB 쿼리를 포함하면 응답 시간 증가 | 낮 | `pg_attribute` 조회는 <1ms. 전체 /health 응답 시간 영향 무시 가능 |
| 기존 `test_health_check_with_db` 가 `healthy: True` 만 확인 | 낮 | 현재 상태(컬럼 부재)에선 `healthy: True` 유지되므로 하위 호환 |
| FG 0-2 revision 이 기존 `test_integration_s2_phase0_to_8` 에 영향 | 낮 | 해당 테스트는 mock 기반, 실 DB 접속 없음 — 영향 없음 |
