## 1. 경영진 요약

FG 0-3 의 목표인 백엔드 커버리지 **services 80% + repositories 80% 동시 달성** 을 선언적으로 완료했다. 2026-04-23 에 인프라 작업 완료 후 후속 **15 세션** 에 걸쳐 **34 모듈 / 813 유닛 테스트 / 13,325줄** 의 테스트 코드를 추가했다. sandbox 환경에서는 실 pytest 실행이 불가능하므로, 최종 공식 종결 선언은 운영자가 CI 환경에서 `coverage_baseline.py --check` 를 실행해 `FG0-3_베이스라인.md` 의 실측치를 갱신한 이후 가능하다.

부가 성과: 세션 11 에서 `diff_service.py` 의 `DiffSummaryGenerator.generate()` NameError 잠재 실버그를 발견하고 세션 14 에서 수정 완료 (line 470). 운영 중 diff 기능 전체 장애를 예방했다.

## 4. 패턴 및 기술 부채 정리

### 4.1 테스트 작성 패턴 (조직 표준화 권고)

세션 1~15 에서 반복적으로 사용된 공통 패턴은 팀 표준으로 공유하기 적합하다:

- **MagicMock 커서 스텁**: `__enter__` / `__exit__` + `execute` / `fetchone` / `fetchall` / `rowcount` 5개 속성으로 대부분 psycopg2 기반 리포를 충분히 모킹. 복수 execute 체인은 `side_effect=[row1, row2, ...]`.
- **async 서비스**: `pytest-asyncio` + `AsyncMock` + `AsyncGenerator` 로 LLM 스트리밍 / 커뮤니케이션 레이어 테스트.
- **플러그인 시스템 우회**: `app.plugins.base` 를 `sys.modules` 교체로 mock — DocumentTypeRegistry 체인 (`.instance().get().rag_plugin().get_context_config()`) 재현.
- **보안 검증 테스트**: 내부 예외 메시지가 외부 응답에 노출되지 않는지 `assert "boom" not in response` 로 강제.
- **SQL injection 방어 검증**: sort_by whitelist fallback 동작 확인 (`"malicious; DROP"` 입력 → `created_at` 으로 안전 fallback).

### 4.2 발견된 실버그 (1건 수정 완료)

| 위치 | 증상 | 세션 발견 | 세션 수정 |
|------|------|----------|----------|
| `diff_service.py:470` | `DiffSummaryGenerator.generate()` 가 정의되지 않은 `nodes_a` 변수 사용 → NameError | S11 | S14 (`nodes_a: Optional[list[Node]] = None` 파라미터 추가) |

수정 후 `test_generate_has_known_nameerror_bug` (버그 고정 가드) 는 `test_generate_accepts_optional_nodes_a_param` + `test_generate_uses_nodes_a_for_deleted_parent_lookup` (리그레션 방지) 로 교체되었다.

### 4.3 선언적 vs 실측 갭

현재 커버리지 수치는 소스 라인 × 경로 기반 추정이다. 실측 확정을 위해서는:

```bash
cd backend
pytest tests/ --cov=app --cov-append --cov-report=xml:coverage.xml
python scripts/coverage_baseline.py --xml coverage.xml --check
```

이 명령이 녹색으로 통과하면 FG 0-3 **공식 종결** 이 선언된다.

## 5. 운영자 인수 체크리스트

운영자 또는 CI 환경에서 아래 절차를 수행해 공식 종결을 선언한다:

### 5.1 로컬 실행 (옵션 A)

1. `backend/tests/integration/conftest.py` 의 testcontainers 의존 확인 (docker daemon 필요)
2. 아래 명령 실행:
   ```bash
   cd backend
   pytest tests/unit/ --cov=app --cov-append --cov-report= --cov-fail-under=0
   pytest tests/integration/ --cov=app --cov-append --cov-report= --cov-fail-under=0
   coverage xml -o coverage.xml
   coverage html -d htmlcov
   python scripts/coverage_baseline.py --xml coverage.xml --check
   ```
3. 결과 확인:
   - **exit 0** → FG 0-3 **공식 종결 선언**
   - exit 1 → 미달 영역 확인 후 `FG0-3_진행가이드.md` §4 의 후속 모듈 추가 작업

### 5.2 GitLab CI 실행 (옵션 B — 권장)

1. `backend/.gitlab-ci.yml` 의 `backend:test` 스텝 녹색 3회 연속 확인
2. Artifact `coverage.xml` 다운로드 후 `scripts/coverage_baseline.py --xml coverage.xml --markdown` 실행 → `FG0-3_베이스라인.md` 본문 자동 갱신
3. Git commit & push (운영자 계정):
   ```
   docs(phase0): FG0-3 실측 베이스라인 갱신 — 공식 종결
   ```
