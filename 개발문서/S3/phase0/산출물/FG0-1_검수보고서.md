## 1. 검수 범위

### 3.4 근거 데이터 첨부 (Phase 0 산출물 규약 §3.1)

본 FG 는 CI 최초 구축 단계로, 아직 "CI 녹색 3회 연속" 게이트를 충족하지 못한다. 샌드박스 환경 제약(Docker 미탑재)으로 로컬 실 실행도 불가능하다. 첨부 근거:

1. **정적 검증 로그** — 본 보고서 §3.1 (ast.parse / yaml.safe_load 모두 통과)
2. **파일 인벤토리** — `backend/tests/integration/*.py` 11개 파일 + conftest.py (총 1,695줄)
3. **API 계약 대조** — §3.3 (서버 라우터 정의와 테스트 호출 경로의 일치 확인)
4. **시나리오 매핑 매트릭스** — §2.3 (작업지시서 10건 ↔ 구현 케이스 18건 일대일 대조)

**아직 미첨부 (운영자 작업 필요)**:

- GitHub Secrets 3종 등록 (`CI_POSTGRES_PASSWORD` / `CI_JWT_SECRET` / `CI_INTERNAL_SECRET`)
- CI 실 실행 녹색 run URL (최소 3회)

## 5. 결론 — "선언적 완료 vs 실질적 완료"

본 FG 0-1 은 Phase 0 산출물 규약 §3.2 에 따라 다음 기준으로 완료도를 재평가한다.

| 기준 | 충족 여부 | 근거 |
|------|----------|------|
| CI 녹색 3회 연속 | ⏸ **미충족** | CI 워크플로는 **작성 완료** 상태. Repository Secret 3종 등록 후 실제 CI 실행 + 3회 연속 녹색이 운영자 액션으로 남아 있음. |
| 실측 리포트 | △ 부분 | 정적 검증 리포트(ast/yaml/api-contract)는 본 보고서에 포함. 실행 결과 아티팩트는 CI 최초 녹색 시점에 자동 생성됨 (`backend-integration-results/`). |
| 불일치 케이스 실패 증명 | △ 부분 | IT-06(BUG-01) / IT-03(race) / IT-04(ACL 폴백) 은 의도적 실패 유도 설계. 실제 구현 회귀 시 fail 함을 보이는 것은 첫 CI 녹색 + 의도적 regression PR 으로 증명 가능. |
| 로컬 pytest 단일 명령 실행 | ✅ 설계 완료 | `cd backend && pytest tests/integration/ -m integration` 한 줄로 실행. testcontainers 자동 기동. 샌드박스 Docker 미탑재로 본 세션 실증 불가. |

본 FG 는 **"인프라·시나리오·보고서 세 축의 선언적 완료"** 상태다. **실질적 완료**는:

1. 운영자가 GitHub Secrets 3종을 등록
2. `backend/.gitlab-ci.yml` 의 `backend:integration` job 이 `main` 브랜치에서 3회 연속 녹색
3. 최초 녹색 run URL + CI 아티팩트 링크를 본 보고서 §3.4 에 추가

의 3단계로 전환된다. Phase 0 게이트(§5)의 CI 녹색 3회 연속 조건과 일치하며, Phase 1 이후 기능 FG 착수 직전까지 이 증명이 누적되어야 한다.
