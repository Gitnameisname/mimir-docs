### 2.3 상태 분류 (task0-5 §2.3)

| status | 구현 판정 | 단위 테스트 |
|--------|----------|------------|
| `not_applicable` | `current_published_version_id IS NULL` | `test_not_applicable_when_no_published_version` |
| `pending` | published 존재 + chunks 0 + 실패 기록 없음 | `test_pending_when_published_but_no_chunks` |
| `in_progress` | transient marker 미구현 → pending 에 흡수 (task §6 리스크 표 허용) | (pending 과 동일 경로) |
| `failed` | `audit_events.action_result != 'success' AND event_type LIKE '%vectoriz%' AND occurred_at > last_vectorized_at` | `test_failed_when_recent_failure_after_last_vectorized` |
| `stale` | `latest_published_version_id NOT IN indexed_version_ids` | `test_stale_when_latest_version_not_in_indexed` |
| `indexed` | 위 외 정상 | `test_indexed_when_latest_version_is_indexed` / `test_indexed_ignores_old_failure` |

## 4. 리스크와 완화

| 리스크 | 심각도 | 완화 |
|--------|-------|------|
| `in_progress` 상태 감지 정확도 부재 | 낮 | 현재는 `pending` 으로 수렴 (task §6 리스크 표 허용). UI 뱃지 라벨도 "대기 중" 으로 통합 표시 검토 가능 |
| `audit_events` 가 실패 이벤트를 저장 안하면 `failed` 미탐지 | **중** | `_trigger_vectorization_async` 에서 실패 시 emit 추가 완료. 기존 오류는 소급 탐지 안되나, 이후 발생한 실패는 모두 기록됨 |
| 대용량 문서 재벡터화로 서비스 부하 | 낮 | 기존 `max_workers=3` ThreadPoolExecutor 재사용 + 쿨다운 10초 |
| Valkey 의존성 | 낮 | in-memory 폴백 동작 (단일 프로세스 내 유효). 경고 로그 1회 |
| 작성자 권한 확장의 부작용 (악의적 재벡터화 반복) | 중 | 쿨다운 10초 + 감사 로그 `actor_id/actor_type` + 추가 대응은 보안보고서 §4 Phase 1+ 권고 |
| 상태 폴링이 네트워크 낭비 | 낮 | 2초 간격 + 수렴 시 즉시 중지 + 60초 상한 |
| compact mode 가 좁은 모바일 화면에서 헤더 깨짐 | 낮 | `flex-col items-end` 로 세로 정렬, `flex-1` 로 title wrap. 실 UA 확인은 후속 |
| 프런트 테스트가 STATUS_DISPLAY 매핑만 검증 | 낮 | 버튼 상호작용/폴링/쿨다운은 E2E 로 이관. CI 에서 Playwright 등 도입 시 보강 (Phase 1+) |

## 5. 결론 — "선언적 완료 vs 실질적 완료"

Phase 0 산출물 규약 §3.2 기준 재평가:

| 기준 | 충족 여부 | 근거 |
|------|----------|------|
| 모든 status 분기 응답 | ✅ 유닛 테스트 | 8건 분기 테스트 + integration IT-02 확장 |
| 권한: admin/작성자/제3자 | ✅ | `TestCanReindex` 4건 |
| 쿨다운 429 + Retry-After | ✅ (설계) | `TestCooldown` + 라우트 구현 |
| 버튼 클릭 → 파이프라인 트리거 → indexed 수렴 | ⏸ **후속 E2E** | 샌드박스 불가. IT-02 확장으로 API 레벨은 검증 |
| MCP Tool 에이전트 호출 성공 | ⏸ 후속 | 샌드박스 agent 미접속 |
| 회귀 테스트 ≥ 12 녹색 | ✅ 19건 작성 (실행은 CI) | |
| UI 리뷰 ≥ 5 | ✅ | `FG0-5_UI리뷰로그.md` |
| 폐쇄망 상태 조회 정상 | ✅ 설계 | Milvus/embedding 호출 없음 |
| 검수·보안 보고서 | ✅ | 본 문서 + `FG0-5_보안취약점검사보고서.md` |

본 FG 는 **코드 레벨 선언적 완료** 상태다. **실질적 완료** 는:

1. FG 0-1 CI 녹색 (기본 게이트)
2. 실 환경에서 버튼 클릭 → 재벡터화 → `indexed` 수렴 E2E 증거 수집
3. MCP Tool 호출 샘플 성공
4. 폐쇄망 모드(Milvus off) 실측

로 전환된다.
