## 2. 체크리스트

### 2.1 사내 문서 외부 유출 (OWASP LLM06, A01)

**가장 중요한 고려사항**. 판정 LLM 에는 다음이 전송된다:

- 질문 (공개 OK)
- 답변 (RAG 가 생성한 문서 본문 내용)
- **citation snippets (원본 문서의 일부 그대로)**

OpenAI 선택 시 이 모든 내용이 OpenAI 서버로 전송된다.

| 항목 | 결과 | 상세 |
|------|------|------|
| 기본 provider 는 안전한가 | ✅ | `build_judge()` 기본값 `dummy` — 외부 호출 없음. OpenAI 사용은 명시적 env 설정 필요 |
| Provider 전환 경고 | ⏸ 잔존 | `RAG_JUDGE_PROVIDER=openai` 설정 시 별도 확인 프롬프트 없음. **권고**: rag_quality 실행 시 OpenAI 전환 감지하면 `confirm to continue [y/N]` 대화형 확인 추가 |
| 폐쇄망 우선 문서화 | ✅ | `FG0-4_AI품질평가보고서.md` §2 에서 Ollama 를 권장으로 명시 |
| `vllm` 프로바이더로 사내 모델 사용 가능 | ✅ | `RAG_JUDGE_BASE_URL=http://vllm.internal:8000` 으로 사내 엔드포인트 지정 |

**권고 (잔존)**: `rag_quality.py` 시작 시 `RAG_JUDGE_PROVIDER=openai` 인 경우:

```python
if provider == "openai":
    logger.warning("⚠️ OpenAI 판정 — citation snippet 이 OpenAI 에 전송됩니다.")
    if not os.environ.get("ACK_EXTERNAL_JUDGE"):
        raise SystemExit(
            "외부 판정 수행 의도를 명시하려면 ACK_EXTERNAL_JUDGE=1 설정 후 재실행하세요."
        )
```

형식의 명시적 opt-in 추가를 Phase 1+ 권고로 남김.

### 2.2 API key 관리 (OWASP A02, CWE-798)

| 항목 | 결과 | 상세 |
|------|------|------|
| 코드에 하드코딩된 키 | ✅ | 없음. 모두 `os.environ.get("RAG_JUDGE_API_KEY")` |
| 로그에 키 출력 | ✅ | `logger.info("judge: provider=%s model=%s")` 만 기록. key 자체는 `Authorization: Bearer ...` 헤더에만 사용 |
| 리포트 JSON 에 키 포함 | ✅ | `report["judge"]` 에 provider/model 만 저장, api_key 미저장 |
| `.env` 파일 권고 | ✅ | AI품질평가보고서 §2 에 `export RAG_JUDGE_API_KEY=...` 명시 (shell history 대신 `.env` 사용 권고 추가 가능) |
| CI secret 참조 | ✅ | FG 0-1 CI 워크플로와 일관 — `${{ secrets.RAG_JUDGE_API_KEY }}` 경로 권장 |

### 2.3 실측 JSON 의 민감정보 (OWASP A01)

`FG0-4_실측_*.json` 은 다음을 포함한다:

- 각 쿼리의 `query` (공개 OK)
- 각 쿼리의 `answer` (RAG 생성 답변 — **사내 문서 내용 일부 반영**)
- `results[*].faithfulness.unsupported_claims` (답변 중 근거 없다고 판단된 문장)
- `results[*].faithfulness.reasoning` (판정 LLM 의 설명, 답변 요약 포함 가능)

| 항목 | 결과 | 상세 |
|------|------|------|
| citation snippet 이 리포트에 포함되는가 | ⚠️ 제한적 | `rag_smoke.QueryResult.citations[*].snippet` 은 리포트에 그대로 저장됨. 사내 기밀 문서면 주의 |
| `unsupported_claims` 에 원본 문구가 들어갈 수 있는가 | ⚠️ | LLM 응답이므로 원문에서 발췌될 가능성 있음. 200자 제한 적용 (`[str(c)[:200] for c in ...]`) |
| `.gitignore` 권고 | ✅ | AI품질평가보고서 §7 에 명시. `docs/개발문서/S3/phase0/산출물/FG0-4_실측_*.json` 을 외부 repo push 전 확인하라고 명시 |

**권고 (잔존)**: 리포트 저장 시 snippet 을 hash (sha256 처음 8자) 로 대체하는 옵션 제공. 현재는 수용.

### 2.4 judge LLM 응답 파싱 (OWASP A03, A08)

| 항목 | 결과 | 상세 |
|------|------|------|
| `_extract_json_blob()` 가 eval/exec 사용하는가 | ✅ | `json.loads` 만 사용. 코드 실행 위험 없음 |
| 악의적 JSON (prototype pollution 등) | ✅ | Python `json.loads` 는 결과를 dict/list 로만 반환. JS 와 달리 prototype 개념 없음 |
| 응답 길이 제한 | ✅ | `reasoning = str(blob.get("reasoning") or "")[:500]`, `unsupported_claims` 각 200자 × 10개 상한 |
| `_SIMPLE_JSON_RE` 의 재귀 regex | ✅ | `(?R)` 사용하지 않음 (Python 미지원). 단순 greedy match 만 사용 — ReDoS 위험 낮음 |
| JSON 파싱 실패 시 NaN 처리 | ✅ | `FaithfulnessResult(score=None, reasoning="parse_failed")` |

### 2.5 로컬 LLM / 내부망 노출 (OWASP A10 SSRF)

| 항목 | 결과 | 상세 |
|------|------|------|
| `RAG_JUDGE_BASE_URL` 에 임의 URL 주입 가능 | ⚠️ 제한적 | 환경변수는 운영자 제어 — 외부 주입 불가. CI 에서 secret 에 저장 |
| 내부망 endpoint (localhost:11434) 호출 정당성 | ✅ | 의도된 동작. Ollama/vLLM 사내 배포 |
| 타임아웃 | ✅ | OpenAI/vLLM 30s, Ollama 60s 기본. 무한 대기 방지 |
| 리다이렉트 / 인증 우회 | ✅ | urllib 기본 동작. 응답 JSON 파싱만 수행 |
