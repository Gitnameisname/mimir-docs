## 4. 설계 결정 반영

### 4.1 뷰 선호 저장 위치 — users.preferences JSONB (채택)
- Alembic revision 1건 + JSONB free-form. 향후 테마/언어 등 확장에도 유연
- 허용 키 엄격화는 애플리케이션 레이어(`UserPreferences` pydantic) 에서. S1 ② generic + config 원칙

### 4.2 PATCH semantics — shallow merge + null 삭제
- task1-3.md Q2 결정: shallow (1 레벨). 중첩 키가 필요해지면 Phase 3 에서 재설계
- null 을 "해당 키 삭제" 로 해석 (JSON:merge-patch 부분 차용)

### 4.3 Debounce 400ms
- task1-3.md Q1 — 토글 빈도 고려. 200~500ms 범위에서 400ms 선택

### 4.4 S2 ⑤ 원칙 (에이전트 = 1급 소비자)
- `/account/preferences` 는 에이전트가 호출해도 동작. actor_type 감사 기록
- "agent 의 view_mode 가 의미 있는가" 는 Phase 3 에서 재검토 (task1-3 R3)

## 6. 알려진 제약

1. **실 TipTap 런타임 검증 부재** — jsdom 미설치 + npm registry 차단으로 headless Editor 실행 불가. NodeId extension 의 `appendTransaction` 동작(신규 블록 auto-UUID, split 후 새 id) 은 integration / 실 브라우저 smoke 에서 확인.
2. **Integration 실 실행은 운영자 후속** — sandbox 에서는 testcontainers 기동 불가 (Docker 미제공).
3. **뷰 모드 변경 PATCH 가 optimistic 으로 즉시 UI 반영되지만 네트워크 장애 시 rollback + toast** — 사용자가 최소 1회는 오프라인 상태 반영을 감내할 수 있음. 오프라인 우선 전략은 Phase 2 이후 검토.
4. **users.preferences 컬럼이 기존 수백만 users 가 있는 실 DB 에서 ADD COLUMN DEFAULT** — PG 11+ 에서는 instantaneous metadata-only change. 본 프로젝트 배포 환경은 PG 16 (`docker-compose.yml` 확인 완료) 이므로 문제 없음.
