## 3. XSS / HTML 인젝션 방어

### 3.1 TipTap schema 기본 동작

TipTap/ProseMirror 는 **whitelist schema** 로 동작한다. 허용되지 않은 노드 타입/마크는 파싱 시 자동 폐기:

- `<script>`, `<iframe>`, `<object>`, `<embed>` → schema 에 없으므로 drop
- `<a href="javascript:...">` → schema 의 link 마크가 URL 을 수용하나 TipTap StarterKit 에는 link 자체가 없음 (별도 extension 필요). 본 FG 에서 link extension 미도입
- `on*=` (onclick 등) 인라인 핸들러 → schema attrs 에 명시되지 않았으므로 parseHTML 에서 무시

### 3.2 NodeId extension 의 `data-node-id` 안전성

```ts
parseHTML: (el) => el.getAttribute("data-node-id"),
renderHTML: (attrs) => attrs.node_id ? { "data-node-id": attrs.node_id } : {},
```

- `data-*` 속성은 HTML 표준 user data 채널로 XSS 벡터 아님
- `node_id` 값이 임의 문자열이 올 수 있으나 **서버 측 `schemas/versions.py::validate_content_snapshot` 가 content 구조를 검증**. 향후 FG 1-3 에서 UUID v4 형식 검증도 추가 고려

### 3.3 HTML 붙여넣기

TipTap 의 paste handler 도 schema 경유. 외부 사이트에서 복사된 HTML 은 허용 노드만 남기고 나머지는 plain text 로 downgrade.

### 3.4 Placeholder 안전성

`@tiptap/extension-placeholder` 는 CSS `::before` 의 `content: attr(data-placeholder)` 로 문자열을 DOM 속성으로 렌더. attr() 은 CSS 수준에서 텍스트만 표시하며 HTML 로 해석되지 않음.

## 4. 권한 / ACL

본 FG 는 권한 체계를 건드리지 않는다. 저장 경로는 FG 1-1 의 PUT `/documents/{id}/draft` 그대로이며 백엔드의 `authorization_service.authorize(action="draft.save")` 가 여전히 게이트. `readOnly` prop 은 `useAuthz().can("document.edit")` 결과로 결정되어 권한 없으면 에디터 자체가 렌더되지 않음.

## 5. 폐쇄망 동등성 (S2 ⑦)

본 FG 에서 추가된 코드 중 외부 HTTP/CDN 호출 **0건**:

- `@tiptap/*` 패키지는 번들 내 OSS 코드만 참조
- `crypto.randomUUID` 는 브라우저/Node 내장 API
- globals.css 에 web font 로드 없음 (프로젝트 기본 폰트만)
- Placeholder/NodeId extension 모두 JS 로컬 실행

## 6. SSR 안전성

Next.js 16 + React 19 SSR 에서 TipTap Editor 가 서버 측에서 인스턴스화되면 `crypto.randomUUID` 호출 / `document.createElement` 호출 등이 실패한다. 본 FG 의 방어:

```ts
useEditor({
  // ...
  immediatelyRender: false,  // TipTap 3.x 공식 SSR 권장 옵션
});
```

클라이언트 hydration 이후에만 Editor 초기화. 서버 번들에는 NodeId extension 코드가 포함되지만 `crypto.randomUUID` 는 호출되지 않는다.

추가 안전장치: `DocumentTipTapEditor` 파일 상단 `"use client"` 지시어. Next.js App Router 가 서버 사이드 평가를 막는다.

## 7. 단축키 충돌 / 위험

### 7.1 `Cmd/Ctrl + Alt + M` (뷰 토글)

- macOS: 기본 예약 없음
- Windows Chrome: 기본 예약 없음
- Ubuntu / GNOME: 일부 윈도 매니저에서 `Alt+M` 을 메뉴 활성화로 쓸 수 있으나 `Ctrl+Alt+M` 조합은 드묾
- 한글 입력 상태에서 `Alt+M` 은 IME 에 따라 동작 달라질 수 있음. UI 리뷰 3회차에서 재검토 예정

### 7.2 `Cmd/Ctrl + S` (저장)

- 기존 동작 유지. 브라우저 기본 "페이지 저장 대화상자" 를 `preventDefault()` 로 차단

### 7.3 우발적 충돌 대응

단축키는 **순수 UI 전환** 만 수행 (뷰 모드 변경) 또는 이미 승인된 저장 경로 호출. 권한 / 데이터 변경 / 폭발적 부작용 없음. 최악의 경우 사용자 혼란만 가능.
