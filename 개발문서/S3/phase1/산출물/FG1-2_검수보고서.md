## 1. 구현 요약

FG 1-1 에서 확정된 `content_snapshot` 단일 정본 위에 TipTap 3.22 기반 에디터를 이식했다. `DocumentNode[]` DIY 블록 편집기는 제거됐고, `editor.getJSON()` 결과를 그대로 저장 바디로 사용하는 구조로 전환. 블록 뷰 ↔ 일반 뷰 토글은 같은 `EditorState` 위에서 CSS class 분기로 구현되어 재마운트 없이 동작한다.

## 4. 설계 결정 반영

### 4.1 D2 — TipTap + 커스텀 NodeId (✓)
- `@tiptap/starter-kit` + `@tiptap/extension-placeholder` + 커스텀 `NodeId` 조합
- `node_id` 는 **snake_case** 로 통일 (서버 snapshot_sync_service 와 동일 키). 직렬화/역직렬화 중간 변환 0건
- `appendTransaction` 플러그인으로 setContent / paste / 분할 등 모든 경로에서 신규 블록에 UUID 자동 부여

### 4.2 Q1 (단축키 충돌) — `⌘/Ctrl + Alt + M` 임시 결정
- Cmd+Shift+V (일반 텍스트 붙여넣기), Cmd+Shift+B (북마크 바) 모두 브라우저 예약. 충돌 회피
- 최종 확정은 UI 리뷰 3회차에서 검토

### 4.3 SSR 호환
- Next.js 16 + React 19 조합에서 TipTap 3.22 의 SSR 경고를 피하기 위해 `immediatelyRender: false` 공식 권장 옵션 적용

### 4.4 하드코딩 금지 재확인
- 블록 타입 어휘(heading/paragraph/bulletList/orderedList/codeBlock) 는 NodeId extension 의 `options.types` 기본값에만 존재. DocumentType 분기 로직 0건
- 뷰 모드는 `"block" | "flow"` 타입 상수 (`EditorViewMode`)
