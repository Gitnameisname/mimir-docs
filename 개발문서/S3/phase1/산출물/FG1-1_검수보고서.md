## 1. 구현 요약

## 4. 설계 결정 반영

### 4.1 D1 — content_snapshot 단일 정본 (✓)
- 쓰기 경로 3개(`save_draft` / `save_draft_nodes` / `agent_proposal_service`) 모두 content_snapshot 을 기준으로 저장하고 nodes 를 파생 동기화
- validator 가 비표준 포맷(`type=="text"` / `type=="document"` / dict 아님) 을 단일 게이트에서 차단

### 4.2 Q1 — schema_version 필드 (✓ 도입)
- `CONTENT_SNAPSHOT_SCHEMA_VERSION = 1` 상수 + doc 루트에 포함

### 4.3 Q2 — MCP Tool 로 save_draft 노출 (보류)
- 본 FG 에서는 `agent_proposal_service` 경로 유지. Phase 3 에서 재검토 (인계 메모)
- S2 ⑤ 원칙(에이전트 = 1급 소비자) 준수 위해 `agent_proposal_service` 도 동일한 `snapshot_sync_service` 를 경유
