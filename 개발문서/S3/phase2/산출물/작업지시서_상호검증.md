## 7. 파생 동기화 트랜잭션 규약 (본 검증에서 확정)

Phase 2 의 쓰기 경로 (`save_draft`, `publish_version`, `agent_proposal` 승인) 에서 파생 동기화는 **다음 순서로, 단일 트랜잭션 내에서** 수행:

```python
with get_db() as conn:
    # 1) 정본
    versions_repository.upsert_content_snapshot(conn, version_id, content_snapshot)

    # 2) 파생 (순서 보장)
    snapshot_sync_service.rebuild_nodes_from_snapshot(conn, version_id, content_snapshot)
    snapshot_sync_service.rebuild_tags_for_version(conn, document_id, version_id, content_snapshot, metadata)
    snapshot_sync_service.rebuild_wikilinks_for_document(conn, document_id, content_snapshot, viewer_scope_ids)

    # 3) commit 은 라우터 경계 (CLAUDE.md §4.2)
```

- **실패 시 전량 롤백** — 파생 중 하나라도 예외 → 전체 트랜잭션 롤백 → 정본 이전 상태 유지
- nodes / tags / wikilinks 셋은 **version 단위(nodes)** vs **document 단위(tags, wikilinks)** 혼재. 기준 단위는 호출자 경계에서 합의된 document_id + version_id 로 전달.

task2-2, task2-3 작업지시서에 본 규약을 반영하기 위한 **후속 수정 권장 포인트** — §4 Step 의 "연쇄 호출" 위치에 본 §7 링크 추가.
