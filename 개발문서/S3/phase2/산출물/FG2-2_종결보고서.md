## 9. 실측 중 발견 + 수정한 버그 2건

### 9.1 BUG-FG22-01 — `documents_service.update_document` 에 `rebuild_tags_for_document` 연쇄 누락

**증상**: TagChipsEditor 에서 `#정책` 입력 → Enter → PATCH 200 + 토스트 "태그가 저장되었습니다" 정상이지만, `document_tags` 테이블이 갱신되지 않아 칩이 UI 에 나타나지 않음.

**원인**: FG 2-2 백엔드 구현 당시 `save_draft` / `agent_proposal` 경로에만 `rebuild_tags_for_document` 를 호출하고, 일반 PATCH 경로 (`update_document`) 에는 연쇄 호출을 추가하지 않았음. TagChipsEditor 는 `metadata.tags` 를 PATCH 로 직접 수정하는 흐름이라 이 경로를 탐.

**수정** (`backend/app/services/documents_service.py`):
- `request.metadata is not None` 분기에서 `versions_repository.get_by_id` 로 현재 활성 draft/published snapshot 조회
- `rebuild_tags_for_document(conn, document_id=..., snapshot=..., metadata=updated.metadata)` 동일 트랜잭션에서 호출
- 활성 version 이 없거나 version row 결손 시 `snapshot=None` 로 안전 fallback (metadata 만으로 재계산)

**회귀 테스트**: `tests/unit/test_documents_service_update_rebuild_tags.py` 5건 (metadata 변경 / title only no-op / empty request no-op / no active version / version row 결손).

### 9.2 BUG-FG22-02 — HashtagMark 에서 `#` 접두사 소실 + mark 경계 확장

**증상-A**: 본문에 `#ai ` 입력 시 HashtagMark mark 는 적용되지만 `.tag-pill` 안 텍스트가 "ai" 만 보이고 `#` 가 사라짐.

**증상-B**: `#ai` 뒤에 이어 쓴 `world` 가 같은 mark 에 흡수되어 `<span class="tag-pill">#aiworld</span>` 로 합쳐짐.

**원인-A**: TipTap `markInputRule` 은 "last capture group" 의 텍스트만 mark 대상으로 보존. 정규식이 `(#([\p{L}\p{N}_/-]{1,64}))` 였기에 last group = `ai` (내부 group) 만 보존되고 바깥 `#` 는 제거됨.

**원인-B**: ProseMirror Mark 의 `inclusive` 속성이 기본값 `true` — mark 경계 끝에 타이핑하면 mark 가 자동으로 확장됨.

**수정** (`frontend/src/features/editor/tiptap/extensions/HashtagMark.ts`):
- INPUT_REGEX / PASTE_REGEX 를 `(#[\p{L}\p{N}_/-]{1,64})` 단일 캡처 그룹으로 재편 → last group 이 `#ai` 전체
- `getAttributes`: `match[1].replace(/^#/, "").toLowerCase()` 로 data-tag 속성값 정규화
- Mark 정의에 `inclusive: false` 추가

**회귀 테스트** (`frontend/tests/HashtagMarkFg22.test.tsx`):
- 기존 6건을 `match[1]` + `extractTag()` 헬퍼 기반으로 재작성
- 신규 1건: "대문자 입력 → lowercase 로 정규화"
- 전체 node:test 258 → **259 passed / 0 failed**
