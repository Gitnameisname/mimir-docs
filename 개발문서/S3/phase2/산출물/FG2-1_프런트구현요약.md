## 2. 설계 판단

### 2.1 URL state 를 "정본" 으로

- 컬렉션/폴더/하위포함 은 **URL query** 로 관리 (`?collection=`, `?folder=`, `?include_subfolders=true`). useState 내부 상태는 URL 변경 시 `useEffect` 로 동기화.
- 사이드바 선택 시 `<Link>` 로 이동 → URL 이 바뀜 → filters 반영. 뒤로가기 / 공유 URL 모두 자연스럽게 동작.

### 2.2 Optimistic update 기본 방침

| 훅 | Optimistic | 이유 |
|----|-----------|-----|
| `useCreateCollection` | 성공 후 prepend | 서버가 id 발급 — create 전 optimistic 은 key 불일치 위험 |
| `useUpdateCollection` | onMutate 에서 list 갱신 | 이름/설명 변경은 UX 가 빠르면 좋음 |
| `useDeleteCollection` | onMutate 에서 제거 | 실패 시 rollback 으로 복구 |
| `useCreate/Rename/MoveFolder` | invalidate 만 | path 재계산이 서버에서 발생 (rename/move 는 하위 폴더 path 까지 변함) → 서버 신뢰 |
| `useDeleteFolder` | onMutate 에서 제거 + rollback | 빈 폴더만 허용이므로 단순 제거 |
| `useSetDocumentFolder` | invalidate | 문서 쪽 목록 갱신 필요 |
| `useAddDocumentsToCollection` | 성공 후 document_count 증분 | Scope 밖 문서는 서버가 조용히 제외하므로 실 inserted 수 기반 반영 |

### 2.3 Scope 밖 문서 추가 안내 톤

- `useAddDocumentsToCollection` 의 `onSuccess` 에서 `rejected > 0` 이면 "n개 추가됨. 접근 권한이 없어 m개는 제외되었습니다." **info** 토스트. 구체 id 는 노출 안 함 (FG 2-0 §3 "존재 유출 방지" 규약)
- 전부 reject 면 (Scope 미스매치) "info" 톤으로 알리고 실패로 취급하지 않음

### 2.4 FoldersTree 의 자동 전개

활성 폴더의 조상을 자동으로 펼쳐준다 (`useEffect` 로 `parent_id` 체인 순회 → `expanded` Set 에 추가). URL 로 `?folder=<id>` 로 진입해도 해당 노드가 보이도록 보장.

### 2.5 Sidebar 레이아웃

- 기존 `SidebarNav flex-1` 이 남은 공간 모두 점유 → `shrink-0` 로 바꾸고 **`SidebarExploreTree` 가 `flex-1 min-h-0 overflow-y-auto`** 로 세로 공간 분배 + 자체 스크롤
- 하단 `SidebarUserPanel` 은 원래처럼 고정
- rail (collapsed) 모드에서는 CollectionsTree / FoldersTree 가 **아이콘만** 렌더 (사이드바 오픈 시 전체 트리로 전환)

### 2.6 DocumentAssignControls — 현재 상태 비노출 이유

현재 문서의 `folder_id` 는 별도 API (`GET /document_folder/{id}` 같은 조회) 가 없으므로 이 라운드는 **"변경 액션" 만** 제공. 드롭다운 기본값은 빈 선택("— 폴더 해제 —"). 현재 상태 표시는 후속 FG (또는 `GET /documents/{id}` 응답에 folder_id 포함) 로 이월.
