## 1. 전체 흐름

```
서버 (정본)                       프런트 (소비 + UX)
────────────                     ─────────────────
/documents/{id}                   documentsApi.get
  └ document_tags:[]  ───▶        └ adapter 에서 {id,name,source} 보존
                                  └ DocumentDetailPage 상단에 TagChipsEditor 마운트
  PATCH /documents/{id}
   ├ metadata.tags:[] ◀───        useSetDocumentTags
                                  (TagChipsEditor 의 칩 추가/제거)
/tags/popular                     usePopularTags
  └ items:[{name,usage_count}]───▶└ Sidebar TagsSection (상위 20)
/tags?q=<term>                    useTagAutocomplete
                                  (TagChipsEditor 의 자동완성 dropdown)
/documents?tag=<name>             documentsApi.list({tag})
  └ ACL + 태그 IN(:name)          └ DocumentListPage 가 ?tag= 을 filter.tag 로 전달
```

본문 에디터의 `#tag` 는 시각 강조용 (`HashtagMark`) 이며 **서버 재파싱 시점에만 document_tags 가 갱신**된다. 클라이언트 state 로 document_tags 를 직접 조작하지 않는다.

### C. 문서 상세 — TagChipsEditor (칩 편집 위젯)

- `src/features/tags/TagChipsEditor.tsx` (신규)
  - 규약
    - **frontmatter 태그만 편집**. inline (source === "inline") 은 회색 readonly chip 으로 표시.
    - frontmatter / both 칩은 `×` 버튼으로 제거 가능.
  - 키보드: Tab / `,` / Enter 확정, Backspace 로 마지막 칩 제거, Esc 로 드롭다운 닫기, ↑↓ 로 자동완성 이동.
  - 자동완성: 150ms debounce. 드롭다운은 Escape / 외부 클릭으로 닫힘.
  - 클라이언트 normalize (`normalizeClient`): 서버 `normalize_tag` 와 **동일 규약**을 선적용.
    `NFKC + toLowerCase + [\p{L}\p{N}_/-]{1,64}`. 서버가 최종 정본.
  - 테스트 편의용: `export const __test__ = { normalizeClient }`.

- `src/features/documents/DocumentDetailPage.tsx`
  - `DocumentAssignControls` 바로 아래에 `<TagChipsEditor ... />` 마운트.
  - `currentTags = doc.document_tags ?? []`, `baseMetadata = doc.metadata ?? {}` 전달.

### 3.4 회귀 방어

- `useSetDocumentTags` 는 `metadata.tags` 만 교체하고 다른 metadata 키는 보존 (`baseMetadata` 병합).
- adapter 가 source 알 수 없는 값을 안전하게 `"inline"` 으로 fallback → readonly 로 떨어지고 사용자가 잘못된 태그를 실수로 제거할 수 없게 함.
- HashtagMark 정규식은 서버 `tag_rules.py` 와 동일 문자 집합 (`\p{L}\p{N}_/-`, 1..64). 본 파일 테스트에 동일 정규식 복제본을 심어 파싱 계약 드리프트를 감지.
