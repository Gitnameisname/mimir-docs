## 4. 아키텍처 최종 상태

### 4.1 DB (Alembic)

```
s3_p1_users_preferences (Phase 1)
  └─ s3_p2_documents_scope_profile  (FG 2-0)
       └─ s3_p2_collections_and_folders  (FG 2-1)
```

### 4.2 신규 테이블

| 테이블 | 설명 |
|--------|------|
| `collections` | owner 소유 임의 문서 집합. UNIQUE(owner_id, name) |
| `collection_documents` | N:M 연결 (collection_id, document_id) PK |
| `folders` | self-referencing 계층 + materialized path + depth ≤ 10 CHECK |
| `document_folder` | N:1 (document_id PK, folder_id) |
| (`documents.scope_profile_id` 컬럼 추가) | FG 2-0 Scope 바인딩, ACL 단독 근거 |

### 4.3 API 추가

| 경로 | 설명 |
|------|------|
| `GET/POST /api/v1/collections` | 목록/생성 |
| `GET/PATCH/DELETE /api/v1/collections/{id}` | 단건/수정/삭제 |
| `POST /api/v1/collections/{id}/documents` | N:M 추가 (Scope 밖 조용히 reject) |
| `DELETE /api/v1/collections/{id}/documents/{doc_id}` | N:M 제거 |
| `GET /api/v1/collections/{id}/documents` | 컬렉션 내 문서 id (viewer Scope 필터) |
| `GET/POST /api/v1/folders` | 전체 트리 / 루트 생성 |
| `GET/PATCH/DELETE /api/v1/folders/{id}` | 단건/이름 변경(path 재계산)/삭제 |
| `POST /api/v1/folders/{id}/move` | 부모 변경 (순환 방지 + 깊이 상한) |
| `PUT /api/v1/documents/{id}/folder` | 문서 폴더 배치/해제 |
| (`GET /documents`, `GET /search` 에 `collection` / `folder` / `include_subfolders` / `q` 추가) | 리스트와 FTS 모두 동일 규약 |
