## 2. 설계 판단 / 결정 사항

### 2.1 ACL 모델 단순화

| 경우 | 동작 |
|------|-----|
| `actor is None` (레거시/내부 호출) | 필터 skip |
| `actor.role in {SUPER_ADMIN, ORG_ADMIN}` | 필터 skip |
| `actor.scope_profile_id` 있음 | `[actor.scope_profile_id]` IN 필터 |
| 인증되었으나 scope 없음 | `[]` — 결과 없음 (S2 ⑥ 차단) |

**Why**: 관리자도 Profile 을 가지지만 Profile 안의 scope 정의 전개(예: `all`)를 현 시점에 해석하려면 scope_definitions 테이블을 건드려야 하는데, 이는 FG 2-0 범위를 넘음. **role 기반 bypass** 로 단순화하고, 향후 admin 의 "자기 Profile + 위임받은 Profile" 집합을 union 으로 확장하는 여지를 `_resolve_viewer_scope_profile_ids` docstring 에 남김.

### 2.2 존재 유출 방지 (404)

Scope 밖 문서를 `get_by_id` 에서 403 이 아닌 **None → 404** 로 돌려보낸다. 서비스 레이어에서 None 이면 `ApiNotFoundError`. 클라이언트는 문서 존재 여부를 유추할 수 없음.

### 2.3 update 전 ACL 확인

`update_document` 가 update 호출 전 `get_document` 로 ACL 을 먼저 통과시킴. Scope 밖 문서의 update 는 get 단계에서 404 로 차단되어 UPDATE 문이 실행되지 않는다. 회귀 테스트 `test_update_enforces_acl_before_mutation` 에서 검증.

### 2.4 search_service / retrieval 경로 분할

| 경로 | FG 2-0 범위 |
|------|-----|
| `search_service.search_documents` | ✓ — documents.d.scope_profile_id 필터 추가. viewer_scope_profile_ids 파라미터를 상위 호출자가 전달할 수 있게 훅 마련. **기본값 None** 이므로 하위호환 유지 |
| `retrieval/*` (FTS/vector/hybrid) | ✗ — 현재 `document_chunks` 의 **delegated ACL** (is_public + accessible_roles/users/orgs) 로 커버. FG 2-0 에서 document-level scope 필터까지 추가하려면 retrieval 전체 설계 재검토 필요. **별도 FG 로 이월**하되 문서화 |

CLAUDE.md §4.1 "폴백 경로(FTS, 로컬 모델)에서도 동일한 ACL 적용 필수" 는 retrieval 에 대해 delegated ACL 이 이미 동작하므로 안전망은 존재. 이중 적용은 보강 차원이므로 별도 FG 로 분리.

### 2.5 update 는 scope_profile_id 변경 불가

FG 2-0 범위에서는 documents 의 scope_profile_id 를 **create 시점에만 주입** 하고, update 경로는 scope 를 바꾸지 않는다. Scope 재할당은 별도 admin API (scope_profile_id 변경) 로 후속 FG 에서.
