## 3. 영향 분석 (제7조 4축)

### 3.1 State (저장 모델)

- **DB 스키마 변경 0** — envelope 은 응답 직렬화 단계의 메타. 어떤 테이블도 신규/변경 없음.

### 3.2 Intent (명세·계약)

- `MCPResponse` 에 신규 Optional 필드 `envelope: MCPReadEnvelope | None`. 외부 클라이언트가 envelope 을 무시해도 기존 본문 키 동작 그대로 (호환성 회귀 테스트 통과 — TestBackwardCompat).
- search_documents 의 `data.results[*]` 에 신규 `_envelope` 키 (Optional). 기존 키 (`document_id` / `document_title` / `content` / `relevance_score` / `citation` 등) 모두 보존.
- `mimir://` URI 4 패턴 표준 확정 — 이전엔 node URI 패턴만 정의되어 있었으나 doc/version/render 패턴이 추가.

### 3.3 Policy (권한·정책)

- R4 (instruction_authority=none) 가 코드 표면에 정착 — Literal 단일값 강제로 코드 어디서도 `system`/`user` 설정 불가.
- R6 (detected_risks 경고) 정책 코드화 — 응답 차단 분기 없음, 메타 부착만.
- 차단 정책은 클라이언트 책임 — 운영 가이드에 명시 (보안보고서 §3.4).

### 3.4 Execution (실행 / 가역성)

| 변경 | 가역성 |
|---|---|
| envelope 스키마 추가 | 가역 — git revert |
| URI 빌더 / parser | 가역 — 백워드 호환 (parse_resource_uri 가 node 패턴만 받던 동작 유지) |
| `_build_envelope` / `_enrich_search_items_with_envelope` | 가역 — 헬퍼 제거 시 envelope 만 빠지고 기존 응답 본문 그대로 |
| `_run_injection_detection` 가 read_annotations 도 처리 | 가역 — 추가형 (기존 search/fetch 동작 보존) |

비가역 변경 0.
