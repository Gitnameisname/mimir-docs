## 4. 영향 분석 (제7조 4축)

### 4.1 State

- **신규 컬럼**: `agent_proposals.idempotency_key VARCHAR(128)` + UNIQUE INDEX (agent_id, idempotency_key) WHERE NOT NULL
- 기존 행 영향 0 (NULL 허용, backfill 불필요)
- Alembic revision 적용 직전 `@최철균` 승인 필수

### 4.2 Intent

- 신규 도구 `save_draft` (L2 / experimental / write_audited) 1개
- TOOL_SCHEMAS 8 → 9
- `MCPResponse` 에 `write_envelope: Optional[MCPWriteEnvelope]` 신규 필드 (read envelope 와 분리)
- `propose_draft` 시그니처에 `content_snapshot` (Optional) + `idempotency_key` (Optional) 추가 — backward compat (default 값 보존)

### 4.3 Policy

- **MCP 표면 최초 쓰기 도구** — 별도 P1 게이트
- `default_enabled=False` — 운영자가 ScopeProfile.allowed_tools 에 명시 등록 의무
- `policy_profile="write_audited"` — Admin UI 가 색상 구분 (orange) 으로 시각 경고
- agent self-approve 거부 — `agent_proposal_service.approve_draft` 의 기존 actor.role 검사 그대로
- 자동 머지 0 — propose 만 (reviewer approval 후 별도 trigger)

### 4.4 Execution (가역성)

| 변경 | 가역성 |
|---|---|
| Alembic revision | 가역 — DROP COLUMN downgrade 가능 (idempotency_key 정보 손실 명시) |
| Schema 추가 (SaveDraftRequest / Data / DraftImpactPreview / MCPWriteEnvelope) | 가역 |
| TOOL_SCHEMAS save_draft entry | 가역 — `status="not_exposed"` 로 즉시 비활성화 가능 |
| tool_save_draft 함수 + dispatcher 분기 | 가역 |
| event_types.md 4종 등재 | 가역 |
| compute_draft_impact / propose_draft 시그니처 확장 | 가역 (default 값 보존) |

비가역 변경 0. **운영 영향**: ScopeProfile 에 save_draft 등록한 후 운영 시작 → 중단 시 ScopeProfile 에서 제거 (즉시 비노출).
