#### 2.3.3 save_draft 도구 호출 동작

```bash
# agent 토큰으로 호출
curl -X POST https://your-mimir/api/v1/mcp/tools/call?tool_name=save_draft \
  -H "Authorization: Bearer ${AGENT_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "params": {
      "document_id": "<test-doc>",
      "content_snapshot": {"type": "doc", "content": [{"type": "paragraph", "content": [{"type": "text", "text": "test"}]}]},
      "idempotency_key": "test-key-1",
      "reason": "smoke test"
    }
  }' | jq '{
    success,
    data: {proposal_id, status, requires_human_approval, audit_event},
    write_envelope: .write_envelope,
    envelope: .envelope
  }'

# expected:
# success: true
# data.status: "proposed"
# data.requires_human_approval: true
# data.audit_event: "agent_proposal.requested"
# write_envelope.content_role: "mutation_proposed"
# write_envelope.instruction_authority: "none"
# envelope: null   <-- read envelope 부재 (write 도구 의도)
```

#### 2.3.4 Idempotent Replay 검증

```bash
# 같은 key 로 두 번 호출
RESPONSE_1=$(curl -X POST ... -d '{... "idempotency_key": "key-X" ...}')
RESPONSE_2=$(curl -X POST ... -d '{... "idempotency_key": "key-X" ...}')

# proposal_id 같음 + 두 번째 응답 message 에 'idempotent' 또는 '동일' 포함
PROPOSAL_1=$(echo $RESPONSE_1 | jq -r '.data.proposal_id')
PROPOSAL_2=$(echo $RESPONSE_2 | jq -r '.data.proposal_id')
[ "$PROPOSAL_1" = "$PROPOSAL_2" ] && echo "✅ idempotent" || echo "❌ FAIL"
```
