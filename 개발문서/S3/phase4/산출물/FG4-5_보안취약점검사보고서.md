## 4. 발견 사항 / 결정

| # | 분류 | 발견 | 결정 |
|---|---|---|---|
| 1 | 의도된 fail-soft | Admin UI 의 manifest fetch 실패 시 도구 토글 비활성 + retry 버튼 | **의도** — 운영자 명시 재시도. fail-open 보다 안전 |
| 2 | 의도된 forward-compat | `mcp_admin_full_view` 가 unknown 필드도 보존 | **의도** — 새 manifest 필드 추가 시 endpoint 코드 변경 없이 자동 노출 (admin only 라 안전) |
| 3 | 운영 영향 | `use_defaults=True` 옵션 신설 — 기존 ScopeProfile 자동 등록 정책 변경 가능 | **의도된 옵트인** — default false 라 기존 운영 영향 0. 운영자가 의식적으로 use_defaults=True 호출 시에만 |
| 4 | UI 미합의 | policy_profile 색상 (read_safe=green / write_audited=orange / experimental=yellow) | UI 리뷰 시점에 합의 — 본 FG 는 임시 정책 |
| 5 | 외부 호환 | strict schema 클라이언트가 새 필드 (requires 등) 거부 가능 | 추가형 변경 — 무시 가능. SDK 업데이트는 별 작업 |
