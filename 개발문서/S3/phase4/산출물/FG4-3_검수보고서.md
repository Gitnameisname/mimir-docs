## 2. 5중 검증 동작 매트릭스

| # | 검사 | 입력 | True | False | None |
|---|---|---|---|---|---|
| 1 | exists | (자동) | document/version/node 모두 존재 | 어느 하나라도 없음 | — |
| 2 | pinned | (자동) | version_id 가 published/archived | latest 거부 (즉시 raise) 또는 draft | — |
| 3 | hash_matches | content_hash + citation_basis | SHA-256(정본 텍스트) == content_hash | 불일치 | — |
| 4 | quoted_text_in_content | quoted_text | 본문에 포함 | 본문에 미포함 | quoted_text 미입력 시 |
| 5 | span_valid | span_offset + span_length | 둘 다 입력 + offset≥0 + length>0 + offset+length≤len(text) | 위 조건 위반 | offset 또는 length 미입력 시 |

종합: `verified = exists AND pinned AND hash_matches AND (quoted_in is not False) AND (span_valid is not False)`.
선택 검사 (None) 는 통과 처리.

## 5. 영향 분석 (제7조 4축)

### 5.1 State (저장 모델)

DB 스키마 변경 0 (Disagreement Record 적응안). citations 테이블 자체가 없으므로 마이그레이션 부재가 자연.

### 5.2 Intent (명세·계약)

- `Citation` Pydantic 모델: `citation_basis: Literal["node_content","rendered_text"]` 필드 추가 (default `node_content` — 호환).
- `VerifyCitationRequest`: `citation_basis` / `quoted_text` / `span_length` 신규 (모두 Optional/default — 호환).
- `VerifyCitationData`: `checks` 필수 + `rendered_snapshot` / `node_snapshot` 추가. 백워드 호환을 위해 `content_snapshot` / `hash_matches` / `version_valid` 유지.

### 5.3 Policy (권한·정책)

- R3 강제: `latest` 입력 즉시 거부 (도구 진입점) — 도구 호출자가 published vN 으로 사전 resolve 필요.
- pinned 정의: `published` 또는 `archived` 만 허용. draft 거부.

### 5.4 Execution (실행 / 가역성)

| 변경 | 가역성 |
|---|---|
| Citation 모델 필드 | 가역 — git revert |
| VerifyCitationRequest/Data | 가역 (호환 필드 유지) |
| tool_verify_citation 재작성 | 가역 — 기존 인터페이스 보존, 신규 입력은 default 로 채워짐 |
