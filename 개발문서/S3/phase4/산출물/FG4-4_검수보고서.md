## 3. CI 게이트 등록 (운영자 후속)

### 3.1 contract_drift 게이트

`.gitlab-ci.yml` (또는 동등) 에 다음 추가 — **운영자 승인 후 적용**:

```yaml
contract_drift:
  stage: test
  script:
    - cd backend && pytest tests/integration/contract_drift/ -v --no-cov
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'
    - if: '$CI_COMMIT_BRANCH == "main"'
  needs: ["unit_tests"]
  allow_failure: false   # 운영자 합의 후 차단 모드. 등록 직후 1주는 allow_failure: true (warning-only).
```

**머지 차단 정책 활성화 절차** (task4-4 §8 리스크 #2 대응):
1. 등록 직후 1주: `allow_failure: true` (warning-only) — 기존 PR 영향 0
2. 운영자가 main 3회 연속 녹색 확인
3. 합의 후 `allow_failure: false` 활성화 → 머지 차단

### 3.2 manifest_drift 게이트

```yaml
manifest_drift:
  stage: test
  script:
    - cd backend && python scripts/dump_mcp_manifest.py --check
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'
  needs: ["unit_tests"]
  allow_failure: false
```

차이가 있으면 fail — 개발자가 `python scripts/dump_mcp_manifest.py` 재실행 후 commit.
