## 1. 위협 모델 (T1~T15)

| # | 위협 | 영향 | 본 FG 의 대응 |
|---|---|---|---|
| T1 | search_nodes 가 ScopeProfile 외 노드 반환 | ACL 우회 | `apply_scope_filter` 결과 → `allowed_doc_ids` post-filter (search_documents 와 동일 패턴). 단일 doc_id 시 SQL 단계 필터, 다중 시 Python post-filter |
| T2 | search_nodes snippet 에 prompt injection 페이로드 → 외부 Chatbot 우회 | jailbreak | `_enrich_search_node_items_with_envelope` 가 per-item prompt_injection_detector 실행 → `_envelope.detected_risks` 부착. 응답 본문 보존 (R6) |
| T3 | read_document_render 가 다른 사용자의 비공개 문서 본문 반환 | ACL 우회 | `_ensure_document_allowed(conn, document_id, acl_extra)` 호출 — fetch_node 와 동일 게이트. document.read 권한 미통과 시 ApiNotFoundError → MCPError |
| T4 | rendered_text 에 인젝션 페이로드 (문서 자체 위변조) | jailbreak | `_run_injection_detection` 가 `rendered_text` 추출 + envelope.detected_risks 부착. R6 — 응답 차단 안 함 |
| T5 | render_hash 가 비결정 (공백/줄바꿈 변동) → citation 검증 깨짐 | 인용 무결성 | `_walk_blocks_for_text` 가 단일 진입점에서 텍스트 빌드. plain_text 는 마지막 rstrip 으로 trailing newline 제거. markdown 은 항상 마지막 \n 으로 정규화 |
| T6 | resolve_document_reference 가 잘못된 문서를 high confidence 로 반환 → 환각 | 인용 환각 | (1) `confidence_threshold` 기본 0.85 — 미달 시 `needs_disambiguation=true` (2) AI 품질 평가 의무 — Precision@1 ≥ 0.85 미달 시 release 차단 |
| T7 | resolve 의 후보 생성 단계가 ACL 누락 → 다른 scope 문서 노출 | ACL 누설 | 5 단계 모두 `allowed_doc_ids` 검사 — `_stage_exact_title` / `_stage_alias` / `_stage_recent_context` / `_vector_semantic` / `_fts_fallback` 각 함수에 분기 |
| T8 | resolve 응답에 단독 `"latest"` 가 포함되어 후속 인용 시 다른 버전 받음 | 인용 변질 | `_Candidate.version_ref` default = `"latest_published"`. R3 회귀 테스트 ([TestR3LatestNeverReturned](../../../../backend/tests/integration/test_mcp_tools_fg42.py)) |
| T9 | resolve 가 외부 LLM 호출에 의존 → 폐쇄망 (S2 ⑦) 깨짐 | 폐쇄망 호환 깨짐 | `_stage_semantic_or_fts` 가 `MIMIR_OFFLINE=1` 또는 벡터 인프라 fail 시 자동 `_fts_fallback`. 폐쇄망 회귀 테스트 통과 |
| T10 | search_nodes 의 결과량 폭발 → 토큰 비용 폭증 / DoS | 자원 남용 | top_k Pydantic 제약 (1~100) + 응답 envelope 의 `items_truncated` 플래그 |
| T11 | read_document_render 의 `version_id="latest"` 입력이 race condition 으로 다른 vN 반환 | 인용 변질 | `resolve_version_id` 가 `VersionsRepository.get_current_published` 호출 — 그 시점의 published 버전. 응답에 resolved vN 명시. (race 가 발생해도 응답이 명시 vN 이라 후속 검증 가능) |
| T12 | manifest 가 코드와 drift (TOOL_SCHEMAS 와 dispatcher 분기 불일치) | 운영 인지 가능성 | `_CURATED_TOOLS` 가 `mcp_exposed_tool_schemas()` 정본. dispatcher 가 unknown tool 호출 시 INVALID_REQUEST. 신규 3 도구 모두 dispatcher 에 분기 추가 — [TestManifestRegistration::test_curated_tools_includes_three](../../../../backend/tests/integration/test_mcp_tools_fg42.py) |
| T13 | resolve 의 candidate 응답에 다른 사용자의 메타 정보 (alias / metadata) 노출 | 정보 누설 | ACL 통과 문서만 후보 — alias 단계가 `allowed_doc_ids` 미통과 결과 제거. 단, alias 검색 SQL 자체가 metadata::jsonb 의 `aliases`/`previous_titles` 만 읽어 다른 PII 미노출 |
| T14 | search_nodes 의 oversample (top_k * 2) 가 oversample 단계에서 너무 많은 행 fetch → 메모리 폭증 | 자원 남용 | `min(top_k * 2, 100)` 으로 상한. top_k=100 시 oversample=100 (변화 없음) |
| T15 | resolve_document_reference 의 reference 입력 자체에 prompt injection 페이로드 (악의적 사용자) | 자체 인젝션 | reference 는 본문 텍스트가 아니라 검색 query 로만 사용. `_safe_ts_query` 가 토큰 분할 후 `&` 결합 — SQL injection 차단. 응답 자체는 메타이므로 R4 (instruction_authority=none) 그대로 |

## 4. 발견 사항 / 결정

| # | 분류 | 발견 | 결정 |
|---|---|---|---|
| 1 | 환각 위험 | resolve 가 high confidence 로 잘못된 문서 반환 가능 | **AI 품질 평가 의무** — Precision@1 ≥ 0.85 미달 시 release 차단. 운영자 실측 대기 |
| 2 | 결정성 | render_hash 가 공백 변동에 민감 | `_walk_blocks_for_text` 단일 진입점 + 양 모드 normalize. FG 4-3 (verify_citation 강화) 가 본 hash 사용 시 결정성 회귀 추가 권고 |
| 3 | 폐쇄망 | semantic 단계가 벡터 인프라에 의존 | `_stage_semantic_or_fts` 의 `try/except` 가 자동 FTS fallback. `MIMIR_OFFLINE=1` 환경변수 우선 — 명시적 폐쇄망 모드 |
| 4 | 자원 남용 | search_nodes oversample → 메모리 | top_k * 2 ≤ 100 상한. 모니터링 시 추가 SQL 필터 (별 라운드) |
| 5 | alias 메타 | `_stage_alias` 가 `documents.metadata.aliases` 가정 — 미존재 시 빈 결과 | **의도** — fail-soft. 운영자가 alias seed 후 활성화 |
| 6 | reference SQL injection | `reference` 입력이 ts_query 에 들어감 | `_safe_ts_query` 가 토큰 분할 + `&` 결합 — 직접 사용자 SQL 미통과 |

## 5. 정합 규칙 (R1~R6 준수 확인)

본 FG 는 R1~R6 모두 영향. 검수보고서 §1 에 상세. 요약:

- **R1 (L4 차단)**: ✅ 신규 3 도구 모두 L0/L1
- **R2 (Scope Profile 단일 ACL)**: ✅ 3 도구 진입점 게이트 + ACL post-filter
- **R3 (pinned)**: ✅ resolve 의 version_ref + render 의 resolved version_id
- **R4 (instruction_authority=none)**: ✅ FG 4-1 envelope 표준 그대로
- **R5 (REST 1:1 복제 금지)**: ✅ 3 도구 모두 도메인 코어 호출 (wrapper 아님)
- **R6 (detected_risks 차단 아닌 경고)**: ✅ search_nodes / read_document_render 본문 인젝션 탐지 + envelope 메타만
