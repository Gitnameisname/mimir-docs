## 1. L0~L4 등급 정의 (CONSTITUTION 제20조 Typed Action Risk)

| 등급 | 정의 | 예시 | MCP 노출 |
|---|---|---|---|
| **L0** | 순수 읽기, 부작용 없음, 가역 | search, fetch, status 조회 | ✅ 가능 |
| **L1** | 검증·해시 등 읽기 + 결정성 검사. DB 변경 없음 | verify_citation, validate | ✅ 가능 |
| **L2** | 단일 사용자 의도의 쓰기, 가역 (draft, undo 가능) | create_draft, save_annotation | 조건부 (FG 4-6) |
| **L3** | 워크플로 전이·상태 변경, 사람 승인 필요 | publish, approve, transition | ❌ MCP 금지 (REST 관리자 only) |
| **L4** | 비가역·파괴적·데이터/스키마 영향 | delete_document, change_schema, reindex | ❌ MCP 영구 금지 (R1) |

## 2. 현재 + 신규 도구 매핑 표

본 표는 **5 현재 도구 + 3 신규 도구 (FG 4-2) + 4 REST 관리자 비노출 도구**를 망라한다. 운영자가 한 번에 검토하기 위함.

### 2.1 현재 MCP 노출 도구 (5)

| # | 도구 | 의도 | risk_tier | maturity | status | exposure_policy | 근거 |
|---|---|---|---|---|---|---|---|
| 1 | `search_documents` | FTS/벡터 검색 | **L0** | stable | enabled | MCP_ENABLED | 순수 read, ACL 통과, 부작용 0. 토큰 폭발 방지로 `top_k` 상한 50 (이미 적용) |
| 2 | `fetch_node` | 노드 전문 조회 | **L0** | stable | enabled | MCP_ENABLED | 순수 read, document_id+node_id 명시 호출, ACL 통과 |
| 3 | `verify_citation` | Citation 5-tuple 해시 검증 | **L1** | stable | enabled | MCP_ENABLED | 결정성 검사, 본문 일부(content_snapshot) 반환. FG 4-3 강화 후에도 L1 유지 (해시·존재성 검사 자체는 부작용 없음) |
| 4 | `mimir.vectorization.status` | 문서 벡터화 상태 조회 | **L0** | beta | enabled | MCP_ENABLED | 읽기 전용 — 재벡터화는 사람 UI 만 (FG 0-5 결정). beta — S3 P0 도입 후 운영 데이터 부족 |
| 5 | `read_annotations` | 인라인 주석 조회 | **L0** | beta | enabled | MCP_ENABLED | 읽기 전용 — 쓰기는 본 도구에 미노출 (FG 3-3 결정). beta — S3 P3 도입 후 운영 데이터 부족 |

## 4. tools/list 응답 노출 정책

`tools/list` (현재 `_CURATED_TOOLS` + `TOOL_SCHEMAS` 기반) 의 외부 응답에 `risk_tier` 를 포함할지 결정:

| 옵션 | 외부 응답 필드 | 장 | 단 |
|---|---|---|---|
| **(A) 보수** (기본 권고) | `name`, `description`, `maturity`, `inputSchema`, `authentication` | 운영 정보 누출 0. 외부가 등급을 모르므로 우회 기획 불가 | 클라이언트가 위험 인지 못 함 (개별 도구 wrap 필요) |
| (B) 부분 노출 | (A) + `risk_tier` | 클라이언트가 L0/L1 분기 가능 | 우회 시도 정찰 정보 제공 |
| (C) 전체 노출 | (A) + 전체 manifest | 디버깅/통합 용이 | 운영 정보 모두 누출 |

**권고: 옵션 A** — 외부에는 `risk_tier`/`status`/`exposure_policy` 비노출. `maturity` 만 노출 (stable/beta/experimental — 운영자 별 클라이언트 호환 안내용).

운영자용 별 endpoint (`GET /admin/mcp/manifest`, FG 4-5 이월) 에서 전체 manifest 조회 가능하도록 분리.

## 5. ScopeProfile.allowed_tools 화이트리스트 (§2.1.6 정합)

§2.1.6 의 `ScopeProfile.allowed_tools` 등록 가능 도구는 **TOOL_SCHEMAS.name 와 정확히 일치**해야 한다. 본 매핑 표의 §2.1 (1~5) + 향후 §2.2 (6~8) 가 화이트리스트 후보 전체.

```python
# backend/app/schemas/mcp.py 에 헬퍼 추가 권고
def known_tool_names() -> frozenset[str]:
    """TOOL_SCHEMAS 에 등재된 모든 도구 이름. ScopeProfile.allowed_tools 검증에 사용."""
    return frozenset(s["name"] for s in TOOL_SCHEMAS)
```

`ScopeProfile.allowed_tools` 에 `not_in known_tool_names()` 인 항목은 Repository CRUD 단계에서 reject 하거나 warn (FG 4-0 §2.1.6.b 결정).
