## 2. 정적 검사

### 2.1 정책 게이트 (`should_expose_viewers`)

```python
def should_expose_viewers(viewer_actor: Optional[ActorContext]) -> bool:
    if viewer_actor is None: return False           # T1: anonymous 차단
    if not viewer_actor.is_authenticated: return False  # T1: 미인증 차단
    sp_id = getattr(viewer_actor, "scope_profile_id", None)
    if not sp_id: return False                       # 누락 시 차단
    return _get_expose_viewers_for_profile(sp_id)
```

→ 모든 진입점 fail-closed False.

### 2.2 _get_expose_viewers_for_profile

```python
try:
    with get_db() as conn:
        repo = ScopeProfileRepository(conn)
        profile = repo.get_by_id(scope_profile_id)
    value = bool(profile.settings.expose_viewers) if profile else False
except Exception:
    logger.warning(...)
    value = False  # T8: DB 장애 차단
```

→ DB 실패 / not-found 모두 차단.

### 2.3 ContributorsService 결합

```python
effective_include_viewers = bool(include_viewers) and should_expose_viewers(viewer_actor)
```

→ 두 조건 AND. 한 쪽이 false 면 viewers fetch 자체 안 함.

### 2.4 Admin PATCH 권한

```python
def update_scope_profile(...):
    _require_admin(actor)
```

→ admin role 외 차단 (T4 OK). organization_id 검증 미적용 (T5 잔존 — §3 참조).

### 2.5 마이그레이션 안전성

```sql
ALTER TABLE scope_profiles
    ADD COLUMN IF NOT EXISTS settings_json JSONB NOT NULL DEFAULT '{}'::jsonb;
```

- IF NOT EXISTS — 재실행 안전
- NOT NULL DEFAULT '{}' — 기존 row 모두 빈 dict 채워짐
- `_settings_from_raw({})` → ScopeProfileSettings() → expose_viewers=False
- 운영 도입 시 기본값 False 보장

### 2.6 PATCH body validation

```python
class ScopeProfileSettingsSchema(BaseModel):
    expose_viewers: bool = Field(default=False, ...)
```

- Pydantic 자동 type 검증 (string / int 등 → 422)
- expose_viewers 외 키는 schema 무시 (extra="ignore" 가 default — Pydantic v2)
