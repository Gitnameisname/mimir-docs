## 3. 변경의 위험 분석

| 위험 | 가능성 | 영향 | 완화 |
|-----|-------|-----|-----|
| node_id anchoring 회귀 (Phase 1) | 낮 | 매우 높 | rebuild_annotation_anchoring 가 nodes_from_prosemirror 의 결과만 사용. Phase 1 회귀 테스트 영향 없음 |
| save_draft 트랜잭션 안에서 anchoring 실패 시 전체 롤백 | 낮 | 중 | mark_orphans 가 단순 UPDATE 쿼리. 실패 가능성 낮음. 실패 시 호출자 트랜잭션 자연 rollback |
| 멘션 알림 폭주 | 중 | 중 | per (author, recipient) 분당 5 건 rate-limit + service skip 시 silent log |
| 자기 자신 멘션 알림 | 낮 | 낮 | service 가 author == recipient 시 skip |
| 답글 cascade 삭제 | 중 | 중 | DDL `parent_id REFERENCES annotations(id) ON DELETE CASCADE` — 답글 자동 삭제 |
| 작성자가 아닌 사용자 본문 수정 | 매우 낮 | 매우 높 | service `update_content` 가 author_id 본인만 허용 (admin 도 본문 수정 X) |
| MCP 권한 우회 (에이전트가 다른 scope 문서의 주석 조회) | 낮 | 매우 높 | tool_read_annotations 가 (1) authorize document.read 재검증 (2) annotations_service.list_for_document 가 ACL get_document 통과 — 이중 게이트 |
| content XSS | 낮 | 중 | 서버 raw 저장. React 가 자동 escape (dangerouslySetInnerHTML 사용 안 함) |
| SQL injection via document_id | 매우 낮 | 매우 높 | psycopg2 params 바인딩만 사용 |
| 다중 워커 알림 dedup | 중 | 낮 | rate-limit 은 DB 쿼리 기반 — 워커 무관 |

## 5. 재검수 (1회차)

자기 점검 추가 발견:

1. ✅ `annotations.parent_id ON DELETE CASCADE` 가 답글 자동 삭제. 별도 service 호출 불필요.
2. ✅ `is_admin` 함수가 `actor.role` 만 보고 ADMIN_ROLES 매칭 — admin role enum 동기화는 별 라운드 (BE-G6 ADMIN_ROLES 단일 진실점 사용)
3. ⚠️ MCP `tool_read_annotations` 가 ACL 차단 시 `MCPError(NOT_FOUND)` 로 매핑 — 정확히 ACL 의 "존재 유출 방지" 시맨틱 보존.
4. ✅ 답글의 답글 평탄화는 service create_annotation 에서 처리 (parent_id 가 답글이면 root 로 변환). UI 도 동일 시맨틱.
5. ⚠️ TipTap mark/gutter 미도입 — 작업지시서 §2.1.8/2.1.9 의 핵심 일부 누락. **결정 사유**: AnnotationsPanel + REST + MCP + 알림 4가지로도 vertical slice 가치 충분. mark/gutter 는 ProseMirror 통합 비용 + 별 ADR 가치 큼. UI 검수 §3 #1 에 기록 + 종결 보고서 §5 잔존에 명시.
