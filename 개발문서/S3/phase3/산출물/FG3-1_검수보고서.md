## 3. 변경의 위험 분석

| 위험 | 발생 가능성 | 영향 | 완화 |
|-----|-----------|-----|-----|
| `document.viewed` emit 폭주로 audit_events 비대 | 중 | 중 | per (actor, doc) 5분 dedup helper. env 로 윈도우 / max_entries 조정 가능. 7일 retention (별도) 와 결합 권장 |
| 다중 워커 환경에서 dedup 누수 | 중 | 낮 | 워커별 독립 LRU. 최대 워커 수 배수 만큼 중복 emit 가능 (4 워커 = 4배). Strict cluster-wide dedup 은 Valkey 도입 후 별 라운드 |
| audit_events.document_id 인덱스 미존재 → 집계 느림 | 낮 | 중 | 기존 `idx_audit_events_document_id` 존재 (connection.py L218). since 필터로 범위 추가 제한 가능 |
| viewers 노출이 정책 없이 기본 켜짐 | 매우 낮 | 매우 높 | 본 FG 의 default 는 false. FG 3-2 가 정책 게이트 결합 (다음 task) |
| `get_many_by_ids` 의 invalid UUID 입력 → DB error | 낮 | 낮 | 입력 시 try/except UUID() 검증 → silently skip |
| display_name 누락된 user → 사용자 ID 노출 | 낮 | 중 | service 에서 `_placeholder_display_name("user")="(알 수 없는 사용자)"` 로 마스킹 |
| service actor / agent emit → contributors 패널 오염 | 중 | 낮 | throttle helper 가 `actor_id=None` (anonymous / service) 면 emit 자체 skip. agent 는 정상 표시 (S2 ⑤ 일관성) |

## 5. 재검수 (1회차)

본 보고서 §1~§4 의 항목을 자기 점검한 결과 추가 발견:

1. ❗ `viewed_throttle.py` 의 `_window_seconds()` 가 음수일 때 `max(0, value)` → 0 = dedup off. 운영자가 `-1` 같은 실수 시 emit 전부 발화. **현재 동작 의도임** (테스트로 강제). UI/로그에 환경변수 검증 경고는 별 라운드.
2. ❗ `get_many_by_ids` 의 silently skip 이 디버깅 어려움 가능. 로그 추가 검토 → 본 라운드 보존 (호출자 책임 명시).
3. ✅ `contributors_repository.list_editors` 의 SQL 에서 `actor_user_id` 정규식 검증 없음 — DB 가 VARCHAR(255) 라 SQL injection 리스크 있는가? params 바인딩 사용하므로 안전.
4. ✅ `contributors_service` 의 `_normalize_actor_type` 가 alien 값을 'user' 로 매핑 — 보안적으로 보수적 (alien 가 agent/system 권한으로 격상되지 않음).

→ 본 보고서 1차 + 재검수 모두 통과.
