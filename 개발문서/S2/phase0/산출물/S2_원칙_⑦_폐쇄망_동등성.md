# S2 원칙 ⑦ — 폐쇄망 동등성

**확정일**: 2026-04-16
**근거**: Season 2 개발계획.md §1, §2.2, §3.6

---

## 1. 정의

모든 기능은 외부 네트워크 없이 동작 가능해야 한다.  
외부 의존(OpenAI, LangSmith, OTel Cloud 등)은 환경변수로 `off` 가능하고,  
`off` 시에도 서비스는 **degrade될 뿐 실패하지 않는다**.

---

## 2. 배경

S2의 1차 소비자인 챗봇은 외부 AI API에 의존하지만, Mimir 자체는 폐쇄망에서도 동작해야 한다.  
기업 내 온프레미스 배포 시나리오, 보안상 외부 API 차단 환경을 모두 지원한다.  
모델 아티팩트·프롬프트 템플릿을 Mimir DB에 저장하고, 외부 MLflow나 SaaS 트레이싱 없이 운영 가능하게 설계한다.

---

## 3. 환경변수 기반 모드 전환

| 환경변수 | 기본값 | 역할 |
|---------|-------|------|
| `EXTERNAL_API_ENABLED` | `false` | 외부 AI API(OpenAI, Anthropic 등) 전체 on/off |
| `OPENAI_API_KEY` | (없음) | 없으면 자동으로 로컬 모델로 전환 |
| `ENABLE_CLOUD_TRACING` | `false` | LangSmith, OTel Cloud 등 외부 트레이싱 on/off |
| `ALLOW_SEED_USERS` | `false` | seed_users.py 실행 허용 (프로덕션 보호) |

---

## 4. 각 영역의 Degrade 정책

| 영역 | 외부 의존 불가 시 | Degrade 동작 | 사용자 알림 |
|------|----------------|-------------|-----------|
| **LLM 생성** | OpenAI API 차단 | vLLM / Ollama / llama.cpp 로컬 모델로 자동 전환 | 없음 (투명 전환) |
| **임베딩** | OpenAI Embedding 차단 | BGE-M3 / E5 로컬 모델 사용 | 없음 (투명 전환) |
| **RAG 검색** | pgvector 장애 | FTS(Full-Text Search)만으로 검색 계속 | "벡터 검색 비활성 — 키워드 검색 모드" 표시 |
| **AI 평가** | 판정 LLM 불가 | Citation-present rate, 원문 매칭(규칙 기반)만 수행 | AI품질평가.md에 "LLM 지표 미측정" 명시 |
| **모니터링** | Cloud trace 차단 | 로컬 로그만 기록 (structlog) | 관리자 대시보드에 "외부 트레이싱 비활성" 표시 |
| **임베딩 재구축** | 인터넷 불가 | 로컬 모델 사용, 오프라인 다운로드 필요 | 관리자에게 모델 경로 설정 안내 |

## 5. 코드 구현 기준

```python
# ❌ 금지: 외부 API 실패 시 전체 기능 중단
def answer(query: str) -> str:
    result = openai.chat(query)  # 실패 시 Exception → 서비스 중단
    return result

# ✅ 올바른 패턴: provider 추상화 + fallback
def answer(query: str) -> str:
    provider = llm_provider_factory.get()  # 환경변수에 따라 OpenAI 또는 로컬 모델
    try:
        result = provider.generate(query)
    except ProviderUnavailableError:
        result = fallback_fts_search(query)  # 기능 degrade, 서비스 유지
    return result
```

## 6. 테스트 매트릭스 (CI에서 의무)

| Matrix | 설명 | 활성 기능 |
|--------|------|---------|
| **Matrix 1 (기본)** | 모든 외부 API 활성 | OpenAI + pgvector + 외부 트레이싱 |
| **Matrix 2 (폐쇄망)** | 모든 외부 API 비활성 | 로컬 모델 + FTS 폴백 + 로컬 로그만 |

**양쪽 매트릭스에서 필수 통과 항목:**
- 기본 문서 CRUD
- 키워드 검색 (FTS)
- 인증/인가 흐름
- Admin UI 접근

**Matrix 2에서만 적용되는 검증:**
- `pgvector_enabled=false` 상태에서 검색 요청 시 FTS로 폴백
- `EXTERNAL_API_ENABLED=false` 상태에서 RAG 요청 시 로컬 모델로 응답

## 8. FAQ — 현장에서의 의문

**Q. 로컬 모델이 외부 모델보다 품질이 낮으면, 폐쇄망 품질 기준은 어떻게 되나?**  
A. AI품질평가.md의 폐쇄망 환경 기준값은 기본값에서 -0.05 완화된다 (Task 0-3 §4-2 참조). "폐쇄망에서도 동작한다"가 목표이지, "동일 품질을 보장한다"가 목표가 아니다.

**Q. 모든 외부 의존을 off하면 MCP도 동작하지 않나?**  
A. MCP 서버 자체(Phase 4)는 내부 Mimir API를 노출하는 것이므로 폐쇄망에서도 동작한다. 단, 챗봇이 외부 LLM을 사용하는 것은 챗봇 프로젝트의 책임 영역이다.

**Q. `EXTERNAL_API_ENABLED=false`인데 관리자가 OpenAI 키를 설정하면 어떻게 되나?**  
A. 환경변수가 우선한다. 키가 있어도 `EXTERNAL_API_ENABLED=false`이면 외부 API 호출이 차단된다.

## 10. S1 원칙 ①④와의 관계

| 측면 | S1 원칙 | S2 원칙 ⑦ |
|------|--------|----------|
| ① API-first | 모든 기능은 API로 먼저 정의 | API는 외부 의존 없이도 동작하도록 설계 |
| ④ 권한은 모든 계층에 적용 | ACL이 모든 경로에 적용 | 폐쇄망 폴백 경로(FTS)에서도 ACL 동일 적용 |

S2 원칙 ⑦은 S1 원칙을 무효화하지 않는다. 폴백 경로에서도 S1 원칙(API-first, 권한 적용)이 유지된다.
