## 1. 개요

S2부터 AI 기능을 포함하는 Feature Group은 `AI품질평가보고서.md`를 의무 산출물로 제출해야 한다.  
이 규약은 평가 항목 7개의 정의, 기준값, 측정 방법, 폐쇄망 적용 방침을 확정한다.

## 2. 평가 항목 7개

### 2-1. Faithfulness (충실성)

| 항목 | 내용 |
|------|------|
| **정의** | 생성된 답변이 제공된 근거 문서와 모순하지 않는가? |
| **측정 방법** | LLM 판정: (질문, 근거 문서, 답변) 3입력 → 0~1 점수 출력 |
| **계산 공식** | 평가 쿼리별 Faithfulness 점수의 평균 |
| **LLM 의존도** | 높음 (판정 LLM 필수) |
| **폐쇄망 지원** | ✅ 로컬 LLM 사용 (vLLM, Ollama) |
| **기준값 (기본)** | ≥ 0.80 |
| **기준값 (폐쇄망)** | ≥ 0.75 |

### 2-2. Answer Relevance (답변 관련성)

| 항목 | 내용 |
|------|------|
| **정의** | 생성된 답변이 사용자 질문에 의미적으로 관련이 있는가? |
| **측정 방법** | embedding 기반 코사인 유사도: cos_sim(embed(질문), embed(답변)) |
| **계산 공식** | 평가 쿼리별 코사인 유사도의 평균 |
| **LLM 의존도** | 없음 (embedding 모델만 필요) |
| **폐쇄망 지원** | ✅ 로컬 embedding (BGE-M3, E5) |
| **기준값 (기본)** | ≥ 0.75 |
| **기준값 (폐쇄망)** | ≥ 0.70 |

### 2-3. Context Precision (컨텍스트 정밀도)

| 항목 | 내용 |
|------|------|
| **정의** | 검색 결과 중 실제로 질문 답변에 기여하는 청크의 비율 |
| **측정 방법** | LLM이 검색 결과 각 청크에 대해 "답변에 필요한가?" 판정 후 집계 |
| **계산 공식** | 필요 청크 수 ÷ 전체 검색 청크 수 |
| **LLM 의존도** | 높음 |
| **폐쇄망 지원** | ✅ 로컬 LLM 판정 |
| **기준값 (기본)** | ≥ 0.75 |
| **기준값 (폐쇄망)** | ≥ 0.70 |

### 2-4. Context Recall (컨텍스트 재현율)

| 항목 | 내용 |
|------|------|
| **정의** | 질문 답변에 필요한 모든 청크를 검색했는가? |
| **측정 방법** | 골든셋 방식: 예상 근거 문서(골든셋) vs 실제 검색 결과 비교 |
| **계산 공식** | 검색된 필요 청크 수 ÷ 골든셋의 필요 청크 수 |
| **LLM 의존도** | 낮음 (골든셋 준비만 필요) |
| **폐쇄망 지원** | ✅ |
| **기준값 (기본)** | ≥ 0.80 |
| **기준값 (폐쇄망)** | ≥ 0.75 |

### 2-5. Citation-present rate (근거 포함 비율)

| 항목 | 내용 |
|------|------|
| **정의** | 응답의 각 문장이 검증 가능한 근거(citation with content_hash)를 가지는 비율 |
| **측정 방법** | 규칙 기반: citation 있는 문장 수 ÷ 전체 답변 문장 수 |
| **계산 공식** | citation 있는 문장 수 ÷ 전체 답변 문장 수 |
| **LLM 의존도** | 없음 (규칙 기반) |
| **폐쇄망 지원** | ✅ 항상 가능 |
| **기준값 (기본)** | ≥ 0.90 |
| **기준값 (폐쇄망)** | ≥ 0.90 (동일) |

### 2-6. 헛소리율 (Hallucination rate)

| 항목 | 내용 |
|------|------|
| **정의** | 근거 없이 생성된 답변의 비율 (Citation-present rate의 보완 지표) |
| **측정 방법** | 규칙 기반: 1 − Citation-present rate |
| **계산 공식** | citation 없는 문장 수 ÷ 전체 답변 문장 수 |
| **LLM 의존도** | 없음 (규칙 기반) |
| **폐쇄망 지원** | ✅ 항상 가능 |
| **기준값 (기본)** | ≤ 0.10 |
| **기준값 (폐쇄망)** | ≤ 0.15 |

### 2-7. 지연·토큰·비용 (Performance Metrics)

| 항목 | 내용 |
|------|------|
| **정의** | 모델별·프롬프트별 실행 성능 및 비용 메타데이터 |
| **측정 항목** | 평균 응답 지연(ms), 입력 토큰 수(평균), 출력 토큰 수(평균), 예상 비용(쿼리당, $) |
| **LLM 의존도** | 없음 (메타데이터 수집) |
| **폐쇄망 지원** | ✅ |
| **기준값** | 없음 — 추이 관찰 (Phase 6 FG6.1에서 최적화 기준 재설정) |

## 3. 기준값 종합표

| 지표 | S2 기본 기준값 | 폐쇄망 환경 | 최초 적용 시점 | 측정 방법 |
|------|-------------|-----------|-------------|---------|
| Faithfulness | ≥ 0.80 | ≥ 0.75 | Phase 2 이후 | LLM 판정 |
| Answer Relevance | ≥ 0.75 | ≥ 0.70 | Phase 1 이후 | embedding 유사도 |
| Context Precision | ≥ 0.75 | ≥ 0.70 | Phase 7 이후 | LLM 판정 + 골든셋 |
| Context Recall | ≥ 0.80 | ≥ 0.75 | Phase 7 이후 | 골든셋 비교 |
| Citation-present rate | ≥ 0.90 | ≥ 0.90 | Phase 2 이후 | 규칙 기반 |
| 헛소리율 | ≤ 0.10 | ≤ 0.15 | Phase 2 이후 | 규칙 기반 |
| 지연·토큰·비용 | 기준값 없음 | 기준값 없음 | 모든 Phase | 메타데이터 |

### 기준값 조정 규칙

1. **초기 Phase 완화**: Phase 1~3에서는 기준값을 -0.05 완화 가능 (AI 기능 초기 도입)
2. **폐쇄망 완화**: 로컬 모델 사용 시 -0.05 완화 (모델 성능 차이 반영)
3. **새 모델/프롬프트 도입 시**: 기준값 재설정 필수 (이전 기준을 그대로 사용 금지)
4. **Phase 9**: 기준값 강화 검토 — S2 완수 기준으로 전체 재측정

## 4. "AI 기능 포함 FG" 판정 기준

다음 중 하나 이상을 포함하면 AI품질평가보고서 작성 **필수**:

- LLM을 사용한 생성 (RAG 응답, 초안 생성, 구조화 추출)
- 벡터 검색 (embedding 기반 유사도 검색)
- 에이전트 판정 (LLM 판정 기반 평가)

### Phase별 AI 기능 포함 FG 목록

| Phase | FG | AI 기능 | 필수 평가 항목 |
|-------|-----|--------|-------------|
| 1 | FG1.1 | LLM Provider | 지연·토큰·비용 (성능 메타) |
| 1 | FG1.2 | Embedding | 지연·토큰·비용 (성능 메타) |
| 2 | FG2.1 | Citation Contract | Citation-present rate |
| 2 | FG2.2 | Retriever/Reranker | Answer Relevance |
| 2 | FG2.3 | 쿼리 재작성 | Citation-present + Faithfulness |
| 3 | FG3.2 | 세션 기반 RAG | 전체 7개 지표 |
| 4 | FG4.2 | Agent Principal | Delegation 감사 메트릭 (actor_type 기록률) |
| 5 | FG5.1 | 에이전트 쓰기 | Faithfulness (제안 품질) |
| 5 | FG5.2 | 프롬프트 주입 방어 | 공격 탐지율 (규칙 기반) |
| 5 | FG5.3 | 에이전트 감사 | 승인률, 반려 패턴 |
| 7 | FG7.1~7.3 | 평가 인프라 | 메타 평가 (평가 도구 정확도) |
| 8 | FG8.1~8.3 | 구조화 추출 | Faithfulness, Context Precision |
| 9 | FG9.2 | 통합 회귀 | 모든 7개 지표 재측정 |

## 5. 평가 수행 불가 시 처리 기준

| 사유 | 처리 |
|------|------|
| 평가 인프라 미구축 (Phase 1~6) | 측정 가능한 지표만 기록. 나머지는 "Phase 7 이후 측정 예정" 명시 |
| 골든셋 미준비 | Context Recall은 골든셋 준비 완료 후 재측정. 다른 지표는 진행 |
| 로컬 LLM 사용 | 폐쇄망 기준값 적용 (표 3 참조) |
| AI 기능 포함이 불분명한 경우 | 포함으로 간주하고 가능한 지표만 측정 |

## 6. 보고서 포맷 요약

AI품질평가보고서 작성 시 반드시 포함해야 하는 섹션:

1. 평가 목표 (이 FG의 AI 기능은 무엇인가)
2. 평가 환경 (일시, 환경, 모델 버전)
3. 평가 데이터셋 (골든셋 규모, 쿼리 분포)
4. 평가 결과 (7개 지표별 실측값)
5. 기준값 대비 (실측값 vs 기준값, 통과 여부)
6. 개선 항목 (미통과 지표의 개선안)
7. 부록: 오류 케이스 (실패 쿼리 3~5개, 원인 분석)
8. 부록: 샘플 쿼리/답변 (선택)

상세 포맷 및 예시 데이터는 `AI품질평가_템플릿.md` 참조.
