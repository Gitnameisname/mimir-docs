`S2_종결보고서.md` 는 "잔존 결함 0건, 테스트 940건 100% 통과, AI 품질 기준 충족 ✅"로 S2 를 종결 선언하였다. 그러나 `S2_전체_재검수보고서_20260418.md` 재검수 결과 다음 두 선언이 **실측 근거로 뒷받침되지 않는 것**으로 확인되었다.

## 3. 보정 항목 C — docker-compose 기본 시크릿 (F-06) — **본 보정과 함께 코드 패치 적용**

### 3.1 패치 내역 (2026-04-18)

다음 기본값을 제거하고 `:?` 구문으로 강제하도록 수정:

| 파일 | 변경 전 | 변경 후 |
|------|---------|--------|
| `docker-compose.yml` x-common-env.POSTGRES_PASSWORD | `${POSTGRES_PASSWORD:-devpassword}` | `${POSTGRES_PASSWORD:?...}` |
| `docker-compose.yml` db.POSTGRES_PASSWORD | `${POSTGRES_PASSWORD:-devpassword}` | `${POSTGRES_PASSWORD:?...}` |
| `docker-compose.yml` backend.JWT_SECRET | `${JWT_SECRET:-dev-jwt-secret-change-in-prod}` | `${JWT_SECRET:?...}` |
| `docker-compose.yml` backend.INTERNAL_SERVICE_SECRET | `${INTERNAL_SERVICE_SECRET:-dev-internal-secret}` | `${INTERNAL_SERVICE_SECRET:?...}` |
| `docker-compose.yml` grafana.GF_SECURITY_ADMIN_PASSWORD | `${GRAFANA_PASSWORD:-admin}` | `${GRAFANA_PASSWORD:?...}` |
| `docker-compose.prod.yml` valkey.--requirepass | `${VALKEY_PASSWORD}` | `${VALKEY_PASSWORD:?...}` |
| `.env.example` GRAFANA_PASSWORD | `admin` (위험 예시) | `change_this_to_strong_password` |

이제 `.env` 가 없거나 위 키가 비어 있으면 `docker compose up` 이 **즉시 실패**한다. 즉 "실수로 dev 기본 시크릿이 프로덕션에 유입" 경로가 차단된다.

### 3.2 개발자 행동

- `.env.example` 을 `.env` 로 복사 후 모든 플레이스홀더를 실제 강한 비밀값으로 치환해야 `compose up` 이 동작한다.
- `monitoring` 프로파일(`--profile monitoring`) 을 사용하지 않는다면 `GRAFANA_PASSWORD` 는 설정할 필요 없다.
