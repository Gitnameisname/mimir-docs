## 5. S1 스프린트 내 릴리스 직전 실사용 검증 이슈 (Phase 14 마감 후 발견·수정)

Phase 14 공식 검수/보안 사인오프 이후, 실제 개발 환경 부팅·UI 사용 과정에서 아래 5건의 경계 조건 이슈가 발견되어 즉시 수정했다. 본 S1 범위 내에서 봉합.

| 이슈 | 원인 | 수정 |
|------|------|------|
| `users` 테이블 부재 상태에서 `seed_users.py` 단독 실행 시 `UndefinedTable` | 시드 스크립트가 `init_db()` 를 호출하지 않음 | `seed_users.main()` 최상단에서 `init_db()` 명시 호출 |
| 로컬 PG에 pgvector 미설치 시 `init_db()` 전체가 `FeatureNotSupported` 로 롤백 → 모든 테이블 생성 중단 | pgvector DDL 이 트랜잭션 스코프 전체를 오염 | pgvector DDL 블록을 SAVEPOINT 로 감싸고 실패 시 `ROLLBACK TO SAVEPOINT` 후 경고 로그로 degrade. 비벡터 스키마는 정상 생성 |
| RAG 채팅이 `openai.PermissionDeniedError: gpt-4o` 로 실패 | 기본 모델이 프로젝트 권한 없는 `gpt-4o` 로 설정 | `llm_model` 기본값 `gpt-4o-mini` 로 변경, `stream_answer` 가 `PermissionDeniedError/NotFoundError/AuthenticationError` 를 사용자 안전 메시지로 변환 |
| 로그인은 이메일만 가능 → 아이디 로그인 · 회원가입 진입 요구 | `LoginRequest.email`만 받음, 로그인 화면에 회원가입 링크 부재 | `LoginRequest.identifier` 도입 + `@model_validator(mode="before")` 로 `email` 별칭 하위호환, `users.username` 컬럼 · 부분 unique index 추가, `get_by_identifier()` 구현, 로그인/회원가입 UI에 아이디 필드와 회원가입 CTA 추가 |
| 문서 생성 직후 `/api/v1/documents/undefined/versions` 500 (UUID 파싱 실패) | `documentsApi.create/update` 가 API envelope `{data, meta}` 를 언래핑하지 않고 반환 → `doc.id`가 `undefined` | `adaptDocument()` 헬퍼로 `r.data ?? raw` 언래핑을 `get/create/update` 전부에 일괄 적용 |

이 5건은 모두 "환경 의존성 또는 경계 조건"에 해당하며, 정적/단위 테스트로는 재현되지 않고 실제 부팅·UI 플로우에서만 드러나는 성격이다. S2에서는 해당 경계를 사전 방어할 수 있는 테스트 게이트를 도입한다(§회고 참조).
