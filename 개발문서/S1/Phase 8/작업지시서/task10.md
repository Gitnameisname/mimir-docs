# Task 8-10. 검색 성능 및 품질 검증

## 1. 작업 목적

구현된 검색 시스템의 성능과 결과 품질을 측정하고 목표 기준을 만족하는지 검증한다.

이 작업의 목표는 다음과 같다.

- 검색 응답 시간 측정 및 목표 기준 만족 여부 확인
- 권한 필터링 포함 쿼리의 성능 분석
- 검색 결과 관련도(정확도) 평가
- 인덱스 갱신 지연 허용 기준 검증
- 대량 데이터 환경 시뮬레이션 테스트

## 4. 주요 검증 대상

### 4-1. 성능 목표 기준

| 측정 항목 | 목표 기준 | 측정 방법 |
|----------|----------|---------|
| 문서 검색 응답 시간 (p50) | < 200ms | API 응답 시간 측정 |
| 문서 검색 응답 시간 (p95) | < 500ms | API 응답 시간 측정 |
| 노드 검색 응답 시간 (p95) | < 800ms | API 응답 시간 측정 |
| 권한 필터링 추가 오버헤드 | < 100ms | 권한 조건 유무 비교 |
| 인덱스 갱신 지연 | < 1초 | 저장 후 검색 가능 시간 |
| 스니펫 생성 추가 오버헤드 | < 50ms | 스니펫 유무 비교 |

---

### 4-2. 쿼리 실행 계획 분석 (EXPLAIN ANALYZE)

분석 대상 쿼리:

1. **기본 문서 검색**: FTS 조건만 적용
2. **권한 필터 포함 검색**: ACL JOIN 추가
3. **DocumentType 필터 포함 검색**: 타입 조건 추가
4. **노드 검색**: Node JOIN + 부모 문서 권한 JOIN
5. **스니펫 포함 검색**: ts_headline 추가

분석 확인 항목:

- 인덱스 사용 여부 (Seq Scan vs. Index Scan)
- 조인 전략 (Hash Join vs. Nested Loop vs. Merge Join)
- 예상 비용 vs. 실제 비용
- GIN 인덱스 활용 여부 확인

---

### 4-3. 대량 데이터 시뮬레이션 테스트

실제 운영 환경을 고려한 데이터 규모에서 성능 측정:

| 시나리오 | 데이터 규모 |
|---------|-----------|
| 소규모 | 문서 1,000건 / 노드 10,000건 |
| 중규모 | 문서 10,000건 / 노드 100,000건 |
| 대규모 | 문서 100,000건 / 노드 1,000,000건 |

각 규모에서 성능 목표 기준 만족 여부 측정.

대규모에서 목표 미달 시 최적화 방안:
- 인덱스 추가 (복합 인덱스, 커버링 인덱스)
- 쿼리 재작성
- 권한 캐싱 전략 도입 여부 결정

---

### 4-4. 검색 결과 관련도(정확도) 평가

검색 결과가 사용자에게 유용한지 평가하는 방법론 정의:

#### 평가 기준

| 기준 | 설명 |
|------|------|
| 정밀도(Precision) | 검색 결과 중 실제로 관련 있는 문서 비율 |
| 재현율(Recall) | 관련 있는 전체 문서 중 검색 결과에 포함된 비율 |
| 관련도 순위 | 가장 관련 있는 문서가 상위에 위치하는지 |

#### 평가 방법

1. 테스트 검색어 집합 정의 (20~30개)
2. 각 검색어별 예상 상위 결과 정의 (정답 셋)
3. 실제 검색 결과와 비교하여 정밀도/재현율 측정
4. 가중치 조정 후 재측정 (A가중치 vs. B가중치 조정)

---

### 4-5. 인덱스 갱신 지연 검증

문서 저장 후 검색 가능 시간을 측정:

1. 새 문서 저장
2. 즉시 검색 쿼리 실행
3. 검색 결과에 포함되는 시간 측정

목표: 문서 저장 후 1초 이내 검색 가능

---

### 4-6. 발견된 문제 최적화 실행

성능 검증 결과 목표 미달 항목에 대한 최적화:

#### 일반적인 최적화 방안

| 문제 | 최적화 방안 |
|------|-----------|
| GIN 인덱스 미사용 | 쿼리 형태 수정, 통계 갱신 |
| ACL JOIN 비용 과다 | ACL 테이블 인덱스 추가 |
| 스니펫 생성 느림 | content_text 길이 제한, 스니펫 길이 축소 |
| 노드 경로 쿼리 느림 | 경로 사전 계산 컬럼 추가 검토 |

---

### 4-7. 검색 성능 기준 문서화

검증 결과를 기반으로 다음을 문서화:

- 현재 성능 기준 (측정값 기반)
- 데이터 규모별 예상 성능 변화
- Phase 10 벡터 검색 도입 시 성능 영향 예측
- 외부 검색 엔진 도입이 필요한 데이터 규모 기준 (임계값 정의)
