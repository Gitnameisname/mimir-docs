# Task 9-7. 변경 비교 UI 설계 및 구현 (User UI 통합)

## 1. 작업 목적

Phase 6 User UI에 변경 비교 뷰어를 통합하여, 사용자가 두 버전 간의 변경 내용을 시각적으로 확인할 수 있도록 한다.

이 작업의 목표는 다음과 같다.

- Unified 뷰와 Side-by-side 뷰 모두 지원하는 diff 뷰어 컴포넌트 설계 및 구현
- 변경 유형별 색상/아이콘 시각 표현 정의
- 버전 선택 UI를 문서 상세 화면에 통합
- 변경된 항목만 보기 토글 구현
- 변경 요약 배너 구현

## 4. 주요 설계 대상

### 4-1. 버전 비교 진입 흐름

#### 문서 상세 화면에서 진입

1. 문서 상세 화면 우측 "버전 이력" 패널
2. 특정 버전 항목에서 "이 버전과 비교" 클릭
3. 비교할 버전 선택 드롭다운 또는 타임라인에서 선택
4. 버전 비교 뷰어 화면으로 이동

#### URL 구조

```
/documents/{doc_id}/versions/{v1_id}/compare/{v2_id}
/documents/{doc_id}/versions/{v1_id}/compare  (직전 버전과 자동 비교)
```

---

### 4-2. 변경 비교 뷰어 페이지 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│ [문서명] 버전 비교                                         │
│ 버전 1.0 (2025-03-01)  ↔  버전 1.1 (2025-03-15)         │
│ [버전 변경] [Unified ▼] [변경만 보기 ○] [인라인 Diff ○]  │
├─────────────────────────────────────────────────────────┤
│ ┌──────────────────────────────────────────────────┐    │
│ │ 변경 요약 배너                                      │    │
│ │ 섹션 2개 추가, 섹션 1개 삭제, 섹션 3개 수정됨        │    │
│ └──────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────┤
│ [변경된 섹션 목록 / 전체 문서 내용]                        │
└─────────────────────────────────────────────────────────┘
```

---

### 4-3. 변경 유형별 시각 표현 정의

| 변경 유형 | 배경색 | 좌측 바 색 | 아이콘 | 설명 |
|----------|--------|----------|--------|------|
| ADDED | 연녹색 (#e6ffed) | 녹색 | + | 추가된 노드 |
| DELETED | 연빨간색 (#ffeef0) | 빨간색 | − | 삭제된 노드 |
| MODIFIED | 연노란색 (#fffbdd) | 노란색 | ~ | 수정된 노드 |
| MOVED | 연파란색 (#e8f4fd) | 파란색 | ↕ | 이동된 노드 |
| UNCHANGED | 기본 배경 | 없음 | 없음 | 변경 없음 |

#### 인라인 Diff 텍스트 표현

| 토큰 타입 | 표시 방식 |
|----------|---------|
| `added` | 녹색 배경 + 밑줄 |
| `deleted` | 빨간색 배경 + 취소선 |
| `unchanged` | 기본 텍스트 |

---

### 4-4. Unified 뷰

변경 사항을 하나의 흐름으로 표시:

```
┌─────────────────────────────────────────────────────┐
│ ~ 2. 정책 본문                               [MODIFIED] │
│   이전: 시스템 접근은 ~~기존 방식으로~~ 처리된다.          │
│   이후: 시스템 접근은 **역할 기반으로** 처리된다.          │
├─────────────────────────────────────────────────────┤
│ + 4. 신규 접근 제어 정책                      [ADDED]   │
│   새로 추가된 섹션 내용...                              │
└─────────────────────────────────────────────────────┘
```

- UNCHANGED 노드는 기본적으로 숨김 (토글로 표시 가능)
- MODIFIED 노드에서 텍스트 인라인 diff 표시 (인라인 diff 토글 ON 시)

---

### 4-5. Side-by-side 뷰

이전/이후 버전을 좌우로 나란히 표시:

```
┌──────────────────────┬──────────────────────┐
│ 버전 1.0 (이전)       │ 버전 1.1 (이후)       │
├──────────────────────┼──────────────────────┤
│ 2. 정책 본문 [수정]   │ 2. 정책 본문 [수정]   │
│ 기존 내용...          │ 변경된 내용...         │
├──────────────────────┼──────────────────────┤
│ (없음)               │ 4. 신규 정책 [추가]    │
│                      │ 새로운 내용...         │
├──────────────────────┼──────────────────────┤
│ 5. 삭제된 섹션 [삭제] │ (없음)                │
└──────────────────────┴──────────────────────┘
```

---

### 4-6. 상단 컨트롤 영역

| 컨트롤 | 설명 |
|--------|------|
| 버전 선택 드롭다운 | 비교 기준 버전 변경 |
| 뷰 모드 선택 | Unified / Side-by-side |
| 변경만 보기 토글 | UNCHANGED 노드 숨기기/표시 |
| 인라인 Diff 토글 | MODIFIED 노드 내 텍스트 diff 표시 여부 |
| 섹션 점프 | 변경된 섹션 목록으로 빠른 이동 |

---

### 4-7. 변경 요약 배너 컴포넌트

페이지 상단에 항상 표시되는 변경 요약:

```
┌────────────────────────────────────────────────────────┐
│ 요약: 섹션 2개 추가, 섹션 1개 삭제, 섹션 3개 수정됨      │
│ 변경된 섹션: [2. 정책 본문] [4. 신규 정책] [외 1개...]   │
│                                          [상세 접기 ∧] │
└────────────────────────────────────────────────────────┘
```

- 변경된 섹션 클릭 시 해당 위치로 스크롤
- 접기/펼치기 가능

---

### 4-8. 상태 처리

| 상태 | 표시 방식 |
|------|---------|
| 로딩 | 노드 Skeleton |
| diff 없음 (두 버전이 동일) | "두 버전 간 변경 사항이 없습니다" |
| diff 너무 큰 경우 | "변경 사항이 매우 많아 요약만 표시됩니다" |
| 권한 없음 | "이 버전에 접근할 권한이 없습니다" |
| 오류 | 재시도 버튼 |

- DiffViewer 컴포넌트가 Unified 뷰로 동작한다

## 7. Codex 작업 지침

- Phase 6 User UI의 공통 컴포넌트 패턴을 재사용한다
- 인라인 diff의 `<mark>` 또는 `<del>` 태그 렌더링 시 XSS 방지를 반드시 적용한다
- 대량 노드 diff 렌더링 시 가상 스크롤(virtual scroll) 적용을 검토한다
- Side-by-side 뷰는 MVP 이후 구현으로 분류하며, Unified 뷰를 우선 구현한다
- UI 리뷰를 최소 5회 이상 수행하여 검토자 관점에서의 UX 품질을 확인한다 (CLAUDE.md 규칙 준수)
