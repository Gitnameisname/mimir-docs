## 1. 편집 화면 설계서 (`/documents/[id]/edit`, `/documents/new`)

### 1-1. 화면 레이아웃

```
┌───────────────────────────────────────────────────────────────┐
│  Edit Header                                                   │
│  [← 취소]  [문서 제목 인라인 편집]    [저장 상태] [저장] [···] │
├───────────────┬───────────────────────────────────────────────┤
│  구조 트리     │  Toolbar                                      │
│  (편집용)     │  [H1][H2][H3] [B][I][U] [목록] [링크] [+블록] │
│               ├───────────────────────────────────────────────┤
│  + 섹션 추가  │  Editor Body                                   │
│               │                                               │
│  ▼ 섹션 1     │  ┌─────────────────────────────────────────┐  │
│    ○ 1.1      │  │  [섹션 1 제목 편집]                      │  │
│    ○ 1.2      │  │  내용 편집 영역...                       │  │
│  ▼ 섹션 2     │  ├─────────────────────────────────────────┤  │
│    ○ 2.1      │  │  [+ 블록 추가]                           │  │
│               │  └─────────────────────────────────────────┘  │
└───────────────┴───────────────────────────────────────────────┘
```

### 1-2. Edit Header 구성

| 요소 | 동작 |
|------|------|
| ← 취소 | Unsaved Changes 있으면 ConfirmDialog → 문서 상세로 이동 |
| 문서 제목 | 인라인 편집 가능한 입력 필드 (클릭 시 편집 활성화) |
| 저장 상태 표시기 | "저장됨" / "저장 중..." / "변경사항 있음" |
| [저장] 버튼 | 명시적 저장 (Primary) — 변경사항 있을 때 활성화 |
| [···] 메뉴 | 메타데이터 편집, 검토 요청 (Draft → In Review) |

### 1-3. 편집 진입 조건

| 진입 경로 | 조건 |
|----------|------|
| 새 문서 생성 | `GET /documents/new` — Author 이상 역할 |
| 기존 문서 편집 | `workflow_status == DRAFT` + Author 역할 |
| Rejected 문서 재편집 | `workflow_status == REJECTED` + 원작성자 |
| 다른 상태 접근 시 | 403/상태 오류 → "현재 상태에서는 편집이 불가합니다" 안내 |

## 2. Node 편집 UX 정의서

### 2-1. Node 타입 및 편집 방식

| Node 타입 | 편집 방식 | 비고 |
|----------|----------|------|
| `section` | 제목 텍스트 편집 | 계층 구조의 컨테이너 |
| `paragraph` | 리치 텍스트 편집 | Toolbar 적용 대상 |
| `heading` | 텍스트 + Level 선택 | H1~H4 |
| `list` | 항목 추가/삭제/들여쓰기 | `ul/ol` 전환 가능 |
| `code_block` | 코드 텍스트 (모노스페이스) | 언어 선택 옵션 |

### 2-2. Node 조작 UX

**Node 추가**:
```
각 Node 하단에 [+ 추가] 버튼 표시 (hover 시)
클릭 → Node 타입 선택 Dropdown:
  [섹션]  [단락]  [제목]  [목록]  [코드 블록]
```

**Node 삭제**:
```
Node 우측 컨텍스트 메뉴 또는 Delete 키
→ ConfirmDialog (내용이 있는 경우만)
```

**Node 순서 변경**:
```
드래그 핸들 (Node 좌측 ⠿ 아이콘) → Drag & Drop
또는: 컨텍스트 메뉴 [위로 이동] [아래로 이동]
```

**Node 들여쓰기 (Depth 변경)**:
```
Tab → 들여쓰기 (자식으로)
Shift+Tab → 내어쓰기 (부모로)
최대 Depth: 5단계
```

### 2-3. 구조 트리 (편집 모드)

- 편집 모드의 구조 트리는 **드래그로 순서 변경 가능** (읽기 모드와 구분)
- 트리에서 삭제 아이콘 노출 (hover 시)
- 하단 [+ 섹션 추가] 버튼으로 최상위 섹션 추가

## 3. 저장/상태 관리 정책 문서

### 3-1. 저장 방식

| 방식 | 트리거 | 동작 |
|------|--------|------|
| 명시적 저장 | [저장] 버튼 클릭 | 즉시 서버 저장 (`PATCH /documents/{id}/draft`) |
| 자동 저장 | 마지막 편집 후 30초 경과 | 백그라운드 저장 (실패 시 Toast 경고) |
| 키보드 저장 | `Cmd/Ctrl + S` | 명시적 저장과 동일 |

### 3-2. 저장 상태 표시

| 저장 상태 | 표시 텍스트 | 스타일 |
|----------|-----------|--------|
| 저장됨 | "저장됨 ✓" | 회색, 작은 텍스트 |
| 변경사항 있음 | "저장되지 않은 변경사항" | 주황색 |
| 저장 중... | "저장 중..." | 회색 + Spinner |
| 저장 실패 | "저장 실패 — 재시도" | 빨간색, 클릭 가능 |

### 3-3. 상태 전이 규칙 (Draft 관리)

```
[새 문서 생성]
  → POST /api/v1/documents → document_id 생성
  → POST /api/v1/documents/{id}/versions → version_id 생성 (DRAFT)
  → /documents/{id}/edit 화면으로 이동

[기존 문서 편집 저장]
  → PATCH /api/v1/documents/{id}/versions/{vid}/draft
  → 동일 버전 갱신 (새 버전 생성 없음)

[검토 요청]
  → POST /api/v1/documents/{id}/versions/{vid}/workflow/submit-review
  → 상태: DRAFT → IN_REVIEW
  → /documents/{id} 읽기 모드로 전환
```

### 3-4. 편집 잠금 및 충돌 처리

| 상황 | 처리 방식 |
|------|----------|
| IN_REVIEW/APPROVED/PUBLISHED 편집 시도 | 편집 페이지 접근 자체 차단 + 안내 메시지 |
| 저장 시 서버 409 (VERSION_NOT_EDITABLE) | Toast 오류 + 최신 상태 재조회 유도 |
| 저장 시 서버 409 (낙관적 락 충돌) | 경고 팝업: "다른 세션에서 수정이 발생했습니다. 새로고침 후 다시 시도하세요." |
| 비정상 종료 후 재진입 | 자동 저장 여부에 따라 "미저장 변경사항이 있습니다" 알림 표시 |

### 3-5. Unsaved Changes 이탈 경고

```
편집 중 이탈 시도 조건:
  - Sidebar 메뉴 클릭
  - 브라우저 뒤로 가기
  - 탭 닫기 / 새로고침

처리:
  - ConfirmDialog:
      제목: "편집 중인 내용이 있습니다"
      내용: "저장하지 않은 변경사항이 사라집니다. 계속하시겠습니까?"
      버튼: [계속 편집] [저장 없이 나가기]
```

## 4. API 연동 포인트

| 기능 | API | Method/Body |
|------|-----|------------|
| 새 문서 생성 | `POST /api/v1/documents` | `{title, document_type, ...}` |
| 버전 생성 | `POST /api/v1/documents/{id}/versions` | `{change_reason?}` |
| Draft 저장 | `PATCH /api/v1/documents/{id}/versions/{vid}/draft` | `{nodes, title, metadata}` |
| 검토 요청 | `POST .../workflow/submit-review` | |
| Node 구조 조회 | `GET /api/v1/documents/{id}/nodes` | 편집 진입 시 로드 |
