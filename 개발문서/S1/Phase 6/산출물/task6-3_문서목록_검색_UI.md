## 1. 문서 목록 화면 명세 (`/documents`)

### 1-1. 화면 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  Page Header                                             │
│  제목: "문서"                          [+ 새 문서 만들기] │
├─────────────────────────────────────────────────────────┤
│  Action Bar                                              │
│  [🔍 문서 검색...]  [상태▼] [유형▼] [작성자▼]  [초기화] │
│                               정렬: [최신 수정순▼]        │
├─────────────────────────────────────────────────────────┤
│  문서 목록                                               │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 제목           유형      상태     작성자   수정일 │    │
│  ├─────────────────────────────────────────────────┤    │
│  │ [문서 행 1]                                      │    │
│  │ [문서 행 2]                                      │    │
│  │ ...                                              │    │
│  └─────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────┤
│  Pagination: [< 이전]  1 2 3 ... 10  [다음 >]  총 N건   │
└─────────────────────────────────────────────────────────┘
```

### 1-2. 화면 상태

| 상태 | 표시 방식 |
|------|----------|
| 초기 로딩 | 행 5개 분량의 SkeletonBlock |
| 데이터 있음 | 문서 행 목록 + 페이지네이션 |
| 데이터 없음 (전체) | EmptyState: "문서가 아직 없습니다" + [새 문서 만들기] 버튼 |
| 데이터 없음 (검색 결과) | EmptyState: "검색 결과가 없습니다. 다른 키워드를 시도해보세요." |
| 오류 | ErrorState + 재시도 버튼 |

## 2. 문서 아이템 UI 정의

### 2-1. 테이블 컬럼 구성

| 컬럼 | 데이터 소스 | 너비 | 정렬 가능 | 비고 |
|------|-----------|------|----------|------|
| 제목 | `document.title` | 가변 (flex) | ✓ | 클릭 시 상세 이동 |
| 유형 | `document.document_type` | 100px | - | `DocumentTypeBadge` |
| 상태 | `version.workflow_status` | 100px | - | `WorkflowStatusBadge` |
| 작성자 | `document.created_by` | 120px | - | 사용자명 |
| 최종 수정일 | `version.updated_at` | 120px | ✓ | 상대 시간 (e.g. "3일 전") |
| 버전 | `version.version_number` | 60px | - | "v1.2" 형식 |

### 2-2. 문서 행 (Row) 상세 구성

```
┌────────────────────────────────────────────────────────────────────┐
│  제목 텍스트 (bold)           [유형 배지]  [상태 배지]              │
│  작성자: 홍길동   수정: 3일 전   v1.2                [···액션 메뉴] │
└────────────────────────────────────────────────────────────────────┘
```

**액션 메뉴 (hover 또는 ··· 클릭 시)**:
- [상세 보기] — 전체 역할
- [편집] — Author + Draft 상태만
- [버전 목록] — 전체 역할

### 2-3. 권한 기반 노출 정책

- 서버가 접근 가능한 문서만 반환 → 목록에서 권한 불필요 문서 자동 제외
- `403` 발생 시 해당 행은 제한 표시 없이 제외 (서버 페이지네이션 기준)
- "새 문서 만들기" 버튼: `Author` 이상 역할일 때만 노출

## 3. 검색/필터/정렬 UI 정의서

### 3-1. 검색 입력

| 항목 | 정의 |
|------|------|
| UI 형태 | `SearchInput` 컴포넌트 (검색 아이콘 + Clear 버튼) |
| 트리거 | Enter 키 또는 0.5초 디바운스 후 자동 실행 |
| 검색 대상 | 제목 기준 (Phase 8에서 전문 검색으로 확장) |
| URL 반영 | `?q=검색어` — 뒤로 가기 / 공유 URL 지원 |
| 최소 길이 | 2자 미만 시 검색 미실행 |

### 3-2. 필터 UI

| 필터 항목 | UI 형태 | 선택 방식 | Query Param |
|----------|---------|----------|------------|
| 문서 상태 | Dropdown | 단일 선택 | `?status=DRAFT` |
| 문서 유형 | Dropdown | 단일 선택 | `?type=...` |
| 작성자 | Dropdown (사용자 목록) | 단일 선택 | `?author=user_id` |

**필터 초기화**: 활성 필터가 있을 때 "초기화" 링크 표시 → 전체 필터 제거

**활성 필터 표시**: Action Bar 하단에 적용된 필터를 태그 형태로 표시
```
[상태: 초안 ×]  [유형: 기술명세 ×]
```

### 3-3. 정렬

| 정렬 기준 | 기본 방향 | Query Param |
|----------|----------|------------|
| 최신 수정순 (기본) | DESC | `?sort=updated_at&order=desc` |
| 생성일순 | DESC | `?sort=created_at&order=desc` |
| 이름순 | ASC | `?sort=title&order=asc` |

### 3-4. 페이지네이션

| 항목 | 정의 |
|------|------|
| 방식 | 번호 기반 페이지네이션 (Infinite Scroll은 Phase 8 검토) |
| 기본 페이지 크기 | 20건 |
| 표시 범위 | 현재 ±2 페이지 + 첫/끝 페이지 |
| URL 반영 | `?page=2` |
| 필터 변경 시 | 자동으로 1페이지로 초기화 |
