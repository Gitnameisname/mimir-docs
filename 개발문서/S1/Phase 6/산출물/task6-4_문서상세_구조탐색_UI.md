## 1. 문서 상세 화면 설계서 (`/documents/[id]`)

### 1-1. 화면 레이아웃

```
┌─────────────────────────────────────────────────────────────────┐
│  Document Header                                                 │
│  [← 문서 목록]  문서 제목 (h1)                  [상태 배지]      │
│  작성자: 홍길동  버전: v1.2  수정: 2026-04-08    [액션 버튼들]  │
├──────────────────┬──────────────────────────────────────────────┤
│  구조 트리        │  문서 본문 (Node 렌더링)                      │
│  (좌측, 220px)   │                                               │
│                  │  [섹션 1 제목]                                │
│  > 섹션 1 ●      │  섹션 1 내용...                               │
│    > 1.1         │                                               │
│    > 1.2         │  [섹션 2 제목]                                │
│  > 섹션 2        │  섹션 2 내용...                               │
│  > 섹션 3        │                                               │
│                  │  [메타데이터 패널 (접기 가능)]                 │
│                  │  문서 유형 / 태그 / 관련 문서 등              │
└──────────────────┴──────────────────────────────────────────────┘
```

### 1-2. Document Header 구성

| 요소 | 위치 | 설명 |
|------|------|------|
| 뒤로 가기 | 좌측 | 문서 목록으로 이동 |
| 문서 제목 | 중앙 좌 | `h1` 수준, 줄 넘침 처리 |
| 상태 배지 | 제목 옆 | `WorkflowStatusBadge` |
| 버전 정보 | 서브텍스트 | "v1.2 · 2026-04-08" 형식 |
| 작성자 | 서브텍스트 | 사용자명 |
| 액션 버튼 그룹 | 우측 | 상태/역할 기반 가변 노출 |

### 1-3. 상태/역할별 액션 버튼 노출 규칙

| 상태 | Viewer | Author (본인) | Reviewer | Approver |
|------|--------|--------------|----------|----------|
| DRAFT | - | [편집] [검토 요청] | - | - |
| IN_REVIEW | - | - | [승인] [반려] | [승인] [반려] |
| APPROVED | - | - | - | [발행] |
| PUBLISHED | - | - | - | [보관] |
| REJECTED | - | [편집] | - | - |

모든 역할 공통: [버전 목록] 버튼 항상 노출

### 1-4. 화면 상태 처리

| 상태 | 처리 방식 |
|------|----------|
| 초기 로딩 | Header SkeletonBlock + 본문 SkeletonBlock (5줄) |
| 로드 성공 | 전체 화면 렌더링 |
| 로드 실패 | ErrorState (전체 화면): "문서를 불러올 수 없습니다" + 재시도 |
| 권한 없음 | PermissionDenied: "이 문서에 대한 접근 권한이 없습니다" |
| 문서 없음 | ErrorState (404): "문서를 찾을 수 없습니다" |

## 2. 트리 탐색 UI 정의서

### 2-1. 구조 트리 패널 구성

```
┌────────────────────────────────┐
│  📄 문서 구조                  │
├────────────────────────────────┤
│  ▶ 1. 개요          ← 접기    │
│    ├ 1.1 배경                  │
│    └ 1.2 목표      ← 현재 위치 ●│
│  ▼ 2. 상세 내용                │
│    ├ 2.1 기능 정의             │
│    └ 2.2 제약 조건             │
│  ▶ 3. 결론                     │
└────────────────────────────────┘
```

### 2-2. 트리 동작 정의

| 동작 | 처리 방식 |
|------|----------|
| 노드 클릭 | 해당 섹션으로 Smooth Scroll + Anchor 이동 |
| 현재 위치 표시 | Scroll 위치 기반으로 활성 노드 자동 하이라이트 |
| 펼치기/접기 | 하위 노드 있는 경우 토글 가능 |
| 기본 상태 | 1단계 노드 펼침, 2단계 이하 접힘 |
| 패널 토글 | Document Header의 [구조 보기] 버튼으로 숨김/표시 |

### 2-3. 트리 데이터 소스

- API: `GET /api/v1/documents/{id}/nodes` (계층 구조 반환)
- 렌더링: Node의 `node_type`, `order`, `parent_id` 기반으로 트리 구성
- 성능: 깊이 5단계 이상은 lazy expand 검토 (Phase 8+)

### 2-4. 섹션 이동 및 앵커 처리

```
URL: /documents/[id]#section-{node_id}
- 트리 항목 클릭 → URL hash 변경 + scroll
- 페이지 진입 시 hash 있으면 자동 이동
- 브라우저 뒤로 가기: hash 히스토리 유지
```

## 3. 메타데이터 및 액션 UI 정의

### 3-1. 메타데이터 패널 표시 항목

문서 본문 하단 또는 우측 패널 (토글 가능):

| 항목 | 표시 값 | 비고 |
|------|--------|------|
| 문서 유형 | `DocumentTypeBadge` | |
| 워크플로 상태 | `WorkflowStatusBadge` | |
| 현재 버전 | "v1.2" | 버전 목록 링크 포함 |
| 생성일 | "2026-03-01" | |
| 최종 수정일 | "2026-04-08 (3일 전)" | |
| 작성자 | "홍길동" | |
| 태그 | 태그 목록 | 있는 경우만 표시 |

### 3-2. 읽기 모드 UX 원칙

1. **본문 우선**: 구조 트리 패널은 기본적으로 열려 있지만, 소형 화면에서는 접힘
2. **스크롤 독립**: 구조 트리는 본문과 독립적으로 스크롤 (sticky 처리)
3. **고밀도 지양**: 본문 영역은 충분한 여백과 줄 간격으로 가독성 확보
4. **노드 타입 렌더링**: Heading → `h2/h3/h4`, Paragraph → `p`, List → `ul/ol` (플러그인 방식)

### 3-3. 특정 버전 열람 (`/documents/[id]/versions/[vid]`)

- 동일 레이아웃 사용
- **읽기 전용** 표시 배너: "이 화면은 v1.1 버전의 내용입니다"
- 액션 버튼: [현재 버전으로 이동], [복원 요청] (Author + 권한 있을 때만)
- 워크플로 액션 버튼 미노출

## 4. API 연동 포인트

| 화면 요소 | API | Method |
|----------|-----|--------|
| 문서 기본 정보 | `GET /api/v1/documents/{id}` | |
| 현재 버전 상세 | `GET /api/v1/documents/{id}/versions/latest` | |
| 노드 트리 | `GET /api/v1/documents/{id}/nodes` | |
| 특정 버전 조회 | `GET /api/v1/documents/{id}/versions/{vid}` | |
| 워크플로 액션 | `POST /api/v1/documents/{id}/versions/{vid}/workflow/{action}` | |
