## 1. RAG 질의 UI 배치안

### 1-1. 배치 위치 결정

**선택: 우측 접이식 패널 (Right Sliding Panel)**

```
┌──────────────────────────────────────────────────────────────────┐
│  Document Header                                          [💬 AI] │
├──────────────┬──────────────────────────────┬────────────────────┤
│  구조 트리   │  문서 본문                    │  RAG 질의 패널     │
│              │                              │  (토글 가능)       │
│              │                              │  ┌──────────────┐  │
│              │                              │  │ 📄 문서 범위  │  │
│              │                              │  ├──────────────┤  │
│              │                              │  │ 질문 입력창   │  │
│              │                              │  ├──────────────┤  │
│              │                              │  │ 답변 영역     │  │
│              │                              │  └──────────────┘  │
└──────────────┴──────────────────────────────┴────────────────────┘
```

**배치 원칙**:
- 기본: 패널 닫힘 (문서 읽기 집중)
- 토글: Document Header 우측 [💬 AI 질의] 버튼
- 패널 너비: 320px (접힘 시 0px, 애니메이션)
- 구조 트리 패널과 동시 표시: 구조 트리 자동 접힘 처리

## 2. 질의 입력/답변 표시 UX 정의서

### 2-1. 질의 범위 정의 및 선택 UI

```
┌──────────────────────────────────────┐
│  질의 범위: [이 문서 ▼]               │
│    ○ 이 문서 (기본)                   │
│    ○ 현재 섹션                        │
│    ○ 전체 문서 집합                   │
└──────────────────────────────────────┘
```

| 범위 | 설명 | API 파라미터 |
|------|------|------------|
| 이 문서 | 현재 열람 중인 문서 내에서만 검색 | `scope: {document_id}` |
| 현재 섹션 | 현재 스크롤 위치 섹션 기준 | `scope: {node_id}` |
| 전체 문서 집합 | 전체 접근 가능 문서 대상 | `scope: null` |

### 2-2. 질의 입력 UI

```
┌──────────────────────────────────────────┐
│  질의 범위: [이 문서 ▼]                   │
├──────────────────────────────────────────┤
│  ┌────────────────────────────────────┐  │
│  │ 이 문서에 대해 질문하세요...         │  │
│  │                                    │  │
│  │                              [전송] │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

| 요소 | 정의 |
|------|------|
| 입력창 | 멀티라인 지원 (최소 2행, 최대 6행) |
| 전송 | Enter (단축키: Shift+Enter = 줄바꿈) |
| 최소 입력 | 5자 미만 시 전송 버튼 비활성화 |
| 질의 중 상태 | 입력창 비활성화 + Spinner |

### 2-3. 답변 표시 구조

```
┌──────────────────────────────────────────┐
│  Q: [사용자 질문 텍스트]                  │
├──────────────────────────────────────────┤
│  [생성 중... ▓▓▓▓                       ]│  ← Streaming 중
│                                          │
│  또는:                                   │
│                                          │
│  A: 답변 본문 텍스트...                  │
│     (길면 [더 보기] 토글 가능)            │
├──────────────────────────────────────────┤
│  근거 문서                               │
│  ┌─────────────────────────────────┐    │
│  │ [1] 섹션 제목 · 문서 제목        │    │
│  │ [2] 섹션 제목 · 다른 문서 제목   │    │
│  └─────────────────────────────────┘    │
└──────────────────────────────────────────┘
```

### 2-4. 상태별 UX

| 상태 | 표시 방식 |
|------|----------|
| 질의 전 (초기) | 예시 질문 3개 표시: "이 문서의 핵심 목적은?", "주요 제약 사항은?" |
| 질의 처리 중 | 입력창 비활성화 + 답변 영역에 Skeleton/Loading 텍스트 애니메이션 |
| 답변 성공 | 답변 본문 + 근거 목록 렌더링 |
| 근거 없음 | "충분한 근거를 찾지 못했습니다. 다른 방식으로 질문해 보세요." |
| 오류 | "답변 생성에 실패했습니다. [다시 시도]" |

## 3. Citation 이동 UX 정의서

### 3-1. Citation 클릭 동작

```
답변 내 [1] 근거 클릭:
  ├─ 같은 문서 내 섹션:
  │   → 해당 섹션으로 Smooth Scroll
  │   → 섹션 하이라이트 (2초 후 사라짐)
  │   → 구조 트리 활성 노드 동기화
  │
  └─ 다른 문서의 섹션:
      → ConfirmDialog: "다른 문서로 이동하시겠습니까? (현재 페이지를 벗어납니다)"
      → 확인 시: 새 탭으로 /documents/{target_id}#section-{node_id} 열기
      → (또는 현재 탭에서 이동 — 정책 결정 필요)
```

### 3-2. Citation 표시 형식

```
근거 목록:
  [1] 3.2 기능 제약 사항  ·  현재 문서  →  클릭 시 스크롤
  [2] 배경 및 목적  ·  기획서 v2.0  →  클릭 시 다른 문서
```

| 항목 | 표시 내용 |
|------|---------|
| 번호 | [1], [2], [3]... (답변 본문에서 인용 위치와 매핑) |
| 섹션 제목 | citation이 가리키는 노드의 제목 |
| 문서 이름 | 현재 문서면 "이 문서", 아니면 문서 제목 표시 |
| 클릭 시 | 위 3-1 동작 |

## 4. 후속 API 연계 메모

### 4-1. 필요한 API (Phase 11 구현 예정)

```
POST /api/v1/rag/query
  Request:
    {
      "question": "질문 텍스트",
      "scope": {
        "type": "document" | "section" | "global",
        "document_id": "...",  // type이 document일 때
        "node_id": "..."       // type이 section일 때
      },
      "session_id": "..."  // 질의 히스토리 유지용 (선택)
    }

  Response:
    {
      "answer": "답변 텍스트",
      "citations": [
        {
          "index": 1,
          "document_id": "...",
          "node_id": "...",
          "section_title": "...",
          "document_title": "..."
        }
      ],
      "confidence": "high" | "medium" | "low"
    }
```

### 4-2. 현재 단계 UI 연계 포인트

| 포인트 | 현재 Phase 6 준비 | Phase 11 연계 |
|--------|-----------------|--------------|
| 패널 컴포넌트 | `RagPanel` 컴포넌트 stub 생성 (UI 골격만) | 실제 API 연결 |
| 상태 모델 | `ragQuery: null` 초기 상태 정의 | 쿼리/응답 상태 관리 |
| Citation 이동 | scroll 함수 인터페이스 정의 | node_id 기반 실제 이동 |
| 패널 토글 | AppLayout에 `ragPanelOpen` 상태 포함 | 동일 유지 |

### 4-3. 편집/승인 화면에서의 RAG 패널 정책

| 화면 | RAG 패널 |
|------|---------|
| 문서 상세 보기 | 허용 (핵심 사용 화면) |
| 문서 편집 화면 | 비허용 (편집 집중 원칙) |
| 버전 상세 보기 | 허용 (특정 버전 기준 질의 가능) |
| 검토/승인 화면 | 허용 (검토 보조 도구로 유용) |
| 문서 목록 | 비허용 (문서 미선택 상태) |
