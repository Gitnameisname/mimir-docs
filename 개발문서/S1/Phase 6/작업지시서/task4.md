

# Task 6-4. 문서 상세 보기 및 구조 탐색 UI 설계

## 1. 작업 목적

사용자가 문서를 **구조적으로 이해하고 탐색할 수 있는 핵심 UI(문서 상세 화면)**를 설계한다.

이 작업의 핵심 목표는 다음과 같다.

- 문서 내용을 명확하게 읽을 수 있는 가독성 높은 UI 제공
- 트리 구조 기반으로 문서를 탐색할 수 있도록 구성
- 문서 상태, 메타데이터, 액션을 한 화면에서 이해 가능하게 구성

---

## 2. 작업 범위

### 포함 범위

- 문서 상세 화면 레이아웃 설계
- 문서 본문 렌더링 구조 정의
- 구조 트리 탐색 UI 설계
- 문서 메타데이터 표시 방식 정의
- 문서 액션 UI 설계

### 제외 범위

- 실제 렌더링 엔진 구현
- 문서 diff 기능 (Phase 9)

## 4. 주요 설계 대상

### 4-1. 문서 상세 화면 레이아웃

#### 기본 구조

- Header (문서 제목)
- Meta 영역 (상태, 작성자 등)
- Main Content (본문)
- 좌측 또는 우측 Tree Panel
- Action 영역

#### 요구사항

- 본문 중심 구조
- 탐색 패널과 분리된 구조
- 확장 가능한 레이아웃

---

### 4-2. 문서 본문 렌더링

문서는 Node 기반 트리 구조로 렌더링된다.

#### 요구사항

- 계층 구조 표현 (Heading, Section 등)
- 들여쓰기 기반 구조 표현
- 긴 문서에서도 성능 유지

---

### 4-3. 구조 탐색 트리 UI

문서 구조를 탐색하기 위한 트리 UI를 설계한다.

#### 기능

- 섹션 목록 표시
- 클릭 시 해당 위치로 이동 (Anchor 이동)
- 현재 위치 하이라이트

---

### 4-4. 문서 메타데이터 표시

문서의 상태 및 주요 정보를 표시한다.

#### 표시 항목

- 문서 상태 (Draft / Review / Approved / Published)
- 작성자
- 생성일 / 수정일
- 버전 정보

---

### 4-5. 문서 액션 UI

문서 상태와 권한에 따라 액션을 제공한다.

#### 주요 액션

- Edit
- Request Review
- Approve / Reject
- View Versions

---

### 4-6. 상태별 UI 처리

#### Loading

- Skeleton 화면

#### Error

- 로드 실패 메시지

#### Empty

- 내용 없음 안내
