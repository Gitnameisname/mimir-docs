# Task 14-10. 사용자/조직/역할 관리 UI 구현

## 1. 작업 목적

관리자가 Admin UI에서 사용자, 조직, 역할을 CRUD하고 권한 매트릭스를 조회할 수 있는 관리 페이지를 구현한다.

이 작업의 목표는 다음과 같다.

- 사용자 목록/상세/생성/수정 UI
- 조직 목록/상세/생성/수정 UI
- 역할 목록/상세/생성/수정 UI
- 역할-권한 매트릭스 뷰
- 사용자 검색/필터링

---

## 2. 작업 범위

### 포함 범위

- `/admin/users` — 사용자 관리 페이지
- `/admin/organizations` — 조직 관리 페이지
- `/admin/roles` — 역할/권한 관리 페이지
- 기존 Admin API 활용 (`/admin/users/*`, `/admin/organizations/*`, `/admin/roles/*`)
- 테이블, 모달, 폼 컴포넌트
- 페이지네이션, 검색, 필터링

### 제외 범위

- Admin API 변경 (기존 API 그대로 사용)
- 사용자 일괄 가져오기/내보내기

---

## 3. 선행 조건

- Task 14-9 Admin 레이아웃 완료
- 기존 Admin API 엔드포인트 동작 확인
- Phase 7 Admin UI 컴포넌트 패턴 이해

## 4. 주요 구현 대상

### 4-1. 사용자 관리 페이지 (`/admin/users`)

```
사용자 관리
[검색: 이름/이메일]  [필터: 역할 ▼]  [필터: 상태 ▼]  [+ 사용자 추가]

┌──────┬────────────┬──────────┬────────┬──────────┬────────┐
│ 이름  │ 이메일      │ 역할      │ 상태   │ 마지막 로그인 │ 작업   │
├──────┼────────────┼──────────┼────────┼──────────┼────────┤
│ 홍길동│ hong@...   │ AUTHOR   │ 활성   │ 2시간 전   │ [편집] │
│ 김철수│ kim@...    │ REVIEWER │ 활성   │ 1일 전    │ [편집] │
│ 이영희│ lee@...    │ VIEWER   │ 정지   │ 7일 전    │ [편집] │
└──────┴────────────┴──────────┴────────┴──────────┴────────┘
                         [< 1 2 3 ... 10 >]

사용자 편집 모달:
  - 이름 변경
  - 역할 변경 (드롭다운)
  - 상태 변경 (ACTIVE / INACTIVE / SUSPENDED)
  - 조직 역할 할당/제거
  - 인증 방식 표시 (local / gitlab)
```

### 4-2. 조직 관리 페이지 (`/admin/organizations`)

```
조직 관리
[검색: 조직명]  [+ 조직 추가]

┌──────────┬────────┬──────────┬────────┐
│ 조직명    │ 상태   │ 구성원 수  │ 작업   │
├──────────┼────────┼──────────┼────────┤
│ 개발팀    │ 활성   │ 12       │ [편집] │
│ 운영팀    │ 활성   │ 8        │ [편집] │
└──────────┴────────┴──────────┴────────┘

조직 상세:
  - 조직 정보 편집
  - 구성원 목록 (사용자 + 조직 내 역할)
  - 구성원 추가/제거
```

### 4-3. 역할/권한 관리 페이지 (`/admin/roles`)

```
역할 관리
[+ 커스텀 역할 추가]

┌──────────┬──────────────────────┬──────┬────────┐
│ 역할명    │ 설명                  │ 유형  │ 작업   │
├──────────┼──────────────────────┼──────┼────────┤
│ SUPER_ADMIN│ 전체 플랫폼 관리      │ 시스템│        │
│ ORG_ADMIN │ 조직/사용자 관리       │ 시스템│        │
│ APPROVER  │ 문서 승인/게시        │ 시스템│        │
│ REVIEWER  │ 문서 검토/코멘트      │ 시스템│        │
│ AUTHOR    │ 문서 생성/수정        │ 시스템│        │
│ VIEWER    │ 문서 읽기 전용        │ 시스템│        │
│ AUDITOR   │ 감사 전용             │ 커스텀│ [편집] │
└──────────┴──────────────────────┴──────┴────────┘
```

시스템 역할은 편집/삭제 불가, 커스텀 역할만 편집 가능.

---

## 6. 완료 기준

- 사용자 목록에서 검색/필터/페이지네이션이 동작한다
- 사용자의 역할, 상태를 변경할 수 있다
- 조직 CRUD 및 구성원 관리가 동작한다
- 역할 목록과 권한 매트릭스가 표시된다
- 시스템 역할은 편집/삭제가 불가능하다
- 데스크탑/모바일 반응형이 동작한다
- UI 디자인 리뷰를 최소 5회 수행한다

---

## 7. Codex 작업 지침

- DataTable 컴포넌트를 범용으로 만들어 다른 관리 페이지에서도 재사용한다
- 테이블 정렬은 서버 사이드로 처리한다 (API 쿼리 파라미터)
- 삭제 동작은 확인 다이얼로그를 표시한다
- SUPER_ADMIN만 사용자 삭제가 가능하도록 UI에서도 제한한다
- 모든 관리 작업에 감사 이벤트가 기록되는지 확인한다
