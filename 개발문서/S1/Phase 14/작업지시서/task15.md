# Task 14-15. 감사 로그 및 API 키 관리 UI

## 1. 작업 목적

감사 로그를 검색·필터링·조회하고, API 키를 CRUD할 수 있는 관리 UI를 구현한다.

이 작업의 목표는 다음과 같다.

- 감사 로그 검색/필터링 UI (기간, 이벤트 유형, 사용자, 결과)
- 감사 로그 상세 조회 UI
- API 키 목록/생성/폐기 UI
- API 키 사용량 조회

---

## 2. 작업 범위

### 포함 범위

- `/admin/audit-logs` UI 페이지
- `/admin/api-keys` UI 페이지
- 기존 Admin API 활용 (`/admin/audit-logs/*`, `/admin/api-keys/*`)
- 감사 로그 필터: 기간, 이벤트 유형, 사용자, 결과(success/failure)
- API 키 생성 시 키 표시 (1회만)
- API 키 폐기

### 제외 범위

- 감사 로그 내보내기 (CSV/Excel)
- API 키 사용 제한 설정 (Rate Limit per key)

---

## 3. 선행 조건

- Task 14-9 Admin 레이아웃 완료
- 기존 `audit_events`, `api_keys` 테이블 이해
- 기존 Admin API 엔드포인트 이해

## 6. 완료 기준

- 감사 로그가 기간/유형/사용자/결과로 필터링된다
- 감사 로그 상세에서 이벤트 전체 정보가 표시된다
- API 키 생성 시 키가 한 번만 표시되고 복사 가능하다
- API 키 폐기 후 즉시 사용이 불가능하다
- 페이지네이션이 동작한다
- UI 디자인 리뷰를 최소 5회 수행한다

---

## 7. Codex 작업 지침

- 감사 로그 목록은 서버 사이드 필터링/페이지네이션으로 처리한다 (대량 데이터)
- API 키는 생성 직후 1회만 표시 — 모달 닫기 후 재확인 불가
- 날짜 범위 필터는 사용자 시간대 기준으로 변환하여 API에 전달한다
- API 키 폐기는 확인 다이얼로그에서 키 이름을 명시하여 오인을 방지한다
- 감사 로그에 민감 정보(비밀번호 등)가 포함되지 않는지 확인한다
