# Task 14-9. 통합 관리 대시보드 UI 구현

## 1. 작업 목적

관리자가 Mimir 플랫폼 전체 상태를 한눈에 파악할 수 있는 통합 관리 대시보드를 구현한다.

이 작업의 목표는 다음과 같다.

- Admin 전용 레이아웃 (사이드바 네비게이션 + 헤더)
- 홈 대시보드 (핵심 메트릭 카드, 시스템 상태, 최근 활동)
- 기존 Phase 7 Admin UI 확장
- 관리 메뉴 구조 설계 및 구현

---

## 2. 작업 범위

### 포함 범위

- Admin 레이아웃 컴포넌트 (사이드바, 헤더, 브레드크럼)
- 홈 대시보드 페이지 (`/admin`)
- 메트릭 카드 컴포넌트 (총 사용자, 총 문서, 활성 세션, 에러 수)
- 시스템 상태 표시 (DB, Valkey, 벡터 DB, Job Runner)
- 최근 감사 로그 위젯
- 최근 에러 위젯
- Admin 사이드바 네비게이션 구조

### 제외 범위

- 개별 관리 페이지 (사용자, 조직, 역할 등 — Task 14-10~15에서 구현)
- 상세 모니터링 차트 (Task 14-12에서 구현)

---

## 3. 선행 조건

- Task 14-8 프론트엔드 인증 상태 관리 완료 (AuthGuard)
- Phase 7 Admin UI 컴포넌트 이해
- 기존 Admin API 이해 (`/admin/dashboard/*`)

---

## 4. 주요 구현 대상

### 4-1. Admin 사이드바 네비게이션 구조

```
/admin
  ├── 대시보드         /admin                  (홈)
  ├── 사용자 관리      /admin/users
  ├── 조직 관리        /admin/organizations
  ├── 역할/권한 관리   /admin/roles
  ├── 시스템 설정      /admin/settings
  ├── 모니터링         /admin/monitoring
  ├── 알림 관리        /admin/alerts
  ├── 배치 작업        /admin/jobs
  ├── 감사 로그        /admin/audit-logs
  └── API 키 관리      /admin/api-keys
```

사이드바는 접기/펼치기 가능, 현재 메뉴 하이라이트.

### 4-3. 홈 대시보드 메트릭 카드

기존 `GET /api/v1/admin/dashboard/metrics` API 활용:

```
┌──────────┬──────────┬──────────┬──────────┐
│ 총 사용자 │ 총 문서   │ 활성 세션 │ 오늘 에러 │
│   142    │  1,847   │    23    │    3     │
│ ↑ 12 이번주│ ↑ 56 이번주│          │ ↓ 감소    │
└──────────┴──────────┴──────────┴──────────┘
```

각 카드는 아이콘, 수치, 변화 추이를 표시.

### 4-4. 시스템 상태 위젯

기존 `GET /api/v1/admin/dashboard/health` API 활용:

```
시스템 상태
  ● PostgreSQL     정상    응답 12ms
  ● Valkey         정상    응답 2ms
  ● 벡터 DB        정상    응답 45ms
```

상태 표시: 정상(초록), 경고(주황), 오류(빨강).

## 6. 완료 기준

- ORG_ADMIN 이상만 Admin 영역에 접근할 수 있다
- 대시보드에서 핵심 메트릭 4종이 카드로 표시된다
- 시스템 구성 요소별 상태가 실시간으로 표시된다
- 최근 활동과 에러가 시간순으로 표시된다
- 사이드바 네비게이션이 모든 관리 메뉴를 포함한다
- 데스크탑/모바일 반응형 레이아웃이 동작한다
- UI 디자인 리뷰를 최소 5회 수행한다

---

## 7. Codex 작업 지침

- 기존 Phase 7 Admin UI 컴포넌트를 최대한 재사용한다
- 대시보드 데이터는 30초 간격 자동 갱신 (polling 또는 SWR)
- API 호출 실패 시 에러 상태를 graceful하게 표시한다 (빈 카드 + 에러 메시지)
- 사이드바 메뉴 구조는 향후 확장을 고려하여 config 기반으로 구성한다
- 다크 모드 지원은 이 단계에서 고려하지 않는다 (향후 확장)
