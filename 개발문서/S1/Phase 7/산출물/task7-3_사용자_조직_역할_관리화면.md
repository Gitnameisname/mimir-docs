## 1. 사용자 관리 화면 설계

### 1-1. 사용자 목록 화면 (`/admin/users`)

#### 화면 레이아웃

```
┌─────────────────────────────────────────────────────────────────┐
│  PageHeader: 사용자 관리                                         │
├─────────────────────────────────────────────────────────────────┤
│  [🔍 이름 또는 이메일 검색]  [상태 ▾]  [조직 ▾]  [역할 ▾]      │
│  활성 필터: [상태: ACTIVE ×]  [전체 초기화]                      │
├──────┬──────────┬──────────────┬──────────┬────────┬────────────┤
│      │ 이름     │ 이메일       │ 조직     │ 역할   │ 상태       │
├──────┼──────────┼──────────────┼──────────┼────────┼────────────┤
│  ☐  │ 홍길동   │ hong@...     │ 개발팀   │ AUTHOR │ 🟢 ACTIVE  │
│  ☐  │ 김철수   │ kim@...      │ 운영팀   │ VIEWER │ 🔴 INACTIVE│
├──────┴──────────┴──────────────┴──────────┴────────┴────────────┤
│  총 350명   ←  1  2  ...  18  →   [20개씩 ▾]                    │
└─────────────────────────────────────────────────────────────────┘
```

#### 목록 테이블 컬럼 정의

| 컬럼 | 데이터 | 정렬 | 비고 |
|------|--------|------|------|
| (체크) | - | - | 다중 선택용 |
| 이름 | `display_name` | ✅ | |
| 이메일 | `email` | ✅ | |
| 소속 조직 | `organizations[0].name` | - | 다중 조직 시 "+N" 표시 |
| 역할 | `role.name` | - | |
| 상태 | `status` | - | StatusBadge |
| 가입일 | `created_at` | ✅ | |
| 최근 로그인 | `last_login_at` | ✅ | 없으면 "없음" |

#### 검색/필터

| 필터 | 옵션 |
|------|------|
| 상태 | ALL / ACTIVE / INACTIVE / SUSPENDED |
| 조직 | 조직 목록 동적 조회 |
| 역할 | SUPER_ADMIN / ORG_ADMIN / AUTHOR / REVIEWER / APPROVER / VIEWER |

### 1-3. 사용자 상태 변경 UX 흐름

1. 상태 변경 버튼(드롭다운) 클릭 → ACTIVE / INACTIVE / SUSPENDED 선택
2. 변경 유형에 따라 다이얼로그 선택:
   - INACTIVE, SUSPENDED → `ConfirmDialog` (사유 입력 선택)
3. 확인 후 → API 호출 → 성공 시 Toast "상태가 변경되었습니다" + 화면 갱신
4. **감사 이벤트 자동 기록**: `USER_STATUS_CHANGED` (감사 로그에 before/after 포함)

| 컬럼 | 데이터 | 설명 |
|------|--------|------|

## 4. 매핑 편집 흐름 정의서

### 4-1. 사용자-조직-역할 매핑 편집 모달

```
┌─────────────────────────────────────────────────────┐
│  홍길동 조직/역할 편집                               │
│                                                     │
│  조직 추가                                          │
│  [조직 선택 ▾ (검색 가능)]  [역할 선택 ▾]  [추가]  │
│                                                     │
│  현재 매핑                                          │
│  조직        │ 역할     │                           │
│  ────────────┼──────────┼───────────────────        │
│  개발팀      │ AUTHOR   │ [제거]                    │
│  운영팀      │ REVIEWER │ [제거]                    │
│                                                     │
│  변경 내역 미리보기                                 │
│  + 추가: 기획팀 / EDITOR                            │
│  - 제거: 운영팀 / REVIEWER                          │
│                                                     │
│                    [취소]  [변경 저장]              │
└─────────────────────────────────────────────────────┘
```

### 4-2. 흐름 단계

1. 사용자 상세 또는 조직 상세에서 "매핑 편집" 버튼 클릭
2. 매핑 편집 모달 표시 (현재 매핑 초기값)
3. 조직/역할 추가 또는 기존 매핑 제거
4. 변경 내역 미리보기 표시
5. "변경 저장" 클릭 → ConfirmDialog ("변경 사항을 저장하시겠습니까?")
6. 확인 → API 호출 → Toast + 모달 닫힘 + 화면 갱신
7. **감사 이벤트 자동 기록**: `USER_ROLE_CHANGED`
