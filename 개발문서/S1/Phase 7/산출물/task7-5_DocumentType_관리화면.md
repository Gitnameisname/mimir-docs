## 1. DocumentType 목록 화면 (`/admin/document-types`)

### 1-1. 화면 레이아웃

```
Admin > Document Types

⚠️  이 화면은 SUPER_ADMIN 전용입니다.

┌────────────────────────────────────────────────────────────────┐
│  [🔍 타입 코드, 표시명 검색]  [상태 ▾]           [+ 새 타입 (비활성)] │
├───────────┬────────────┬──────────┬────────────┬──────┬────────┤
│ 타입 코드  │ 표시명    │ 상태     │ 메타필드 수  │ 문서 수│ 생성일 │
├───────────┼────────────┼──────────┼────────────┼──────┼────────┤
│ POLICY    │ 정책 문서  │ 🟢 ACTIVE│ 8          │ 124  │ 2025-01│
│ MANUAL    │ 매뉴얼     │ 🟢 ACTIVE│ 6          │ 87   │ 2025-01│
│ REPORT    │ 보고서     │ ⚫ INACTIVE│ 5         │ 43   │ 2025-02│
└───────────┴────────────┴──────────┴────────────┴──────┴────────┘
```

### 1-2. 목록 컬럼 정의

| 컬럼 | 데이터 | 비고 |
|------|--------|------|
| 타입 코드 | `type_code` | 예: POLICY, MANUAL, REPORT |
| 표시명 | `display_name` | |
| 설명 | `description` | 툴팁으로 표시 (공간 절약) |
| 상태 | `status` | StatusBadge |
| 메타데이터 필드 수 | `schema_field_count` | |
| 문서 수 | `document_count` | 클릭 시 해당 타입 문서 목록 이동 |
| 생성일 | `created_at` | |

**주의**: 타입 목록은 DB에서 동적 조회 — 하드코딩 금지

## 3. 타입 상태 변경 흐름 문서

### 3-1. 비활성화 흐름

```
비활성화 버튼 클릭
→ DangerDialog 표시:
  ─────────────────────────────────────────────
  ⚠️  DocumentType 비활성화: POLICY

  이 타입을 사용하는 문서: 124건

  비활성화하면:
  • 이 타입으로 새 문서를 생성할 수 없습니다
  • 기존 문서는 그대로 유지됩니다
  • 인덱싱 대상에서 제외될 수 있습니다

  변경 사유 (선택):
  [___________________________________]

             [취소]  [비활성화]
  ─────────────────────────────────────────────
→ 확인 → API 호출 → Toast "POLICY 타입이 비활성화되었습니다"
→ 감사 이벤트 자동 기록: DOCUMENT_TYPE_DEACTIVATED
```

### 3-2. 활성화 흐름

```
활성화 버튼 클릭
→ ConfirmDialog: "POLICY 타입을 다시 활성화하시겠습니까?"
→ 확인 → API 호출 → Toast + 화면 갱신
→ 감사 이벤트: DOCUMENT_TYPE_ACTIVATED
```

## 4. 메타데이터 스키마 표시 구조 정의서

### 4-1. 사람이 읽기 쉬운 표 형식

```
필드명         타입      필수    기본값    설명
──────────────────────────────────────────────────────
title          string    Y       -         문서 제목
author         string    Y       -         작성자
version        string    N       "1.0"     버전 표기
```

- 타입은 `string / number / boolean / date / array / object / enum` 중 하나로 표시
- 필수 여부는 ✅ / - 로 표시
- 기본값이 없으면 `-` 표시

### 4-2. JSON Schema 원본

- Expand/Collapse 가능한 코드 블록으로 제공
- 구문 강조(Syntax Highlight) 적용
- "복사" 버튼 제공

## 5. 플러그인 설정 확장 포인트 정의서

| 설정 카테고리 | 현재 표시 | MVP 이후 |
|--------------|-----------|---------|
| Editor | 현재 설정값 조회 전용 | 편집 가능 |
| Renderer | 현재 설정값 조회 전용 | 편집 가능 |
| Default Workflow | 현재 연결 워크플로 표시 | 변경 가능 |
| Chunking 전략 | 현재 전략명 표시 | 전략 선택 가능 |

### 신규 타입 추가 (Reserved)

- 목록 화면에 "새 타입 추가" 버튼 UI 위치 예약 → 현재 `disabled` 상태로 표시
- 추가 시 필요한 필드: 타입 코드(고유), 표시명, 설명, 메타데이터 스키마 JSON
