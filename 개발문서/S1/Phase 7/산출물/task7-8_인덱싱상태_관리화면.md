## 2. 상태별 표시 방식 정의서

| 상태 | 배지 | 설명 |
|------|------|------|
| PENDING | 회색 | 처리 대기 중 |
| RUNNING | 파란색 (펄스) | 현재 처리 중 |
| COMPLETED | 녹색 | 정상 완료 |
| FAILED | 빨간색 | 실패 (사유 표시) |
| SKIPPED | 노란색 | 건너뜀 (권한/조건 미충족) |

## 4. 실패 분류 및 재처리 UX 흐름 문서

### 4-1. 실패 분류

| 오류 유형 | 설명 | 권장 조치 |
|-----------|------|-----------|
| `DOCUMENT_NOT_FOUND` | 문서 삭제됨 | 인덱싱 항목 제거 (재시도 불필요) |
| `SCHEMA_PARSE_ERROR` | 문서 구조 파싱 실패 | 문서 메타데이터 확인 후 재시도 |
| `VECTOR_STORE_ERROR` | 벡터 저장소 오류 | 시스템 상태 확인 후 재시도 |
| `PERMISSION_ERROR` | 권한 정보 처리 실패 | 권한 설정 확인 |
| `TIMEOUT` | 처리 시간 초과 | 문서 크기 확인 후 재시도 |

### 4-2. 단건 재시도 흐름

1. FAILED 작업 선택 → "재시도" 버튼 클릭
2. ConfirmDialog (선택): "재시도하시겠습니까?"
3. 확인 → 상태 PENDING 전환 → Toast "재시도 대기열에 추가되었습니다"
4. 목록 자동 갱신

### 4-3. 일괄 재시도 흐름

1. "전체 실패 재시도" 버튼 클릭 (SUPER_ADMIN 전용)
2. DangerDialog:
   ```
   일괄 재시도: FAILED 상태 31건
   
   최대 100건까지 일괄 재시도 가능합니다.
   처리 부하가 증가할 수 있습니다.
   
          [취소]  [일괄 재시도]
   ```
3. 확인 → API 호출 → Toast
