# Task 7-2. Admin 공통 레이아웃 및 UI 패턴 설계

## 1. 작업 목적

Admin UI 전반에서 일관된 운영자 경험을 제공하기 위해 **공통 레이아웃 구조와 UI 패턴을 정의**한다.

이 작업의 핵심 목표는 다음과 같다.

- 모든 Admin 화면에서 재사용 가능한 레이아웃 구조 확립
- 운영 화면에 적합한 테이블/필터/상세 패널/폼 패턴 정의
- 이후 개별 관리 화면(Task 7-3 ~ 7-10) 설계의 기반 마련

---

## 2. 작업 범위

### 포함 범위

- Admin 공통 레이아웃 구조 정의
- 공통 UI 컴포넌트 식별 및 패턴 정의
- 상태 배지 및 경고 표시 규칙 정의
- 확인 다이얼로그 및 위험 액션 UX 정의
- 운영 화면 공통 액션 패턴 정의

### 제외 범위

- 실제 UI 스타일링 (색상, 디자인 시스템)
- 개별 관리 화면 상세 설계

---

## 3. 선행 조건

- Task 7-1 (Admin UI 정보 구조 및 접근 정책 설계) 완료

---

## 4. 주요 설계 대상

### 4-1. Admin 전체 레이아웃 구조

#### 기본 구조

- Top Header
- Left Sidebar (Navigation)
- Breadcrumb Bar
- Main Content Area
- 선택적 상세 패널 (Side Panel / Drawer)

#### 요구사항

- 모든 Admin 화면에서 동일한 구조 유지
- 데이터 밀도 높은 운영 환경에 최적화
- 상세 정보는 페이지 이동 없이 Side Panel로 표시 가능한 구조

---

### 4-2. Top Header 구성

Admin Header에 포함될 요소 정의:

- 시스템 로고 / Admin 식별 레이블
- 현재 로그인한 관리자 정보
- 글로벌 알림 / 경고 지표
- 운영 환경 표시 (Production / Staging 구분)

---

### 4-3. Left Sidebar (Navigation)

Admin 전용 네비게이션 구조:

#### 포함 항목

- Dashboard
- Users (사용자 / 조직 / 역할)
- Permissions (권한 정책)
- Document Types
- Audit Log
- Background Jobs
- Indexing
- API Keys

#### 요구사항

- 현재 위치 강조 (active state)
- 권한에 따른 메뉴 노출 제어 반영
- 섹션 그룹핑 (운영 / 보안 / 모니터링)

---

### 4-4. Breadcrumb 구조

- 현재 페이지 위치를 계층적으로 표시
- Admin > Users > {user_name} 형태
- 상위 경로 클릭 시 이동 가능

---

### 4-5. 공통 테이블 패턴

운영 목록 화면에 공통으로 사용하는 테이블 패턴 정의:

#### 구성 요소

- 컬럼 헤더 (정렬 가능)
- 행 선택 (단일/다중)
- 페이지네이션 또는 무한 스크롤 기준 정의
- 행 클릭 시 상세 패널 또는 상세 페이지 진입
- 빈 상태 / 로딩 상태 처리

#### 요구사항

- 컬럼 너비 조정 가능
- 상태 배지 표시 일관성

---

### 4-6. 검색 및 필터 패턴

- 상단 검색바: 키워드 검색
- 필터 패널: 상태/날짜/타입 등 다중 조건 필터
- 활성 필터 표시 및 초기화 버튼

---

### 4-7. 상태 배지 규칙

운영 화면에서 사용하는 상태 표현 기준 정의:

| 상태 유형 | 예시 값 | 시각 표현 |
|-----------|---------|-----------|
| 정상/활성 | ACTIVE, SUCCESS | 녹색 배지 |
| 비활성/중단 | INACTIVE, DISABLED | 회색 배지 |
| 진행 중 | PENDING, RUNNING | 파란색 배지 |
| 경고 | WARNING, RETRYING | 노란색 배지 |
| 실패/오류 | FAILED, ERROR | 빨간색 배지 |

---

### 4-8. 상세 패널 (Side Panel / Drawer)

목록에서 항목 선택 시 우측에 표시되는 상세 정보 패널:

#### 구성

- 항목 핵심 정보 요약
- 주요 메타데이터
- 액션 버튼 (편집, 상태 변경 등)

#### 요구사항

- 목록 화면을 유지하면서 상세 확인 가능
- 패널 열기/닫기 전환 가능

---

### 4-9. 확인 다이얼로그 및 위험 액션 UX

운영에서 취소 불가한 액션에 대한 확인 절차 정의:

#### 일반 확인 다이얼로그

- 메시지 + 확인/취소 버튼

#### 위험 액션 다이얼로그

- 위험성 경고 메시지 강조
- 변경 대상 명시
- (선택) 변경 사유 입력 필드
- Danger 스타일 확인 버튼

#### 적용 대상

- 사용자 비활성화
- 권한 정책 변경
- API 키 폐기
- DocumentType 비활성화

---

### 4-10. 공통 상태 표현 패턴

#### Loading

- 테이블 Skeleton
- 상세 패널 Spinner

#### Empty

- 데이터 없음 안내 메시지
- 관련 생성/추가 액션 유도

#### Error

- 오류 메시지 표시
- 재시도 버튼

#### Permission Denied

- 접근 권한 없음 안내
- 상위 화면 복귀 안내

---

### 4-11. 공통 컴포넌트 분류

#### Layout

- AdminLayout
- AdminHeader
- AdminSidebar
- Breadcrumb

#### Data Display

- DataTable
- StatusBadge
- SidePanel / Drawer

#### Controls

- SearchBar
- FilterPanel
- ConfirmDialog
- DangerDialog

#### Feedback

- LoadingSkeleton
- EmptyState
- ErrorState
