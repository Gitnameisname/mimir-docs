# Task 7-12. 구현 우선순위 및 MVP 범위 확정

## 1. 작업 목적

Phase 7 Admin UI의 실제 개발 착수 순서와 MVP 범위를 확정한다.

이 작업의 핵심 목표는 다음과 같다.

- 설계된 모든 화면 중 MVP에서 반드시 구현할 항목과 이후 확장 항목을 분리한다
- 운영상 가장 시급한 화면의 우선순위를 설정한다
- MVP 완료 기준을 명확히 정의하고 후속 Phase와의 연결 포인트를 정리한다

---

## 4. 주요 설계 대상

### 4-1. 화면별 중요도 평가 기준

각 화면의 MVP 포함 여부를 다음 기준으로 평가한다:

| 평가 기준 | 설명 |
|----------|------|
| 운영 필수성 | 플랫폼 운영에 반드시 필요한가 |
| 보안 중요도 | 권한/감사 관련 핵심 기능인가 |
| 의존성 | 다른 화면 구현의 선행 조건인가 |
| 구현 복잡도 | 단기간 구현 가능한가 |
| 후속 Phase 연계 | 향후 기능 확장의 기반이 되는가 |

---

### 4-2. MVP 포함 화면 (Phase 7 필수)

| 순위 | 화면 | 근거 |
|------|------|------|
| 1 | Admin UI 공통 레이아웃 | 모든 화면의 기반 |
| 2 | 시스템 상태 대시보드 | 운영자 첫 화면, 전체 현황 파악 |
| 3 | 사용자 목록/상세 조회 | 계정 관리 운영 필수 |
| 4 | 역할 및 기본 권한 조회 | 보안 감사 필수 |
| 5 | 감사 로그 조회 | 보안 추적 운영 필수 |
| 6 | DocumentType 목록/상세 조회 | 플랫폼 설정 파악 필수 |
| 7 | 백그라운드 작업 상태 조회 | 운영 모니터링 필수 |
| 8 | Admin UI API 연계 매핑 | 구현 기준 문서 |

---

### 4-3. MVP 이후 확장 화면 (Phase 7 확장 또는 후속 Phase)

| 화면 | 분류 | 이유 |
|------|------|------|
| 권한 정책 편집 | Phase 7 확장 | 조회는 MVP, 편집은 신중한 설계 필요 |
| 사용자 상태 변경 액션 | Phase 7 확장 | 조회 MVP 완료 후 추가 |
| 조직 구조 상세 관리 | Phase 7 확장 | 복잡한 매핑 편집 |
| API 키 발급/폐기 | Phase 7 확장 | 보안 중요, 충분한 검토 필요 |
| 외부 연계 설정 관리 | Phase 7 확장 | 외부 의존성 있음 |
| 인덱싱 재시도/재처리 | Phase 7 확장 또는 Phase 10 | Phase 10과 연계 |
| 고급 필터/통계/시각화 | Phase 후속 | 기본 운영 안정화 이후 |
| 대시보드 실시간 갱신 | Phase 후속 | WebSocket 구현 필요 |

### 4-5. MVP 완료 기준

다음 조건을 모두 만족할 때 Phase 7 MVP 완료로 본다:

#### 구조/공통

- [ ] Admin UI 공통 레이아웃이 구현되어 있다
- [ ] User UI와 라우팅이 완전히 분리되어 있다
- [ ] 관리자 권한 기반 접근 제어가 적용되어 있다

#### 핵심 화면

- [ ] 시스템 상태 대시보드가 동작하며 핵심 지표를 표시한다
- [ ] 사용자 목록/상세 화면이 동작한다
- [ ] 역할 및 기본 권한 조회가 가능하다
- [ ] 감사 로그 목록 및 필터링이 동작한다
- [ ] DocumentType 목록/상세 조회가 동작한다
- [ ] 백그라운드 작업 상태 목록 조회가 동작한다

#### 보안/감사

- [ ] 모든 Admin 액션에 권한 검증이 적용된다
- [ ] 주요 관리 액션이 감사 이벤트로 기록된다
