## 3. 발견 결함

### BUG-01 · 중요(High) · [수정 완료] 워크플로 발행(Publish) 시 문서/버전 상태 미동기화

| 항목 | 내용 |
|------|------|
| **발생 위치** | `backend/app/services/workflow_service.py` — `perform_action()` |
| **증상** | `workflow/publish` API 호출 성공 후 `workflow_status`는 `"published"`로 변경되나, `versions.status`, `versions.published_by`, `versions.published_at`, `documents.current_published_version_id`, `documents.status`가 모두 이전 값(`"draft"`, `null`) 유지 |
| **재현 절차** | 1. 문서 생성 → 2. Draft 저장 → 3. submit-review → 4. approve → 5. publish API 호출 → 6. `GET /documents/{id}` 조회 |
| **영향** | 검색 결과에서 발행 문서로 노출되지 않음. 프론트엔드에서 `current_published_version_id: null`로 발행 버전 추적 불가. `published_at` 미기록으로 발행 이력 손실 |
| **원인** | `workflow_repository.update_workflow_status()`가 `versions.workflow_status` 컬럼만 갱신. PUBLISH 전이 시 필요한 `versions.update_status()` / `documents_repository.update_version_pointers()` / `documents.status` 업데이트 누락 |
| **수정** | `workflow_service.py`의 `perform_action()` — `target_status == WorkflowStatus.PUBLISHED` 조건 분기 추가:<br>① `versions.update_status(status="published", published_by, published_at)` 호출<br>② `documents_repository.update_version_pointers(current_published_version_id=version_id)` 호출<br>③ `documents.status = 'published'` 직접 UPDATE |
| **수정 파일** | `backend/app/services/workflow_service.py` |
| **검증** | 수정 후 publish API 호출 → `documents.status: "published"`, `current_published_version_id: <version_id>` 정상 확인 |
| **상태** | ✅ 수정 완료 |

### BUG-02 · Medium · [수정 완료] 벡터화 서비스 — nodes 테이블 미연동

| 항목 | 내용 |
|------|------|
| **발생 위치** | `backend/app/services/vectorization_service.py` — `_fetch_nodes()` |
| **증상** | `PUT /draft` API로 저장한 문서는 `nodes` 테이블에 행이 없어 벡터화 시 `chunks_created: 0` |
| **원인** | `save_draft`(content_snapshot 경로)는 `nodes` 테이블에 직접 기록하지 않음. `_fetch_nodes()`가 `nodes` 테이블만 조회 |
| **수정** | `_parse_nodes_from_snapshot()` 메서드 추가: `versions.content_snapshot`(ProseMirror JSON) 파싱 → `nodes` 테이블 INSERT 후 반환 |
| **상태** | ✅ 수정 완료 |

### BUG-03 · High · [수정 완료] 벡터화 트랜잭션 롤백 — 토큰 사용량 기록 FK 오류

| 항목 | 내용 |
|------|------|
| **발생 위치** | `vectorization_service.py` — `_record_token_usage()` |
| **증상** | `chunks_created: 2` 반환에도 `document_chunks` 테이블에 행이 없음 |
| **원인** | `_record_token_usage()`에서 `embedding_token_usage.job_id` FK 오류 발생 시 psycopg2 트랜잭션이 aborted 상태로 전환 → 이후 `conn.commit()` 실패 → 청크 전체 롤백 |
| **수정** | `_record_token_usage()` 내부에 `SAVEPOINT sp_token_usage` / `ROLLBACK TO SAVEPOINT` 추가 |
| **상태** | ✅ 수정 완료 |

### BUG-04 · Medium · [수정 완료] pgvector 컬럼 차원 불일치 (1536 vs 768)

| 항목 | 내용 |
|------|------|
| **발생 위치** | `document_chunks.embedding` 컬럼, `app/config.py` |
| **증상** | SBERT 임베딩(768차원) 저장 시 `expected 1536 dimensions, not 768` 오류 |
| **원인** | `document_chunks.embedding` 컬럼이 `vector(1536)`으로 생성됨. 로컬 SBERT 모델은 768차원 출력 |
| **수정** | `ALTER TABLE document_chunks ALTER COLUMN embedding TYPE vector(768)`, HNSW 인덱스 재생성, `config.py` 기본값 768로 변경 |
| **상태** | ✅ 수정 완료 |

### BUG-05 · Medium · [수정 완료] 로컬 SBERT 임베딩 프로바이더 미구현

| 항목 | 내용 |
|------|------|
| **발생 위치** | `backend/app/services/embedding_service.py`, `app/config.py` |
| **증상** | `EMBEDDING_SVC_URL` 설정에도 OpenAI 프로바이더 사용 → 403 Permission Denied |
| **원인** | `get_embedding_provider()`가 `OPENAI_API_KEY` 존재 시 항상 OpenAI 사용. 로컬 SBERT 서비스 호출 프로바이더 미구현. `EMBEDDING_SVC_URL` 환경변수가 설정 필드에 매핑되지 않음 |
| **수정** | `SBERTEmbeddingProvider` 클래스 신규 구현 (`POST /embed`), `config.py`에 `embedding_svc_url`·`embedding_dim` 필드 추가, `get_embedding_provider()` 우선순위 변경 (로컬 서비스 > OpenAI > placeholder) |
| **상태** | ✅ 수정 완료 |
