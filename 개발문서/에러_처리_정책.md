# Mimir 에러 처리 정책

**작성일**: 2026-04-18  
**적용 범위**: `backend/app/` 전체

---

## 1. 핵심 원칙

에러를 두 종류로 구분한다:

| 종류 | 정의 | 처리 방식 |
|------|------|-----------|
| **Critical** | 요청을 정상적으로 처리할 수 없는 상태 | 예외를 re-raise하거나 `ApiError` 변환 후 호출자에게 전파 |
| **Recoverable** | 일부 기능 저하는 있지만 폴백으로 서비스 가능한 상태 | `logger.warning` 기록 후 폴백 값 반환 |

**`except Exception: pass` 는 절대 허용하지 않는다.**

---

## 2. 레이어별 책임

### API 레이어 (`app/api/`)
- HTTP 요청 처리 중 발생한 에러는 `ApiError`로 변환하여 클라이언트에 반환.
- `handlers.py`가 `RequestValidationError`, `ApiError`, 일반 `Exception`을 일괄 처리.
- 감사 이벤트(`log_api_event`) 기록 실패는 `logger.warning` 후 무시 — 감사 실패가 서비스 실패를 유발해서는 안 됨.

### 서비스 레이어 (`app/services/`)
- **Critical 경로**: DB 저장, 벡터화, 임베딩 등 비즈니스 로직 핵심 연산 → `ApiError` 또는 도메인 예외 re-raise.
- **Recoverable 경로**: 플러그인 설정 조회, 프롬프트 템플릿 로드, 외부 서비스 optional call → `logger.warning` + 폴백.

```python
# Good — Recoverable 폴백
try:
    config = plugin.get_config()
except Exception as exc:
    logger.warning("플러그인 config 조회 실패 (%s), 기본값 사용: %s", document_type, exc)
    config = DEFAULT_CONFIG

# Good — Critical 전파
try:
    result = await embedding_service.embed(text)
except Exception as exc:
    logger.error("임베딩 실패: %s", exc)
    raise ApiError(503, "임베딩 서비스를 사용할 수 없습니다") from exc
```

### Repository 레이어 (`app/repositories/`)
- DB 에러는 그대로 상위로 전파 (서비스 레이어에서 처리).
- 트랜잭션 롤백 실패는 `logger.warning` 후 원본 예외를 다시 raise.

### 배경 작업 (`app/services/*_batch.py`, `background_tasks.py`)
- 작업 단위(row/document) 실패는 `logger.error` 후 다음 항목 진행 (`continue`).
- 작업 전체 실패는 `logger.error` + `log_api_event(actor_type="system", result="failure")`.
- 롤백 실패는 `logger.warning` (원본 예외 이미 기록됨).

## 3. 금지 패턴

```python
# Bad — 예외 정보 소실
except Exception:
    pass

# Bad — 변수 없는 bare except
except Exception:
    return default_value  # 왜 실패했는지 알 수 없음

# Bad — 너무 넓은 except (비즈니스 로직을 포함하는 경우)
try:
    entire_function_body()
except Exception:
    logger.error("something went wrong")
```

## 4. 로그 레벨 선택 기준

| 레벨 | 사용 시점 |
|------|-----------|
| `logger.debug` | 내부 디버그 정보 (캐시 miss, SSE 파싱 등) |
| `logger.info` | 정상 완료 이벤트 (생략 가능) |
| `logger.warning` | Recoverable 폴백, 추적 실패, 예상 가능한 외부 오류 |
| `logger.error` | Critical 경로 실패, 관리자 확인 필요 |
| `logger.exception` | `logger.error` + 스택 트레이스 (동기 컨텍스트에서 상세 진단 필요 시) |
