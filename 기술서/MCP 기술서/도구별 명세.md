## 1. search_documents (L0, stable)

**의도**: FTS / 하이브리드(벡터+FTS) 문서 검색. 일반 질의응답.

**입력**:
```json
{
  "query": "string (1..500)",
  "scope": "string (optional, ScopeProfile.scope_name)",
  "document_types": ["POLICY", ...],
  "top_k": 5,
  "conversation_id": "uuid",
  "access_context": {...}
}
```

**출력 (data)**:
```json
{
  "results": [
    {
      "document_id": "uuid",
      "document_title": "string",
      "version_id": "uuid",
      "node_id": "string",
      "content": "string (snippet)",
      "citation": {"document_id": "...", "version_id": "...", "node_id": "...", "content_hash": "..."},
      "relevance_score": 0.85,
      "_envelope": { ... 항목별 MCPItemEnvelope (FG 4-1 §2.1.5) ... }
    }
  ],
  "total_count": 5,
  "retrieval_method": "fts"
}
```

**envelope**: `content_role=retrieved_evidence`, `streaming_supported=True`. SSE 호출 시 `_envelope` 가 각 결과 청크에 부착.

**ACL**: ScopeProfile 의 `_resolve_acl_filter` post-filter (document_id 집합).

## 3. verify_citation (L1, stable) — FG 4-3 5중 검증

**의도**: 인용의 시간 후 변조 검증. 5 검사 모두 통과해야 `verified=True`.

**입력**:
```json
{
  "document_id": "uuid",
  "version_id": "vN or UUID — 'latest' 거부 (R3)",
  "node_id": "string",
  "content_hash": "sha-256 hex",
  "citation_basis": "node_content | rendered_text",
  "quoted_text": "string (선택)",
  "span_offset": 0,
  "span_length": 100,
  "access_context": {...}
}
```

**출력**:
```json
{
  "verified": true,
  "checks": {
    "exists": true,
    "pinned": true,
    "hash_matches": true,
    "quoted_text_in_content": true,
    "span_valid": true
  },
  "current_hash": "...",
  "rendered_snapshot": null,
  "node_snapshot": "...",
  "message": "검증 성공."
}
```

**5 검사**:
1. **exists** — document/version/node 모두 존재
2. **pinned** — `version_id="latest"` 즉시 거부 + `versions.status` 가 `published`/`archived` 만 통과 (draft 거부)
3. **hash_matches** — `citation_basis` 분기:
   - `node_content`: SHA-256(`document_chunks.source_text`)
   - `rendered_text`: SHA-256(rendered text via `read_document_render` 와 동일 walker)
4. **quoted_text_in_content** — `quoted_text` 입력 시 정본 텍스트에 포함되는지 (None 시 검사 skip)
5. **span_valid** — `span_offset` + `span_length` 입력 시 텍스트 범위 내 (None 시 skip)

**envelope**: `content_role=tool_metadata`, detected_risks 적용 안 함.

**상세**: [Citation 모델.md](Citation%20모델.md)


**출력**:

## 6. read_document_render (L0, beta) — FG 4-2

**의도**: ProseMirror 트리가 아닌 **렌더링된 텍스트** + node_anchors 매핑. citation_basis=rendered_text 의 정본.

**입력**:
```json
{
  "document_id": "uuid",
  "version_id": "vN | UUID | 'latest' (자동 resolve)",
  "format": "plain_text | markdown",
  "include_node_anchors": true,
  "access_context": {...}
}
```

**출력**:
```json
{
  "document_id": "...", "version_id": "vN (resolved)",
  "format": "plain_text",
  "rendered_text": "전체 텍스트",
  "render_hash": "sha-256 hex",
  "node_anchors": [
    {"node_id": "...", "offset_start": 0, "offset_end": 25}, ...
  ]
}
```

**envelope**: `source = mimir://documents/{id}/versions/{vN}/render`. detected_risks 적용 (rendered_text 스캔).

**R3 강제**: `version_id="latest"` 입력 → `resolve_version_id(conn, doc_id, "latest")` 로 즉시 vN resolve. 응답의 `version_id` 는 항상 구체값.

**구현**: `RenderService.render_version` + `_walk_blocks_for_text` (plain/markdown 두 모드 + rstrip 정규화 — render_hash 결정성).

## 7. resolve_document_reference (L1, experimental) — FG 4-2

**의도**: 자연어 참조 (`"그 영업 매뉴얼"`, `"v2024 보안 정책"`) → `document_id` + `version_ref` 정규화. confidence + needs_disambiguation.

**입력**:
```json
{
  "reference": "string (1..500)",
  "scope": "string",
  "preferred_doc_types": ["POLICY", ...],
  "context": {"recent_document_ids": ["..."], "conversation_id": "..."},
  "max_candidates": 5,
  "confidence_threshold": 0.85,
  "access_context": {...}
}
```

**출력**:
```json
{
  "resolved": true,
  "needs_disambiguation": false,
  "best_match": {
    "document_id": "...", "version_ref": "latest_published",
    "title": "...", "confidence": 0.99, "match_kind": "exact_title",
    "_envelope": { ... }
  },
  "candidates": [...]
}
```

**5 단계 알고리즘** (`document_resolver_service.resolve_reference`):
1. **exact_title** — 제목 정확 일치 → confidence 0.99
2. **alias** — `documents.metadata.aliases` / `previous_titles` 일치 → 0.95
3. **recent_context** — `context.recent_document_ids` 안 부분 일치 → 0.85
4. **semantic** — `SearchService.search_documents_hybrid` (벡터) → ∈ [0.0, 0.94]
5. **fts_fallback** — `MIMIR_OFFLINE=1` 또는 벡터 인프라 fail 시 자동 → ∈ [0.0, 0.90]

각 단계가 `allowed_doc_ids` 검사. dedupe (같은 document_id 는 가장 높은 confidence 만). 임계 미달 시 `needs_disambiguation=True` + `best_match=null`.

**envelope**: `content_role=tool_metadata`, source = null (후보별 envelope source 사용). R3: `version_ref` 는 항상 `"vN"` 또는 `"latest_published"` — 단독 `"latest"` 절대 미반환.

## 9. save_draft (L2, experimental) — FG 4-6 — **MCP 표면 최초 쓰기 도구**

**의도**: 에이전트가 사람 reviewer 의 승인을 받아 draft 변경을 제안. **자동 머지 0** — propose 만.

**4 사전 조건**:
1. **idempotency** — `(agent_id, idempotency_key)` UNIQUE INDEX
2. **human approval** — `agent_proposal_service.propose_draft` 위임 (S2 Phase 5 자산), agent self-approve 거부
3. **impact preview** — `compute_draft_impact` 가 nodes/chars 차이 사전 계산
4. **감사 로그 4종** — `agent_proposal.{requested,approved,merged,rolled_back}`

**입력**:
```json
{
  "document_id": "uuid (optional, None 이면 신규 문서)",
  "document_type_id": "string (신규 문서 시)",
  "title": "string (max 500)",
  "content_snapshot": { "type": "doc", "content": [...] },
  "metadata": {...},
  "reason": "string (max 1000)",
  "idempotency_key": "string (1..128) — 필수",
  "scope": "string",
  "access_context": {...}
}
```

**출력**:
```json
{
  "proposal_id": "...",
  "status": "proposed",
  "document_id": "...",
  "version_id": "v-new",
  "impact": {
    "document_id": "...", "target_version_id": null,
    "overwrites_existing_draft": false,
    "nodes_added": 1, "nodes_modified": 0, "nodes_deleted": 0,
    "chars_added": 25, "chars_removed": 0,
    "summary": "신규 문서 생성 — 노드 1개, 본문 25자."
  },
  "requires_human_approval": true,
  "audit_event": "agent_proposal.requested",
  "message": "draft 가 제안되었습니다. reviewer approval 후 적용됩니다."
}
```

**write_envelope** (read envelope 와 분리):
```json
{
  "content_role": "mutation_proposed",
  "instruction_authority": "none",
  "impact": { ... },
  "proposal_id": "...",
  "requires_human_approval": true,
  "audit_chain": ["agent_proposal.requested"]
}
```

**ACL**:
- `_check_tool_allowed("save_draft")` — ScopeProfile.allowed_tools 등록 의무 (default_enabled=False — 운영자 명시 등록)
- `_ensure_document_allowed` — 다른 scope 의 document 거부
- agent_id 부재 → 403

**Idempotent Replay**: 같은 `(agent_id, idempotency_key)` 재호출 → `_lookup_idempotent_proposal` 가 기존 proposal 재사용 (새 INSERT 0). 응답에 `idempotent_replay=True` 표시.

**상세**: [Tool ACL 모델.md](Tool%20ACL%20모델.md), [API 기술서/관리자 API.md](../API%20기술서/관리자%20API.md) (proposal approve/reject 경로)

## 부록 A — 응답 헤더 / 에러 코드

| code (MCPErrorCode) | HTTP | 의미 |
|---|---|---|
| `UNAUTHORIZED` | 401/403 | 인증 부재 또는 ACL 미통과 |
| `NOT_FOUND` | 404 | 리소스 부재 |
| `INVALID_SCOPE` | 400 | scope 어휘 잘못됨 |
| `INVALID_CITATION` | 400 | citation 검증 실패 |
| `RATE_LIMIT` | 429 | 도구별/에이전트별 한도 초과 |
| `INVALID_REQUEST` | 400/409 | 요청 형식 오류 / 충돌 |
| `AGENT_DISABLED` | 403 | 킬스위치 활성화됨 |
| `INTERNAL_ERROR` | 500 | 도구 실행 오류 |
