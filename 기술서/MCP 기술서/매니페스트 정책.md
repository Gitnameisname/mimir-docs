# MCP 매니페스트 정책

도구 매니페스트의 4 + 5 = 9 필드 + 외부/운영자 view 분리 + drift 게이트.

**선행 문서**: [MCP 표면 개요.md](MCP%20표면%20개요.md)
**소스 정본**: `backend/app/schemas/mcp.py::TOOL_SCHEMAS`

## 1. 9 manifest 필드

### 1.1 Phase 4 1라운드 (FG 4-0) — 4 필드

| 필드 | Literal | 의미 |
|---|---|---|
| `risk_tier` | `L0`/`L1`/`L2`/`L3`/`L4` | 헌법 제20조 Typed Action Risk |
| `maturity` | `stable`/`beta`/`experimental`/`disabled`/`forbidden` | 운영 단계 — `forbidden` = R1 절대 금지 |
| `status` | `enabled`/`disabled`/`not_exposed` | 런타임 노출 상태 — `not_exposed` = tools/list 미응답 + tools/call 거부 |
| `exposure_policy` | `MCP_ENABLED`/`MCP_DISABLED`/`REST_ADMIN_ONLY` | 인터페이스 정책 |

### 1.2 Phase 4 이월 (FG 4-5) — 5 필드

| 필드 | 타입 | 의미 |
|---|---|---|
| `default_enabled` | bool | 신규 ScopeProfile 의 `use_defaults=True` 시 자동 등록 후보 |
| `requires` | list[str] | 의존성 — 다른 capability/도구 (런타임 검사는 별 라운드) |
| `preferred_use` | str | 권장 사용 사례 (자연어, 외부 안내) |
| `policy_profile` | `read_safe`/`write_audited`/`admin_only`/`experimental` | 운영 그룹화 (런타임 분기는 별 라운드) |
| `streaming_supported` | bool | SSE 스트림 (`tools/call/stream`) 가능 여부 |

## 2. 외부/운영자 view 분리 정책 (도구등급화_매핑.md §4 옵션 A)

| 필드 | 외부 (`tools/list` / `mcp_exposed_public_view`) | 운영자 (`/admin/mcp/manifest` / `mcp_admin_full_view`) |
|---|---|---|
| `name` / `description` / `maturity` | ✅ | ✅ |
| `requires` / `preferred_use` / `streaming_supported` (FG 4-5) | ✅ | ✅ |
| `risk_tier` / `status` / `exposure_policy` | ❌ | ✅ |
| `default_enabled` / `policy_profile` (FG 4-5) | ❌ | ✅ |
| `authentication` / `inputSchema` | ✅ | ✅ |

**근거**:
- `default_enabled` / `policy_profile` 은 **운영 정찰 가능성** — 외부가 어떤 도구가 default 인지 / 어떤 그룹인지 추론하면 우회 시도에 도움이 됨. 운영자 전용.
- `requires` 는 클라이언트가 의존성 인지해야 동작 (예: search → fetch_node 흐름).
- `preferred_use` 는 외부 안내용 (description 보강).
- `streaming_supported` 는 클라이언트가 stream endpoint 호출 분기에 필요.
- `risk_tier` 비노출 — 본 정책의 핵심: 외부에 등급을 알리지 않음.

## 3. 헬퍼 함수 (`backend/app/schemas/mcp.py`)

| 함수 | 용도 |
|---|---|
| `known_tool_names() -> frozenset[str]` | TOOL_SCHEMAS 의 도구 이름 집합 — `ScopeProfile.allowed_tools` 검증 정본 |
| `is_tool_mcp_exposed(tool) -> bool` | R1 차단 (L4 / forbidden / not_exposed 거부) — 정의 단계 안전망 |
| `mcp_exposed_tool_schemas() -> list[dict]` | 노출 도구만 반환 — `_CURATED_TOOLS` / `tools/list` 정본 |
| `mcp_exposed_public_view(tool) -> dict` | 외부 응답용 — 화이트리스트 키만 |
| `mcp_admin_full_view(tool) -> dict` | 운영자 응답용 — 전체 (forward-compat 위해 unknown 필드 보존) |
| `default_enabled_tool_names() -> list[str]` | `default_enabled=True` 인 노출 도구 (정렬) |
| `tools_by_policy_profile(profile) -> list[str]` | profile 별 노출 도구 (정렬) |

## 4. Drift 게이트 (FG 4-4 §2.1.6)

`backend/scripts/dump_mcp_manifest.py --check` — `MCP_도구_매니페스트.json` 가 코드 상수와 동기화되어 있는지 검증.

**CI 통합** (운영자 후속):
```yaml
manifest_drift:
  stage: test
  script:
    - cd backend && python scripts/dump_mcp_manifest.py --check
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'
  needs: ["unit_tests"]
```

차이가 있으면 fail → 개발자가 `python scripts/dump_mcp_manifest.py` 재실행 후 commit 의무.

## 5. R1 (L4 차단) 자동 보호

**3 계층 방어**:

1. **정의 단계 안전망** (compile time)
   - TOOL_SCHEMAS 에 L4/forbidden/not_exposed 등재 0 — 회귀 테스트가 발견 즉시 fail
   - [test_not_exposed_tools.py::TestNoForbiddenInToolSchemas](../../../backend/tests/integration/contract_drift/test_not_exposed_tools.py)

2. **런타임 차단** (`is_tool_mcp_exposed`)
   - `risk_tier="L4"` 또는 `maturity="forbidden"` 또는 `status="not_exposed"` 거부
   - `_CURATED_TOOLS` (`mcp_exposed_tool_schemas()` 결과) 외 도구 호출 시 INVALID_REQUEST

3. **REST 관리자 도구 분리**
   - `publish_document` / `reindex_document` / `change_schema` / `delete_document` 는 **TOOL_SCHEMAS 에 절대 추가하지 않음** (도구등급화_매핑.md §3.3)
   - REST_ADMIN_ONLY — 사람 관리자 UI 만

**가짜 L4 회귀**: TOOL_SCHEMAS 에 가짜 L4 도구 임시 주입 → `_CURATED_TOOLS` 자동 제외 입증 ([test_not_exposed_tools.py::TestFakeL4ToolBlocked](../../../backend/tests/integration/contract_drift/test_not_exposed_tools.py)).

## 6. 변경 이력 — 도구 추가 시 절차

도구 추가 시 의무 단계 (manifest drift 게이트가 자동 강제):

1. `TOOL_SCHEMAS` 에 entry 추가 — 9 필드 모두 default 명시
2. `tool_<name>` 함수 신설 (`backend/app/mcp/tools.py`) — `_check_agent_write_blocked` + `_check_tool_allowed` 첫 줄
3. `_dispatch_tool` 분기 추가 (`backend/app/api/v1/mcp_router.py`)
4. `_build_envelope` (read) 또는 `_build_write_envelope` (write) 분기 추가
5. `_run_injection_detection` 본문 추출 갱신 (해당 시)
6. 테스트 작성 — manifest 등재 + envelope + ACL + R1~R6 회귀
7. `python scripts/dump_mcp_manifest.py` 재생성 + commit
8. Admin UI 가 동적 fetch 라 별도 변경 0 (FG 4-5)
