## documents

문서 플랫폼의 최상위 엔티티.

```sql
CREATE TABLE documents (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title       VARCHAR(500) NOT NULL,
    document_type VARCHAR(100) NOT NULL,        -- FK 없음, DocumentType은 플러그인으로 관리
    status      VARCHAR(50) NOT NULL DEFAULT 'draft',
    metadata    JSONB NOT NULL DEFAULT '{}',    -- 타입별 커스텀 필드
    summary     TEXT,
    created_by  VARCHAR(255),
    updated_by  VARCHAR(255),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    -- Phase 4 추가
    current_draft_version_id     UUID REFERENCES versions(id) ON DELETE SET NULL,
    current_published_version_id UUID REFERENCES versions(id) ON DELETE SET NULL,
    -- Phase 8 추가
    search_vector tsvector        -- FTS 인덱스 컬럼 (트리거로 자동 갱신)
);
```

**인덱스:**
- `idx_documents_status` — status 필터링
- `idx_documents_document_type` — 유형 필터링
- `idx_documents_created_at DESC` — 기본 정렬
- `idx_documents_updated_at DESC` — 수정 시간 정렬
- `idx_documents_title` — 제목 정렬/필터
- `idx_documents_search_vector (GIN)` — FTS

**status 값:** `draft`, `published`, `archived`, `deprecated`

## versions

문서의 버전 단위 스냅샷.

```sql
CREATE TABLE versions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    document_id     UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    version_number  INTEGER NOT NULL,
    label           VARCHAR(200),
    status          VARCHAR(50) NOT NULL DEFAULT 'draft',
    change_summary  TEXT,
    source          VARCHAR(50) NOT NULL DEFAULT 'manual',  -- manual | system | import
    metadata        JSONB NOT NULL DEFAULT '{}',
    created_by      VARCHAR(255),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    -- Phase 4 추가
    parent_version_id          UUID REFERENCES versions(id) ON DELETE SET NULL,
    restored_from_version_id   UUID REFERENCES versions(id) ON DELETE SET NULL,
    title_snapshot             VARCHAR(500),      -- publish 시 문서 제목 복사
    summary_snapshot           TEXT,
    metadata_snapshot          JSONB,
    content_snapshot           JSONB,             -- 전체 노드 트리 JSON
    published_by               VARCHAR(255),
    published_at               TIMESTAMPTZ,
    -- Phase 5 추가
    workflow_status            VARCHAR(50),       -- draft | in_review | approved | published | rejected | archived
    -- Phase 8 추가
    search_vector              tsvector,
    UNIQUE (document_id, version_number)
);
```

**인덱스:**
- `idx_versions_document_id`
- `idx_versions_created_at DESC`
- `idx_versions_version_number` on `(document_id, version_number DESC)`
- `idx_versions_status` on `(document_id, status)`
- `idx_versions_search_vector (GIN)`

## search_index_stats (뷰)

FTS 인덱싱 현황 집계 뷰.

```sql
CREATE OR REPLACE VIEW search_index_stats AS
SELECT
    'documents' AS table_name,
    COUNT(*) AS total_rows,
    COUNT(*) FILTER (WHERE search_vector IS NOT NULL) AS indexed_rows,
    COUNT(*) FILTER (WHERE search_vector IS NULL) AS unindexed_rows
FROM documents
UNION ALL
SELECT 'versions', COUNT(*), ... FROM versions
UNION ALL
SELECT 'nodes', COUNT(*), ... FROM nodes;
```

# S3 신규 테이블 — 상세

## scope_profiles (S2 P4 + S3 확장)

에이전트 / 사용자에 바인딩되는 ACL 필터 템플릿. S3 에서 `settings_json` (FG 3-2) 과 `allowed_tools` (FG 4-0) 컬럼 추가.

```sql
CREATE TABLE scope_profiles (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name            VARCHAR(255) NOT NULL,
    description     TEXT,
    organization_id UUID,
    -- S3 Phase 3 FG 3-2 (boot DDL)
    settings_json   JSONB,           -- {expose_viewers: bool, ...}
    -- S3 Phase 4 FG 4-0 (Alembic s3_p4_scope_profile_allow_tools)
    allowed_tools   JSONB NOT NULL DEFAULT '[]'::jsonb,  -- MCP tool 화이트리스트
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_scope_profiles_allowed_tools_nonempty
    ON scope_profiles ((jsonb_array_length(allowed_tools) > 0))
    WHERE jsonb_array_length(allowed_tools) > 0;
```

**`allowed_tools` 정책**: 빈 배열 = default-deny. 등록 가능 도구는 `app.schemas.mcp.known_tool_names()` 만 (manifest 정합 강제).

**관련 문서**: [MCP 기술서/Tool ACL 모델.md](../MCP%20기술서/Tool%20ACL%20모델.md)

## agent_proposals (S2 + S3 P4 FG 4-6 확장)

에이전트 제안 통합 큐. S3 에서 `idempotency_key` 추가 (FG 4-6).

```sql
CREATE TABLE agent_proposals (
    id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    agent_id         UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    proposal_type    VARCHAR(50) NOT NULL,  -- 'draft' | 'transition'
    reference_id     UUID NOT NULL,
    status           VARCHAR(50) NOT NULL DEFAULT 'pending',
    reviewed_by      VARCHAR(255),
    review_notes     TEXT,
    review_timestamp TIMESTAMPTZ,
    -- S3 Phase 4 FG 4-6 (Alembic s3_p4_agent_prop_idempotency)
    idempotency_key  VARCHAR(128),
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    CONSTRAINT check_proposal_type CHECK (proposal_type IN ('draft', 'transition')),
    CONSTRAINT check_proposal_status CHECK (status IN ('pending', 'approved', 'rejected', 'withdrawn'))
);

CREATE UNIQUE INDEX idx_agent_proposals_agent_idempotency
    ON agent_proposals(agent_id, idempotency_key)
    WHERE idempotency_key IS NOT NULL;
```

**`idempotency_key`**: NULL 허용 (기존 호출자 호환). UNIQUE INDEX `WHERE NOT NULL` — 같은 (agent_id, key) 재호출 시 같은 proposal 반환 (`save_draft` 도구의 사전 조건).

```sql
CREATE INDEX idx_tags_name_normalized_pattern ON tags(name_normalized text_pattern_ops);

CREATE TABLE document_tags (
    document_id UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    tag_id      UUID NOT NULL REFERENCES tags(id) ON DELETE CASCADE,
    source      VARCHAR(12) NOT NULL DEFAULT 'inline',  -- inline | frontmatter | both
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (document_id, tag_id)
);
```

**name_normalized**: NFKC + lowercase + `[\w/-]{1,64}` 정규화 후 UNIQUE. 인라인 `#tag` + frontmatter `tags: [...]` 양쪽 지원 — 블로커1 결정서 합의.
