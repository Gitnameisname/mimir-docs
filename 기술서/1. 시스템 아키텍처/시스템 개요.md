# 시스템 개요

---

## 1. 플랫폼 정의

Mimir는 **범용 문서/지식 플랫폼**이다. 단순한 문서 저장소가 아니라, 조직의 모든 문서를 구조화하여 저장하고 검색·질의응답·워크플로 자동화까지 제공하는 인프라 레이어다.

**핵심 설계 원칙:**

1. **API-first** — 모든 기능은 REST API로 먼저 정의된다. UI는 API의 클라이언트일 뿐이다.
2. **Document = 구조, Type = 동작** — Document 모델은 Generic이고, DocumentType이 청킹·렌더링·워크플로 등 동작을 정의한다.
3. **권한은 모든 계층에 적용** — API, MCP, 검색, 벡터, RAG 모든 레이어에서 역할 기반 접근 제어를 적용한다.
4. **Dual UI** — User UI (업무 사용자)와 Admin UI (운영 관리자)를 처음부터 분리 설계한다.
5. **Agent-Facing Surface (S3 신설)** — 외부 AI 에이전트는 REST 가 아닌 **MCP (Mimir Control Protocol) 표면**을 통해 정본을 사용한다. ScopeProfile 기반 ACL · pinned citation · L0~L4 도구 등급 · L4 영구 차단(R1).

## 3. 주요 컴포넌트

### 3.1 Backend (FastAPI)

| 컴포넌트 | 경로 | 역할 |
|----------|------|------|
| `main.py` | `backend/app/main.py` | FastAPI 앱 생성, 미들웨어 등록, startup hook |
| API Router | `backend/app/api/v1/router.py` | `/api/v1/*` 모든 라우터 집계 |
| MCP Router | `backend/app/api/v1/mcp_router.py` | `/api/v1/mcp` dispatcher + envelope builder (S3 P4) |
| Auth Layer | `backend/app/api/auth/` | JWT 발급/검증, API Key 검증, 세션, Agent Principal (S3 P4) |
| Middleware | `backend/app/api/context/`, `security/` | RequestContext, 보안 헤더, 크기 제한 |
| Services | `backend/app/services/` | 모든 비즈니스 로직 |
| MCP Tools | `backend/app/mcp/tools.py` | 9 도구 함수 (search/fetch/verify/render/resolve 등) |
| MCP URI Builder | `backend/app/mcp/uri_builder.py` | mimir:// URI 4 패턴 빌더 + R3 강제 |
| Repositories | `backend/app/repositories/` | DB 접근 계층 (psycopg2) |
| Plugins | `backend/app/plugins/` | DocumentType 플러그인 레지스트리 |
| Observability | `backend/app/observability/` | 구조화 로그, Prometheus 메트릭 |

### 3.2 Frontend (Next.js 14)

| 영역 | 경로 | 역할 |
|------|------|------|
| App Router | `frontend/src/app/` | 페이지 라우팅 (Next.js App Router) |
| Features | `frontend/src/features/` | 기능별 컴포넌트 (documents, admin, rag, search, diff) |
| Components | `frontend/src/components/` | 공용 UI 컴포넌트 |
| API Client | `frontend/src/lib/api/` | Backend REST API 클라이언트 |
| Types | `frontend/src/types/` | TypeScript 타입 정의 |

### 3.3 Infrastructure

| 컴포넌트 | 구현 | 역할 |
|----------|------|------|
| DB | PostgreSQL 16 + pgvector | 모든 영구 데이터, 벡터 인덱스 (HNSW) |
| Cache | Valkey 8 | 세션 토큰, 응답 캐시, Rate Limit 카운터 |
| Monitoring | Prometheus + Grafana | HTTP 메트릭, 알림 |
| CI/CD | GitHub Actions | lint → test → build → deploy |

## 6. MCP 표면 (S3 Phase 4 신설)

외부 AI 에이전트는 **MCP (Mimir Control Protocol) 2025-11-25** dispatcher 를 통해 정본을 사용한다.

```
[Agent Client]
     │ POST /api/v1/mcp  { jsonrpc, method:"tools/call", params:{name, arguments} }
     │ + X-API-Key (agent_principal.api_key 매핑)
     ▼
[Auth Layer] → AgentPrincipal 추출 (api_keys.principal_kind="AGENT")
     │
     ▼
[ScopeProfile 로드] → scope_profiles.allowed_tools (JSONB) 조회
     │
     ▼
[Tool ACL] (R1·R2)
   1. manifest.exposure_policy ≠ "never"   (R1: L4 영구 차단)
   2. tool_name ∈ ScopeProfile.allowed_tools  (R2: ACL 단일 결정점)
     │
     ▼
[MCP Tool Execution]
   - L0 read: search_documents, search_nodes, fetch_node, read_annotations,
              read_document_render, resolve_document_reference,
              mimir.vectorization.status
   - L1 verify: verify_citation
   - L2 write (가역 propose): save_draft (4 사전 조건 — idempotency / human approval / impact preview / 4종 audit)
   - L3 워크플로 / L4 파괴: REST 전용 — MCP 영구 미노출 (R1·R5)
     │
     ▼
[Envelope Wrap]
   { source: { uri: "mimir://...", citation: {...} },
     scope:  { actor_kind, scope_profile_id, allowed_tools },
     audit:  { request_id, audited_at, event_type },
     detected_risks: [...]   ← R6: 차단 아닌 경고
   }
     │
     ▼
[Response]
```

**5 절대 규칙 (R1~R5) + R6** — 자세한 내용은 [보안 모델 §12](../6.%20보안%20모델/보안%20모델.md) 및 [MCP 표면 개요](../MCP%20기술서/MCP%20표면%20개요.md) 참조.
