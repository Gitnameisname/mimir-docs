# 보안 모델

---

## 1. 인증 보안

### JWT (HS256)

- 알고리즘: **HS256** (HMAC-SHA256)
- 필수 클레임: `sub` (actor_id), `exp` (만료 시각)
- 기본 만료: 120분
- Secret: `JWT_SECRET` 환경 변수 (운영 환경 필수, 미설정 시 startup 실패)
- `algorithms=["HS256"]` 명시적 지정 (알고리즘 혼용 공격 방지)

### API Key

- 원문 저장 안 함 — SHA-256 해시만 DB에 보관
- `key_prefix` (앞 8자리)로 인덱스 조회 → `hmac.compare_digest()`로 타이밍 공격 방지
- 범위(scope): `READ_ONLY` / `READ_WRITE`
- 만료일(expires_at) 설정 가능

### 세션 토큰

- `secrets.token_urlsafe(32)` 생성 (암호학적 난수)
- Valkey 저장: `session:{token}` 키, TTL 24시간

### 내부 서비스 토큰

- HMAC-SHA256 서명: `HMAC(INTERNAL_SERVICE_SECRET, "mimir-internal")`
- `hmac.compare_digest()` 타이밍 공격 방지
- Secret 미설정 시 자동 거부 (fail-closed)

## 2. 인가 (Authorization)

### 역할 기반 접근 제어 (RBAC)

6단계 역할: `VIEWER < AUTHOR < REVIEWER < APPROVER < ORG_ADMIN < SUPER_ADMIN`

모든 API 엔드포인트는 진입 시 역할 확인:
```python
require_authenticated(actor)   # 401 — 미인증
require_admin_access(actor)    # 403 — ORG_ADMIN 미만
```

### 심층 방어 (Defense-in-Depth)

권한 검사가 4개 계층에서 모두 적용된다:
1. **API 계층** — 엔드포인트 진입 시 역할 확인
2. **서비스 계층** — 리소스별 작업 권한 확인
3. **검색 계층** — 쿼리에 역할 기반 필터 자동 추가
4. **벡터/RAG 계층** — `accessible_roles`, `is_public` 컬럼 필터

## 3. 입력 유효성 검사

### 요청 크기 제한

`RequestSizeLimitMiddleware` — Content-Length 기준 10MB 초과 시 413.

### 파일명 보안

```python
sanitize_filename(filename)
# PurePosixPath(filename).name 으로 경로 순회 방지
# ".." 포함, "/" 포함 → 거부
```

### Null 바이트 방지

```python
contains_null_byte(s)
# "\x00" 포함 → True → 거부
```

### UUID 파라미터 검증

```python
validate_uuid_param(value)
# UUID 형식 아니면 400 반환
```

### X-Request-ID 길이 제한

128자 초과 시 잘라냄 (로그 폭탄 방지).

## 5. 보안 헤더

`SecurityHeadersMiddleware`가 모든 응답에 다음 헤더를 추가한다.

| 헤더 | 값 |
|------|-----|
| `X-Content-Type-Options` | `nosniff` |
| `X-Frame-Options` | `DENY` |
| `Content-Security-Policy` | `default-src 'none'; frame-ancestors 'none'` |
| `Strict-Transport-Security` | `max-age=31536000; includeSubDomains` (production만) |
| `Referrer-Policy` | `strict-origin-when-cross-origin` |
| `Permissions-Policy` | `geolocation=(), microphone=(), camera=()` |
| `Server` | (제거됨) |

## 6. SQL 인젝션 방지

psycopg2 파라미터화 쿼리를 일관되게 사용한다.

```python
# ✅ 안전 — 사용자 입력을 파라미터로 전달
cur.execute("SELECT * FROM users WHERE email = %s", (email,))

# ✅ 안전 — Safe Dynamic WHERE 패턴
conditions = []
params = []
if search:
    conditions.append("title ILIKE %s")
    params.append(f"%{search}%")   # 사용자 입력은 파라미터로
where = "WHERE " + " AND ".join(conditions) if conditions else ""
cur.execute(f"SELECT * FROM documents {where}", params)

# ❌ 위험 — 절대 금지
cur.execute(f"SELECT * FROM users WHERE email = '{email}'")
```

AST 정적 분석으로 26개 f-string SQL 패턴을 검사한 결과 전원 Safe Pattern 확인.

## 7. 민감 정보 보호

### 로그 마스킹

`JsonFormatter._sanitize_record()`가 다음 필드를 자동 마스킹:

```
password, secret, token, api_key, api-key, authorization, jwt, private_key
→ 값을 "***" 로 교체
```

이메일은 부분 마스킹: `user@example.com` → `us**@example.com`

### 환경 변수

```bash
# 운영 환경 필수 설정 (미설정 시 startup 실패)
JWT_SECRET=<strong-random-secret>
POSTGRES_PASSWORD=<strong-password>
INTERNAL_SERVICE_SECRET=<strong-random-secret>
```

개발 환경 기본값(devpassword 등)은 절대 운영에 사용 금지.

### .gitignore

```
.env, .env.local, .env.*.local    # 환경 변수 파일
*.pem, *.key, *.crt, secrets/     # 인증서 및 키
*.dump, *.dump.sha256              # DB 백업
*.log                              # 로그 파일
```

## 8. Rate Limiting

`slowapi` 기반 엔드포인트별 요청 제한.

| 엔드포인트 | 제한 |
|-----------|------|
| 일반 읽기 | 200 req/min |
| 쓰기 | 60 req/min |
| RAG 질의 | 20 req/min |

초과 시 `429 Too Many Requests`, `Retry-After` 헤더 포함.

## 10. CI 보안 스캔

```yaml
- name: Bandit security scan
  run: bandit -r app/ -ll  # HIGH 이상 취약점 감지 시 빌드 실패
```

`-ll` 플래그: LOW 이하 무시, MEDIUM/HIGH만 실패 처리.
