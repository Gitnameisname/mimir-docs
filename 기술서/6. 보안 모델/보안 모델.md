## 1. 인증 보안

### JWT (HS256)

- 알고리즘: **HS256** (HMAC-SHA256)
- 필수 클레임: `sub` (actor_id), `exp` (만료 시각)
- 기본 만료: 120분
- Secret: `JWT_SECRET` 환경 변수 (운영 환경 필수, 미설정 시 startup 실패)
- `algorithms=["HS256"]` 명시적 지정 (알고리즘 혼용 공격 방지)

### API Key

- 원문 저장 안 함 — SHA-256 해시만 DB에 보관
- `key_prefix` (앞 8자리)로 인덱스 조회 → `hmac.compare_digest()`로 타이밍 공격 방지
- 범위(scope): `READ_ONLY` / `READ_WRITE`
- 만료일(expires_at) 설정 가능

### 세션 토큰

- `secrets.token_urlsafe(32)` 생성 (암호학적 난수)
- Valkey 저장: `session:{token}` 키, TTL 24시간

### 내부 서비스 토큰

- HMAC-SHA256 서명: `HMAC(INTERNAL_SERVICE_SECRET, "mimir-internal")`
- `hmac.compare_digest()` 타이밍 공격 방지
- Secret 미설정 시 자동 거부 (fail-closed)

## 2. 인가 (Authorization)

### 역할 기반 접근 제어 (RBAC)

6단계 역할: `VIEWER < AUTHOR < REVIEWER < APPROVER < ORG_ADMIN < SUPER_ADMIN`

모든 API 엔드포인트는 진입 시 역할 확인:
```python
require_authenticated(actor)   # 401 — 미인증
require_admin_access(actor)    # 403 — ORG_ADMIN 미만
```

### 심층 방어 (Defense-in-Depth)

권한 검사가 4개 계층에서 모두 적용된다:
1. **API 계층** — 엔드포인트 진입 시 역할 확인
2. **서비스 계층** — 리소스별 작업 권한 확인
3. **검색 계층** — 쿼리에 역할 기반 필터 자동 추가
4. **벡터/RAG 계층** — `accessible_roles`, `is_public` 컬럼 필터

## 6. SQL 인젝션 방지

psycopg2 파라미터화 쿼리를 일관되게 사용한다.

```python
# ✅ 안전 — 사용자 입력을 파라미터로 전달
cur.execute("SELECT * FROM users WHERE email = %s", (email,))

# ✅ 안전 — Safe Dynamic WHERE 패턴
conditions = []
params = []
if search:
    conditions.append("title ILIKE %s")
    params.append(f"%{search}%")   # 사용자 입력은 파라미터로
where = "WHERE " + " AND ".join(conditions) if conditions else ""
cur.execute(f"SELECT * FROM documents {where}", params)

# ❌ 위험 — 절대 금지
cur.execute(f"SELECT * FROM users WHERE email = '{email}'")
```

AST 정적 분석으로 26개 f-string SQL 패턴을 검사한 결과 전원 Safe Pattern 확인.

## 11. Scope Profile 기반 ACL (S2 Phase 4 + S3 확장)

### 11.1 모델

`scope_profiles` 테이블 — 에이전트 / 사용자에 바인딩되는 ACL 필터 템플릿.

```python
@dataclass
class ScopeProfile:
    id: str
    name: str
    description: Optional[str]
    organization_id: Optional[str]
    scopes: list[ScopeDefinition]      # scope_name → acl_filter (FilterExpression)
    settings: ScopeProfileSettings     # FG 3-2 — viewers 노출 등 운영 옵션
    allowed_tools: list[str]           # FG 4-0 — MCP tool 화이트리스트
```

`scope_definitions` 테이블이 scope_name 별 `acl_filter` (JSONB FilterExpression) 저장. `apply_scope_filter` 가 FilterExpression → SQL WHERE 절로 변환.

### 11.2 권한은 모든 계층에 적용 (S1 ④)

ScopeProfile ACL 이 4 계층에 일관 적용:

1. **API 계층** — RBAC + ScopeProfile resolve
2. **서비스 계층** — `documents_service.get_document(actor=...)` 가 ACL 통과 여부 검증 → ApiNotFoundError (존재 유출 방지)
3. **검색 계층** — `apply_scope_filter` 가 FTS / 벡터 / hybrid 검색 SQL 에 자동 추가
4. **MCP 계층** — `_resolve_acl_filter` post-filter + `_ensure_document_allowed` 게이트 (read 도구) + tool-level allow 검사

### 11.3 settings_json (FG 3-2)

```json
{ "expose_viewers": false }
```

`expose_viewers=False` (default) — Contributors 패널의 "최근 열람자" 섹션 비노출. 보수적 default — 사생활 우선.

### 11.4 allowed_tools — tool-level ACL (FG 4-0)

`scope_profiles.allowed_tools JSONB DEFAULT '[]'::jsonb`:

- **default-deny**: 빈 배열 시 모든 MCP tool 호출 거부
- **per-tool whitelist**: `["search_documents", "fetch_node", ...]` 등록한 도구만 호출 허용
- **검증**: `known_tool_names()` 외 거부 (manifest 정합 강제)
- **default_enabled** (FG 4-5): 신규 ScopeProfile `use_defaults=True` 시 자동 등록 후보 (현재 6개)

진입점 게이트:
```python
def tool_X(request, actor, conn):
    _check_agent_write_blocked(actor)
    _check_tool_allowed(actor, "X", conn=conn)  # ← 본 게이트
    # ...
```

`_check_tool_allowed` 분기:
- `actor.actor_type != AGENT` → 통과 (사람/시스템 비대상)
- AGENT + `scope_profile_id` None → 거부
- AGENT + `allowed_tools` 미포함 → 거부
- DB 조회 실패 → 거부 (fail-closed)

**Audit**: `scope_profile.allowed_tools.changed` event_type — before/after + affected_agents 기록.

**상세**: [MCP 기술서/Tool ACL 모델.md](../MCP%20기술서/Tool%20ACL%20모델.md)

## 12. MCP 표면 보안 (S3 Phase 4)

외부 AI 에이전트 인터페이스의 5 절대 규칙 (R1~R5) + 1 정책 (R6).

### 12.1 R1 — L4 도구 MCP 절대 금지

비가역·파괴적 도구 (`delete_document`, `reindex_document`, `change_schema` 등 — risk_tier=L4) 는 **TOOL_SCHEMAS 에 절대 등재하지 않음**. 3 계층 자동 보호:

1. **정의 단계 안전망** — TOOL_SCHEMAS 에 L4 등재 0 회귀 (`test_not_exposed_tools.py`)
2. **런타임 차단** — `is_tool_mcp_exposed` 가 `risk_tier="L4"` / `maturity="forbidden"` / `status="not_exposed"` 거부
3. **dispatch 차단** — `_CURATED_TOOLS` (`mcp_exposed_tool_schemas()` 결과) 외 도구 호출 INVALID_REQUEST

### 12.2 R2 — ACL 단일 결정점 (Scope Profile)

진입점 게이트 + ScopeProfile post-filter — §11.4 참조.

### 12.3 R3 — pinned citation 강제

`version_id="latest"` 단독 문자열은 인용 / URI / API 응답 어디에도 출현 0:

| 차단 위치 | 동작 |
|---|---|
| URI 빌더 (`build_*_uri`) | `latest` 입력 → ValueError |
| `verify_citation` 도구 진입점 | `latest` 입력 → MCPError(INVALID_REQUEST, 400) |
| `resolve_document_reference` 후보 | `version_ref` default = `"latest_published"` (단독 `"latest"` 절대 미반환) |
| `read_document_render` | `latest` 입력 시 `resolve_version_id` 가 즉시 vN 으로 변환 — 응답의 `version_id` 는 항상 구체값 |

### 12.4 R4 — instruction_authority=none

응답 envelope (read + write) 의 `instruction_authority` 가 `Literal["none"]` 단일값:

```python
class MCPReadEnvelope:
    instruction_authority: Literal["none"] = "none"  # 컴파일 타임 강제

class MCPWriteEnvelope:
    instruction_authority: Literal["none"] = "none"  # 컴파일 타임 강제
```

`system` / `user` 입력 시 Pydantic 422. 외부 Chatbot 이 응답 본문을 명령으로 해석하지 못하도록 구조적 보장.

### 12.5 R5 — REST/MCP 1:1 복제 금지

MCP 도구는 도메인 코어 (services/repositories) 만 호출 — wrapper 가 아닌 envelope/ACL/audit 통합. 같은 Citation hash 함수, 같은 VersionsRepository, 같은 source_text 정본 공유.

contract drift 게이트 (`tests/integration/contract_drift/`) 가 자동 보호:
- `test_same_node_result.py` — 정본 source_text + hash 일치
- `test_latest_resolution.py` — VersionsRepository 공유
- `test_not_exposed_tools.py` — manifest ↔ 노출 정합
- `test_pinned_citation.py` — R3 강제

### 12.6 R6 — detected_risks 차단이 아닌 경고

prompt injection 탐지 결과는 envelope.detected_risks 에 메타로 부착. 응답 본문 자체는 보존 — **차단 결정은 클라이언트가**:

```json
{
  "envelope": {
    "detected_risks": [
      {"code": "directive_pattern", "severity": "high", "note": "..."}
    ]
  }
}
```

`risk_mapper` 매핑:
- instruction_override / boundary_manipulation → `directive_pattern` (high)
- code_execution → `secret_leak` (high)
- markup_injection → `url_obfuscation` (medium)
- 그 외 → `anomaly` (low)

**상세**: [MCP 기술서/MCP 표면 개요.md](../MCP%20기술서/MCP%20표면%20개요.md)

## 13. 도구 등급화 (L0~L4)

CONSTITUTION 제20조 Typed Action Risk 기반.

| 등급 | 정의 | 예시 | 검증 |
|---|---|---|---|
| **L0** | 순수 읽기, 부작용 없음, 가역 | search / fetch / status | TOOL_SCHEMAS 에 6개 |
| **L1** | 검증·해시 등 결정성 검사 | verify_citation, resolve_reference | 2개 |
| **L2** | 가역 쓰기, 사람 승인 후 머지 | save_draft (FG 4-6) | 1개 (experimental) |
| **L3** | 워크플로 전이, 사람 승인 필요 | publish, approve | **MCP 영구 금지** |
| **L4** | 비가역·파괴적 | delete, reindex | **R1 영구 금지** |

L3 / L4 는 REST 관리자 UI 만 — TOOL_SCHEMAS 에 절대 추가하지 않음 (영구 정책).

### 13.1 4 사전 조건 (L2 도구 — `save_draft`, FG 4-6)

L2 write 도구가 노출되기 위한 4 사전 조건:

| 조건 | 구현 |
|---|---|
| **idempotency** | `agent_proposals.idempotency_key` UNIQUE INDEX (Alembic `s3_p4_agent_prop_idempotency`) |
| **human approval** | `agent_proposal_service.propose_draft` (S2 자산) — propose 만, 자동 머지 0. `requires_human_approval=True` 항상 |
| **impact preview** | `compute_draft_impact()` — chars/nodes 차이 사전 계산 (DB 변경 0) |
| **감사 로그 4종** | `agent_proposal.{requested, approved, merged, rolled_back}` |

`save_draft` 의 `default_enabled=False` — 운영자가 ScopeProfile.allowed_tools 에 명시 등록한 경우에만 호출 가능. **MCP 표면 최초 쓰기 도구** (별도 P1 게이트).
