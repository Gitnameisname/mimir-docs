# 플러그인 개발 가이드

새로운 DocumentType을 추가하는 방법을 단계별로 설명한다.

---

## 1. 플러그인 파일 생성

`backend/app/plugins/builtin/` 또는 `backend/app/plugins/custom/`에 새 파일을 작성한다.

```python
# backend/app/plugins/custom/guide.py

from app.plugins.base import (
    DefaultDocumentTypePlugin,
    ChunkingPlugin, ChunkingConfig,
    RAGPlugin, PromptTemplate,
    SearchPlugin, SearchBoostConfig,
    WorkflowPlugin,
)


class GUIDEPlugin(DefaultDocumentTypePlugin):
    """가이드라인 문서 타입 플러그인."""

    type_code = "GUIDE"
    display_name = "가이드라인"
    description = "업무 절차 및 가이드 문서"

    # -----------------------------------------------------------------------
    # 청킹 전략
    # -----------------------------------------------------------------------
    def get_chunking_plugin(self) -> ChunkingPlugin:
        class GUIDEChunkingPlugin(ChunkingPlugin):
            def get_config(self) -> ChunkingConfig:
                return ChunkingConfig(
                    max_tokens=450,
                    min_tokens=50,
                    overlap=40,
                    strategy="section",     # paragraph | section | qa_pair
                    parent_context_depth=2,
                )
        return GUIDEChunkingPlugin()

    # -----------------------------------------------------------------------
    # RAG 설정
    # -----------------------------------------------------------------------
    def get_rag_plugin(self) -> RAGPlugin:
        class GUIDERAGPlugin(RAGPlugin):
            top_n = 5
            max_context_tokens = 6000

            def build_system_prompt(self, context: str) -> str:
                return (
                    "당신은 가이드라인 전문가입니다. "
                    "제공된 가이드라인 문서를 바탕으로 명확하고 실용적인 답변을 제공하세요.\n\n"
                    f"참고 가이드라인:\n{context}"
                )
        return GUIDERAGPlugin()

    # -----------------------------------------------------------------------
    # 검색 설정
    # -----------------------------------------------------------------------
    def get_search_plugin(self) -> SearchPlugin:
        class GUIDESearchPlugin(SearchPlugin):
            def get_boost_config(self) -> SearchBoostConfig:
                return SearchBoostConfig(
                    boost_fields={
                        "target_audience": 3.0,  # metadata 필드 부스트
                        "title": 2.0,
                    }
                )
        return GUIDESearchPlugin()

    # -----------------------------------------------------------------------
    # 워크플로 정책
    # -----------------------------------------------------------------------
    def get_workflow_plugin(self) -> WorkflowPlugin:
        class GUIDEWorkflowPlugin(WorkflowPlugin):
            requires_approval = True
            default_workflow = "standard-approval-flow"
        return GUIDEWorkflowPlugin()
```

## 2. 레지스트리에 등록

### 방법 A: 코드 등록 (권장)

`backend/app/plugins/builtin/__init__.py`에 추가:

```python
from app.plugins.custom.guide import GUIDEPlugin

def register_builtin_plugins() -> None:
    registry = DocumentTypeRegistry.instance()
    registry.register(POLICYPlugin())
    registry.register(MANUALPlugin())
    registry.register(REPORTPlugin())
    registry.register(FAQPlugin())
    registry.register(GUIDEPlugin())   # ← 추가
```

### 방법 B: DB 등록 (Admin UI)

Admin UI에서 새 문서 유형을 생성하면 `document_types` 테이블에 저장된다.  
코드 플러그인이 없으면 `DefaultDocumentTypePlugin`이 사용됨 (기본 동작).

## 4. 플러그인 규격 테스트

`PluginConformanceTest`를 상속하면 11개 규격 테스트를 자동으로 실행한다.

```python
# backend/tests/unit/test_guide_plugin.py

from tests.test_plugins import PluginConformanceTest
from app.plugins.custom.guide import GUIDEPlugin


class TestGUIDEPlugin(PluginConformanceTest):
    plugin_class = GUIDEPlugin

    # 타입별 추가 검증
    def test_chunking_max_tokens_reasonable(self):
        plugin = GUIDEPlugin()
        config = plugin.get_chunking_plugin().get_config()
        assert 100 <= config.max_tokens <= 1000
```

**PluginConformanceTest가 검증하는 항목:**

1. `type_code` 속성 존재 및 비어있지 않음
2. `get_chunking_plugin()` 반환값이 `ChunkingPlugin` 인스턴스
3. `ChunkingConfig.max_tokens` > 0
4. `ChunkingConfig.overlap` < `max_tokens`
5. `get_rag_plugin()` 반환값이 `RAGPlugin` 인스턴스
6. `top_n` > 0
7. `build_system_prompt(context)` 문자열 반환
8. `get_search_plugin()` 반환값이 `SearchPlugin` 인스턴스
9. `get_workflow_plugin()` 반환값이 `WorkflowPlugin` 인스턴스
10. `requires_approval` 불리언 타입
11. `default_workflow` 문자열 타입

## 6. Admin UI에서 설정 오버라이드

코드로 등록된 플러그인의 기본값을 Admin UI에서 조정할 수 있다.

```
Admin UI → 문서 유형 관리 → GUIDE → 플러그인 설정 탭
```

6개 탭에서 설정 가능:
1. **기본 정보** — display_name, description, status
2. **Metadata Schema** — schema_fields JSON 편집
3. **청킹 설정** — max_tokens, min_tokens, overlap
4. **RAG 설정** — top_n, max_context_tokens, system_prompt
5. **검색 설정** — boost_fields
6. **워크플로** — requires_approval, default_workflow

API로 직접 설정:
```
PUT /api/v1/admin/document-types/GUIDE/plugin
{
  "chunking": {"max_tokens": 500, "overlap": 60},
  "rag": {"top_n": 7}
}
```

## 7. 기본 동작 (DefaultDocumentTypePlugin)

코드 플러그인 없이 DB에만 등록된 타입은 기본값을 사용한다.

| 항목 | 기본값 |
|------|--------|
| 청킹 전략 | `paragraph`, max_tokens=400, overlap=50 |
| RAG top_n | 5 |
| RAG 컨텍스트 | 6000 tokens |
| 검색 부스트 | 없음 (기본 가중치) |
| 승인 필요 | True |
| 워크플로 | `standard-approval-flow` |
