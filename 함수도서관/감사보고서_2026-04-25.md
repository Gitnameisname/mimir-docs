## 1. 방법

1. 서브에이전트 2개 (백엔드 / 프론트) 를 병렬 기동해 seed 패턴 grep → 후보 식별.
2. 각 후보에 대해 본 세션에서 `Grep` 으로 **직접 스팟 체크** 재조회 → 서브에이전트 추정치 보정.
3. 최종 후보에 대해 배치 위치·시그니처·주의점을 `backend.md` / `frontend.md` 에 기록.

## 2. 규칙·헌법 체크리스트

- ☑ CONSTITUTION 제8조 Single Responsibility — 후보 유틸은 모두 단일 책임.
- ☑ CONSTITUTION 제11조 Function Index — 본 도서관이 제11조의 프로젝트 구체화 (`functions.index.md` 개념).
- ☑ CONSTITUTION 제15조 Bounded Change — 이번 턴은 **감사 + 문서만**. 코드 변경 없음.
- ☑ CONSTITUTION 제32조 Reviewable PRs — 후속 구현은 1 유틸 = 1 PR 분할 전략.
- ☑ CLAUDE.md §S2 ⑤ 스코프 하드코딩 금지 — 제안 유틸 중 DB·ACL 계열은 scope 리터럴을 만들지 않음.
- ☑ CLAUDE.md §S2 ⑥ actor_type 기록 — `audit_helpers.emit_audit` 가 이 원칙을 런타임에서 강제한다.
- ☑ CLAUDE.md §S2 ⑦ 폐쇄망 degrade — 외부 I/O 유틸은 off 상태에서 실패 대신 로컬 로그로 강등한다.

### 3.2 보류·반려

- **리포지토리 CRUD 베이스 클래스** — ACL 슬롯이 도메인별로 달라 일반화 시 누락 위험. `fetch_*_as` + `clamp_pagination` + `uuid_str_or_none` 조합으로 점진 개선에 머무름.
- **re.compile 상수 모음** — 의미가 도메인별로 다름. 도메인 경계 넘는 재사용만 별도 유틸로 추출.
- **session.commit()/flush() 통합** — 프로젝트는 psycopg 직연결·수동 트랜잭션 패턴이라 공통 헬퍼 의의가 낮음.

### 3.3 숫자 보정 메모

- 서브에이전트가 추정한 `uuid_str_or_none = 201회` 는 `str(...)` 일반 카운트를 포함해 과대.
- 스팟 체크 정규식 `str([A-Za-z_]*[Ii]d) if` = 13회가 보다 정확한 경계.
- `actor_type` 참조 302 는 import·모델 정의·테스트를 포함하므로 실제 함수화 대상과는 다르다.
