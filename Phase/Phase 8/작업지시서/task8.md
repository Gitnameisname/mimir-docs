# Task 8-8. 검색 결과 UI 설계 및 구현 (User UI 통합)

## 1. 작업 목적

Phase 6에서 구축한 User UI에 검색 기능을 통합하여 사용자가 문서를 빠르게 탐색할 수 있는 검색 경험을 제공한다.

이 작업의 목표는 다음과 같다.

- 글로벌 검색 바를 User UI 헤더에 통합
- 검색 결과 페이지 설계 및 구현
- 스니펫 및 키워드 하이라이팅 표시
- 노드 단위 검색 결과에서 문서 내 위치로 직접 이동
- 검색 필터 패널 구현

## 4. 주요 설계 대상

### 4-1. 글로벌 검색 바

#### 위치 및 동작

- User UI 상단 헤더에 항상 표시
- 검색 바 클릭 또는 단축키(Ctrl+K / Cmd+K)로 활성화
- 활성화 시 검색 입력창 포커스 및 오버레이 표시 (선택)

#### 검색 바 UX 흐름

1. 검색어 입력 (디바운스 적용, 300ms)
2. Enter 또는 검색 버튼 클릭 시 검색 결과 페이지 이동
3. URL: `/search?q={검색어}&type=...&status=...`

#### 검색 바 구성 요소

- 검색 아이콘
- 텍스트 입력 필드 (placeholder: "문서 제목, 내용, 키워드 검색...")
- 검색어 지우기 버튼 (X)
- 검색 버튼

---

### 4-2. 검색 결과 페이지 (`/search`)

#### 페이지 레이아웃

```
┌──────────────────────────────────────────────────┐
│ 검색 바 (검색어 수정 가능)                          │
├────────────────┬─────────────────────────────────┤
│ 필터 패널      │ 검색 결과 목록                     │
│ (좌측)        │ 총 N개 결과  정렬 선택              │
│               │ ─────────────────────────────── │
│ DocumentType  │ [결과 카드 1]                     │
│ 상태           │ [결과 카드 2]                     │
│ 날짜 범위      │ ...                               │
│               │ [페이지네이션]                     │
└────────────────┴─────────────────────────────────┘
```

#### URL 파라미터 동기화

- 검색어, 필터, 정렬, 페이지 정보를 URL 쿼리 파라미터로 관리
- 뒤로 가기, 공유, 새로고침 시 검색 상태 복원

---

### 4-3. 검색 결과 카드 - 문서 단위

```
┌─────────────────────────────────────────────────┐
│ [POLICY 배지] 정보보안 기본 정책          점수 ••• │
│ 보안팀 · PUBLISHED · 2025-03-01                  │
│                                                  │
│ ...본 정책은 <mark>보안</mark>을 위한 기본         │
│ <mark>정책</mark>을 정의하며, 모든 임직원에게...    │
│                                                  │
│ 이 문서에서 3개 섹션 매칭 [섹션 펼쳐보기 ▼]       │
└─────────────────────────────────────────────────┘
```

#### 카드 구성 요소

- DocumentType 배지
- 문서 제목 (클릭 시 문서 상세 이동)
- 부가 정보 (소속 조직, 상태, 날짜)
- 스니펫 (키워드 하이라이팅 포함)
- 매칭된 노드 수 + 섹션 펼쳐보기 토글

---

### 4-4. 노드 매칭 섹션 펼쳐보기

문서 카드 하단에 매칭된 노드 목록을 펼쳐서 표시:

```
  ▼ 3개 섹션에서 발견
  ├ 2.3 접근통제 정책
  │   ...시스템 <mark>접근</mark>은 역할 기반으로...
  │   [이 섹션으로 이동 →]
  ├ 3.1 보안 감사
  │   ...<mark>보안</mark> 감사는 분기별로...
  │   [이 섹션으로 이동 →]
  └ 4.2 위반 처리
      ...위반 사항에 대한 <mark>정책</mark>은...
      [이 섹션으로 이동 →]
```

#### "이 섹션으로 이동" 동작

- 문서 상세 페이지로 이동하며 해당 노드 위치로 스크롤/앵커 이동
- URL: `/documents/{document_id}#node-{node_id}`

---

### 4-5. 노드 단위 검색 결과 카드 (`/search?mode=nodes`)

노드 단위 검색 결과를 개별 카드로 표시하는 뷰:

```
┌─────────────────────────────────────────────────┐
│ [섹션] 2.3 접근통제 정책                          │
│ 경로: 정보보안 기본 정책 > 2. 정책 본문 > 2.3...   │
│                                                  │
│ ...시스템 <mark>접근통제</mark>는 역할 기반으로...  │
│                                              [→] │
└─────────────────────────────────────────────────┘
```

---

### 4-6. 검색 필터 패널

#### 필터 항목

**DocumentType 필터**
- 체크박스 목록 (타입명 + 해당 결과 수)
- 예: ☑ 정책 (12), ☑ 매뉴얼 (8), □ 보고서 (3)

**문서 상태 필터**
- PUBLISHED (기본 선택)
- DRAFT, IN_REVIEW 등 (권한에 따라 노출)

**날짜 범위 필터**
- 생성일 기준
- 오늘 / 최근 7일 / 최근 30일 / 직접 입력

#### 필터 UX

- 필터 변경 시 즉시 검색 결과 갱신
- 활성 필터 상단에 태그 형태로 표시 (개별 제거 가능)
- "필터 초기화" 버튼

---

### 4-7. 검색 결과 정렬 옵션

- 관련도 순 (기본)
- 최신순 (수정일 기준)
- 오래된 순
- 제목 가나다 순

---

### 4-8. 상태 처리

#### 로딩 상태

- 검색 결과 Skeleton 카드 표시

#### 결과 없음 상태

- "'{검색어}'에 대한 검색 결과가 없습니다" 메시지
- 검색어 수정 안내
- 필터 초기화 버튼

#### 오류 상태

- "검색 중 오류가 발생했습니다" 메시지
- 재시도 버튼

---

### 4-9. 문서 목록 화면에서의 인라인 검색

기존 `/documents` 목록 화면에서도 검색/필터 기능 제공:

- 검색 바는 문서 목록 상단에 배치
- 검색 시 URL 파라미터로 상태 유지
- 전체 검색 결과가 필요한 경우 "전체 검색으로 보기" 링크

## 7. Codex 작업 지침

- Phase 6 User UI의 공통 레이아웃 및 컴포넌트 패턴을 최대한 재사용한다
- 스니펫의 `<mark>` 태그를 렌더링할 때 XSS를 방지하기 위해 HTML 이스케이프 후 안전한 렌더링 방식을 사용한다 (dangerouslySetInnerHTML 대신 DOMPurify 등 활용 고려)

- UI 리뷰를 최소 5회 이상 수행하여 검색 UX 품질을 확인한다 (CLAUDE.md 규칙 준수)
