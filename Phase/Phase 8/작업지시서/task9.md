# Task 8-9. Admin 검색 인덱스 관리 기능 확장

## 1. 작업 목적

Phase 7 Admin UI에 검색 인덱스 관리 기능을 추가하여, 운영자가 검색 인덱스 상태를 파악하고 수동 재인덱싱을 실행할 수 있도록 한다.

이 작업의 목표는 다음과 같다.

- 검색 인덱스 현황 조회 화면 설계 및 구현
- 수동 재인덱싱 트리거 UI 제공
- 인덱스 오류/누락 문서 확인 기능
- Phase 7 대시보드에 검색 인덱스 상태 지표 추가

---

## 2. 작업 범위

### 포함 범위

- Admin 검색 인덱스 관리 화면 설계
- 인덱스 현황 조회 API 설계 및 구현
- 수동 재인덱싱 API 설계 및 구현
- Phase 7 대시보드 카드 확장

### 제외 범위

- 검색 인덱스 자동 갱신 로직 (Task 8-5, 8-6에서 다룸)
- 검색 결과 품질 분석 (Task 8-10에서 다룸)

## 3. 선행 조건

- Task 8-5 (전문 검색 구현) 완료
- Phase 7: Admin UI 공통 레이아웃 및 패턴 이해

## 4. 주요 설계 대상

### 4-1. Admin 검색 인덱스 관리 화면 (`/admin/search-index`)

Phase 7 Admin 메뉴에 "Search Index" 항목 추가.

#### 인덱스 현황 요약 섹션

상단에 요약 지표 카드:

| 지표 | 설명 |
|------|------|
| 전체 인덱싱 대상 문서 수 | search_vector 대상 문서 총수 |
| 인덱싱 완료 문서 수 | search_vector NOT NULL인 문서 수 |
| 인덱싱 누락 문서 수 | search_vector NULL인 문서 수 |
| 마지막 전체 인덱싱 일시 | 배치 실행 완료 시각 |

#### 인덱스 대상 목록

문서별 인덱싱 상태를 테이블로 표시:

| 컬럼 | 설명 |
|------|------|
| 문서명 | 문서 제목 |
| DocumentType | 타입 |
| 인덱싱 상태 | INDEXED / NOT_INDEXED / EXCLUDED |
| 마지막 인덱싱 일시 | search_vector 마지막 갱신 시각 |
| 인덱스 크기 | search_vector 길이 (토큰 수) |

#### 필터

- 인덱싱 상태 필터 (INDEXED / NOT_INDEXED / EXCLUDED)
- DocumentType 필터

---

### 4-2. 수동 재인덱싱 기능

#### 단건 재인덱싱

- 특정 문서 선택 후 "재인덱싱" 버튼
- 확인 다이얼로그 후 실행
- 실행 결과 Toast 표시

#### 누락 문서 일괄 재인덱싱

- "누락 문서 일괄 재인덱싱" 버튼
- 대상 건수 확인 다이얼로그
- 백그라운드 배치 작업으로 실행
- Phase 7 백그라운드 작업 목록에서 진행 상황 확인 가능

#### 전체 재인덱싱

- "전체 재인덱싱" 버튼 (SUPER_ADMIN 전용)
- 강화된 경고 다이얼로그 (서비스 영향 안내)
- 백그라운드 배치 작업으로 실행

---

### 4-3. 재인덱싱 API

#### 엔드포인트

| 메서드 | 경로 | 설명 | 권한 |
|--------|------|------|------|
| GET | `/admin/search-index/status` | 인덱스 현황 조회 | ADMIN |
| GET | `/admin/search-index/documents` | 문서별 인덱싱 상태 목록 | ADMIN |
| POST | `/admin/search-index/reindex/{document_id}` | 단건 재인덱싱 | ADMIN |
| POST | `/admin/search-index/reindex-missing` | 누락 문서 일괄 재인덱싱 | ADMIN |
| POST | `/admin/search-index/reindex-all` | 전체 재인덱싱 | SUPER_ADMIN |

---

### 4-4. Phase 7 대시보드 카드 확장

Phase 7의 시스템 상태 대시보드에 검색 인덱스 지표 카드 추가:

```
┌──────────────────────────────┐
│ 검색 인덱스 상태              │
│ 인덱싱 완료: 1,234건          │
│ 누락: 5건  ← 0 초과 시 강조   │
│ [관리 →]                     │
└──────────────────────────────┘
```

- 누락 건수가 0보다 많으면 경고 색상으로 표시
- 카드 클릭 시 `/admin/search-index` 이동

- Admin 메뉴에 검색 인덱스 관리 항목이 추가되어 있다
- 인덱스 현황(총수, 완료, 누락)이 조회된다
- 수동 재인덱싱(단건/누락/전체)이 동작한다
- 재인덱싱 작업이 백그라운드 작업 목록에서 추적 가능하다
- 대시보드에 검색 인덱스 상태 카드가 추가되어 있다

## 7. Codex 작업 지침

- Phase 7 Admin UI의 공통 레이아웃 및 패턴을 사용한다
