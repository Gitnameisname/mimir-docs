# Task 7-9. API 키 및 외부 연계 관리 화면 설계

## 1. 작업 목적

외부 시스템 연계를 위한 API 키 및 외부 연계 설정을 운영자가 안전하게 관리할 수 있는 화면을 설계한다.

이 작업의 핵심 목표는 다음과 같다.

- API 키 발급/조회/폐기 흐름을 정의하고 키 노출을 최소화하는 UX 설계
- 외부 시스템 연계 설정을 운영자가 조회하고 관리할 수 있도록 한다
- 향후 AI 도구 및 외부 앱 연계 정책 반영 가능 구조 확보

## 4. 주요 설계 대상

### 4-1. API 키 목록 화면 (`/admin/api-keys`)

#### 목록 테이블 컬럼

- 키 이름/별칭
- 키 앞 8자리 (마스킹, 예: `mimir_ab12****`)
- 발급 주체 (사용자/시스템)
- 사용 목적/설명
- 상태 (ACTIVE / REVOKED / EXPIRED)
- 만료일
- 마지막 사용 일시
- 발급일

#### 기능

- 키워드 검색 (키 이름, 발급 주체)
- 상태 필터
- 만료 임박 키 강조 (7일 이내)
- 행 클릭 시 상세 패널 진입

---

### 4-2. API 키 상세 패널

#### 표시 정보

- 키 기본 정보 (이름, 목적, 발급자)
- 상태 및 만료일
- 권한 범위 (Scope)
- 사용 통계 (최근 사용 횟수, 마지막 사용 IP)
- 발급 이력 (재발급 시 이전 키 정보)

#### 보안 규칙

- 키 전체 값은 **발급 시 1회만 표시**하고 이후 마스킹
- 마스킹된 상태에서 복사 버튼 제공하지 않음
- "키 값 분실 시 재발급 안내" 메시지 상시 표시

---

### 4-3. API 키 발급 흐름

신규 API 키 발급 UX:

1. "새 API 키 발급" 버튼 클릭
2. 발급 폼 표시:
   - 키 이름/별칭 (필수)
   - 사용 목적 설명
   - 권한 범위 선택 (Scope: READ_ONLY / FULL_ACCESS / CUSTOM)
   - 만료일 설정 (선택, 기본값 있음)
3. 발급 확인 다이얼로그
4. 발급 완료 후 **키 값 전체 표시 화면** (1회성):
   - 경고 메시지: "이 키는 지금만 표시됩니다. 반드시 안전한 곳에 저장하세요."
   - 키 값 복사 버튼
   - 확인 체크박스 후 닫기 버튼
5. 감사 이벤트 자동 기록

---

### 4-4. API 키 폐기 흐름

키 폐기 UX:

1. 키 목록 또는 상세에서 폐기 버튼 클릭
2. 위험 확인 다이얼로그:
   - 폐기 대상 키 정보 표시
   - 현재 이 키를 사용하는 연계 시스템 경고 (알 수 있는 경우)
   - 폐기 사유 입력 (필수)
   - Danger 스타일 확인 버튼
3. 폐기 완료 후 키 상태 REVOKED 전환
4. 감사 이벤트 자동 기록

---

### 4-5. 외부 연계 설정 화면 (`/admin/api-keys/integrations`)

외부 시스템 연계 설정 목록:

#### 목록 테이블 컬럼

- 연계 시스템명
- 연계 유형 (WEBHOOK / OAUTH / API)
- 상태 (ACTIVE / INACTIVE)
- 마지막 동기화 일시
- 담당자

#### 연계 상세 패널

- 연계 기본 정보
- 인증 방식 및 마스킹된 설정값
- 연계 테스트 버튼 (ping/health check)
- 활성/비활성 전환

---

### 4-6. 보안 경고 및 마스킹 규칙

| 상황 | 처리 방식 |
|------|----------|
| 키 전체 값 표시 | 발급 시 1회만, 이후 마스킹 |
| 만료 임박 (7일 이내) | 노란 배지 + 경고 안내 |
| 이미 만료된 키 | 회색 배지 + 재발급 안내 |
| 장기간 미사용 키 | 경고 표시 (기준: 90일) |
| 연계 인증 설정값 | 항상 마스킹 표시 |

## 7. Codex 작업 지침

- 코드 작성 금지 (설계 중심)
- API 키 보안은 최우선 고려 사항이다 — 키 노출 최소화 원칙을 설계의 모든 단계에서 반영한다
- 발급 시 1회 표시 원칙은 타협 없이 반영한다
- 폐기 시 영향 범위를 운영자에게 명확히 전달하는 UX를 설계한다
- 감사 이벤트 자동 기록 포인트를 설계에서 명시한다
