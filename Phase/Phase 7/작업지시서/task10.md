# Task 7-10. 백그라운드 작업 모니터링 화면 설계

## 1. 작업 목적

플랫폼의 비동기 작업과 장기 실행 작업 상태를 운영자가 추적하고 관리할 수 있는 화면을 설계한다.

이 작업의 핵심 목표는 다음과 같다.

- 작업 종류/상태/대상 리소스를 한눈에 파악할 수 있는 목록 화면 설계
- 실패 작업에 대한 원인 분석과 재시도/취소 액션 UX 설계
- 장기 실행 작업의 진행률 모니터링 구조 정의

---

## 4. 주요 설계 대상

### 4-1. 백그라운드 작업 목록 화면 (`/admin/jobs`)

#### 목록 테이블 컬럼

- 작업 ID
- 작업 타입 (DOCUMENT_EXPORT, WORKFLOW_TRANSITION, INDEX_BUILD 등)
- 대상 리소스 (리소스 타입 + ID + 표시명)
- 상태 (PENDING / RUNNING / COMPLETED / FAILED / CANCELLED)
- 진행률 (진행 중인 경우 %)
- 요청자
- 시작 시각
- 완료/실패 시각
- 소요 시간

#### 기능

- 상태별 필터 (다중 선택)
- 작업 타입 필터
- 요청자 검색
- 기간 필터
- 행 클릭 시 상세 패널 진입
- 실패 건수 상단 배너 표시

---

### 4-2. 작업 상태 전이 표시

각 작업의 상태 전이 흐름을 시각적으로 표현:

```
PENDING → RUNNING → COMPLETED
                 ↘ FAILED
RUNNING → CANCELLED (수동 취소)
FAILED → PENDING (재시도 시)
```

상태 배지:

| 상태 | 색상 | 설명 |
|------|------|------|
| PENDING | 회색 | 큐에 대기 중 |
| RUNNING | 파란색 (애니메이션) | 실행 중 |
| COMPLETED | 녹색 | 정상 완료 |
| FAILED | 빨간색 | 실패 |
| CANCELLED | 회색 (취소선) | 취소됨 |

---

### 4-3. 작업 상세 패널

목록에서 작업 선택 시 우측 패널로 표시:

#### 표시 정보

**기본 정보**

- 작업 ID, 타입, 요청자
- 상태 및 처리 시각
- 대상 리소스 링크

**진행 현황 (RUNNING 상태)**

- 진행률 Progress Bar
- 현재 처리 단계 표시
- 예상 남은 시간 (가능 시)

**오류 정보 (FAILED 상태)**

- 오류 코드 및 메시지
- 발생 위치 (단계)
- 스택트레이스 Expand/Collapse
- 재시도 이력

**완료 정보 (COMPLETED 상태)**

- 처리 결과 요약
- 소요 시간

#### 주요 액션

- 재시도 (FAILED 상태)
- 취소 (PENDING / RUNNING 상태)
- 강제 종료 (RUNNING 상태, 위험 액션)

---

### 4-4. 실패 작업 분석 UX

#### 오류 유형 분류

| 오류 유형 | 설명 | 권장 조치 |
|-----------|------|-----------|
| RESOURCE_NOT_FOUND | 대상 리소스가 삭제됨 | 작업 정리 |
| PERMISSION_DENIED | 권한 부족 | 권한 확인 후 재시도 |
| EXTERNAL_SERVICE_ERROR | 외부 연계 오류 | 연계 상태 확인 후 재시도 |
| TIMEOUT | 처리 시간 초과 | 리소스 크기 확인 후 재시도 |
| SYSTEM_ERROR | 시스템 내부 오류 | 로그 확인 필요 |

#### 실패 작업 분석 화면

FAILED 상태의 작업 상세에서:

1. 오류 요약을 사람이 읽기 쉬운 형태로 표시
2. 권장 조치 안내 표시
3. 관련 시스템 로그 링크 (가능 시)
4. 재시도 버튼

---

### 4-5. 재시도 / 취소 / 강제 종료 액션 설계

#### 재시도

- FAILED 작업에 대해 재시도 버튼 제공
- 확인 다이얼로그 (선택): 동일 파라미터로 재시도 확인
- 재시도 시 작업 상태 PENDING으로 전환
- 원래 실패 작업과 재시도 작업 연관 표시

#### 취소

- PENDING/RUNNING 작업에 대해 취소 버튼 제공
- RUNNING 작업 취소 시 경고 메시지 표시 (현재 처리 중 상태 안내)
- 확인 다이얼로그 후 취소 처리

#### 강제 종료

- 응답 없는 RUNNING 작업에 대한 강제 종료
- 위험 액션으로 분류: 강화된 확인 다이얼로그
- 사유 입력 필수
- 감사 이벤트 기록

---

### 4-6. 장기 실행 작업 모니터링

처리 시간이 긴 작업 (임계값 초과 시)에 대한 특별 표시:

- 경고 아이콘 표시 (예: 30분 이상 RUNNING)
- 예상 종료 시간 대비 지연 여부 표시
- 운영자 알림 포인트 (이메일/슬랙 연계 Reserved)

---

### 4-7. 작업 현황 요약 섹션

목록 상단에 현재 작업 현황 요약 표시:

- 전체 작업 수
- 상태별 건수
- 실패율 (최근 24시간)
- 현재 RUNNING 작업 수 및 장기 실행 경고 수

- 재시도/취소/강제 종료 액션 흐름이 기술되어 있다

## 7. Codex 작업 지침

- 코드 작성 금지 (설계 중심)
- 실패 작업에 대한 원인 파악이 용이하도록 오류 정보를 최대한 명확하게 표시하는 UX를 설계한다
- 재시도/취소/강제 종료는 의도치 않은 실행을 방지하는 확인 절차를 반드시 포함한다
- 강제 종료는 위험 액션으로 분류하여 별도 강화 확인 절차를 적용한다
- 감사 이벤트 연계 포인트를 설계에서 명시한다
