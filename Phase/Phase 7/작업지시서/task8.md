# Task 7-8. 벡터화 / 인덱싱 상태 관리 화면 설계

## 1. 작업 목적

문서 구조화 및 인덱싱 파이프라인의 상태를 운영자가 파악하고 관리할 수 있는 화면을 설계한다.

이 작업의 핵심 목표는 다음과 같다.

- 인덱싱 작업의 대기/진행/완료/실패 상태를 운영자가 조회할 수 있도록 한다
- 실패 원인을 파악하고 재처리 액션을 취할 수 있는 UX를 설계한다
- Phase 10 (벡터 검색) 운영 화면과의 연계 포인트를 구조에 반영한다

---

## 2. 작업 범위

### 포함 범위

## 4. 주요 설계 대상

### 4-1. 인덱싱 상태 목록 화면 (`/admin/indexing`)

#### 목록 테이블 컬럼

- 작업 ID
- 문서명 / 문서 ID
- DocumentType
- 버전 정보
- 인덱싱 타입 (FULL / INCREMENTAL / REINDEX)
- 상태 (PENDING / RUNNING / COMPLETED / FAILED / SKIPPED)
- 시작 시각
- 완료 시각
- 소요 시간
- 재시도 횟수

#### 기능

- 상태별 필터 (다중 선택)
- DocumentType 필터
- 기간 필터
- 키워드 검색 (문서명)
- 행 클릭 시 상세 패널 또는 상세 화면 진입

---

### 4-2. 상태별 표시 방식

| 상태 | 배지 색상 | 설명 |
|------|----------|------|
| PENDING | 회색 | 처리 대기 중 |
| RUNNING | 파란색 (애니메이션) | 현재 처리 중 |
| COMPLETED | 녹색 | 정상 완료 |
| FAILED | 빨간색 | 실패 (사유 표시) |
| SKIPPED | 노란색 | 건너뜀 (권한/조건 미충족) |

실패 건수가 1개 이상일 경우 상단에 요약 배너로 표시한다.

---

### 4-3. 인덱싱 작업 상세 패널

목록에서 항목 선택 시 우측 패널로 표시:

#### 표시 정보

**기본 정보**

- 작업 ID
- 대상 문서 정보 (ID, 이름, 타입, 버전)
- 인덱싱 타입
- 상태 및 처리 시각

**실패 정보 (FAILED 상태일 경우)**

- 오류 코드
- 오류 메시지 (전체 스택트레이스는 Expand/Collapse)
- 재시도 이력 (횟수, 각 시도의 오류 요약)

**인덱싱 결과 (COMPLETED 상태일 경우)**

- 처리된 청크 수
- 벡터화된 청크 수
- 권한 반영 여부
- 버전 기준 인덱싱 여부

#### 주요 액션

- 재시도 (FAILED 상태)
- 재색인 강제 실행 (COMPLETED 상태)
- 건너뜀 사유 확인 (SKIPPED 상태)

---

### 4-4. 실패 사유 및 재처리 UX

#### 실패 분류

| 오류 유형 | 설명 | 권장 조치 |
|-----------|------|-----------|
| DOCUMENT_NOT_FOUND | 문서가 삭제됨 | 인덱싱 항목 제거 |
| SCHEMA_PARSE_ERROR | 문서 구조 파싱 실패 | 문서 확인 후 재시도 |
| VECTOR_STORE_ERROR | 벡터 저장소 오류 | 시스템 상태 확인 후 재시도 |
| PERMISSION_ERROR | 권한 정보 처리 실패 | 권한 설정 확인 |
| TIMEOUT | 처리 시간 초과 | 재시도 |

#### 재시도 흐름

1. FAILED 작업 선택
2. 재시도 버튼 클릭
3. 영향 정보 확인 다이얼로그 (선택)
4. 재시도 실행 → 상태가 PENDING으로 전환
5. 결과 Toast 및 목록 갱신

#### 일괄 재시도

- 필터 기준으로 FAILED 건수 선택 후 일괄 재시도
- 최대 N건 일괄 재시도 제한 안내

---

### 4-5. 인덱싱 현황 요약 섹션

목록 상단에 요약 지표 표시:

- 전체 작업 수
- 상태별 건수 (PENDING / RUNNING / COMPLETED / FAILED)
- 최근 24시간 실패율

---

### 4-6. Phase 10 연계 포인트

벡터 검색(Phase 10) 운영 화면과의 연계를 위한 구조 예약:

- 인덱싱 완료 항목에서 "벡터 검색 결과 확인" 링크 Reserved
- 벡터 저장소 상태 표시 섹션 Reserved (현재는 미구현 상태로 표시)
- 청크/임베딩 상세 정보 조회 포인트 Reserved

## 7. Codex 작업 지침

- 코드 작성 금지 (설계 중심)
