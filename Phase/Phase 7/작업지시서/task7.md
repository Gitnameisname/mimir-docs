# Task 7-7. 시스템 상태 대시보드 설계

## 1. 작업 목적

운영자가 플랫폼 전체 상태를 빠르게 파악하고 이상 징후를 즉시 인지할 수 있는 시스템 상태 대시보드를 설계한다.

이 작업의 핵심 목표는 다음과 같다.

- 운영자가 Admin UI 진입 시 가장 먼저 보는 화면으로, 핵심 운영 지표를 한눈에 제공
- 정상 상태보다 이상 상태를 더 쉽게 발견할 수 있는 구조
- 각 요약 지표에서 상세 관리 화면으로 빠르게 이동할 수 있는 흐름 제공

---

## 2. 작업 범위

### 포함 범위

- 핵심 운영 지표 카드 설계
- 시스템 컴포넌트 상태 표시 구조 설계
- 최근 오류/경고/이상 상태 요약 설계
- 최근 감사 이벤트 요약 설계
- 대시보드에서 각 상세 화면으로의 이동 흐름 정의

### 제외 범위

- 실시간 모니터링 구현 (WebSocket 등)
- 고급 통계 시각화 (차트 라이브러리 상세 구현)
- 외부 모니터링 시스템 연동

## 4. 주요 설계 대상

### 4-1. 핵심 지표 카드 (`/admin/dashboard`)

상단에 주요 운영 지표를 카드 형태로 배치한다.

#### 필수 카드 항목

| 카드 | 표시 내용 | 이상 조건 | 이동 대상 |
|------|----------|-----------|-----------|
| 총 사용자 수 | 전체 / 활성 수 | - | `/admin/users` |
| 활성 문서 수 | 전체 문서 수 | - | - |
| 진행 중 작업 수 | PENDING/RUNNING 건수 | 임계값 초과 시 경고 | `/admin/jobs` |
| 실패 작업 수 | FAILED 건수 | 0 초과 시 강조 | `/admin/jobs?status=FAILED` |
| 최근 감사 이벤트 수 | 24시간 내 이벤트 수 | CRITICAL 이벤트 존재 시 강조 | `/admin/audit-logs` |
| 인덱싱 실패 수 | FAILED 인덱싱 건수 | 0 초과 시 강조 | `/admin/indexing?status=FAILED` |

#### 카드 디자인 원칙

- 정상: 기본 스타일
- 경고: 노란색 테두리 + 경고 아이콘
- 오류/이상: 빨간색 테두리 + 오류 아이콘
- 카드 클릭 시 해당 상세 화면으로 이동

---

### 4-2. 시스템 컴포넌트 상태 섹션

각 시스템 컴포넌트의 현재 상태를 표시한다.

#### 컴포넌트 목록

| 컴포넌트 | 상태 표시 기준 |
|----------|--------------|
| API Server | 응답 여부 |
| Database | 연결 상태 |
| Background Job Queue | 큐 상태 (정상/지연/중단) |
| Indexing Pipeline | 파이프라인 상태 |
| (Phase 8+) Vector Store | 연결 상태 |

#### 상태 표시 방식

- 녹색 점: 정상 (HEALTHY)
- 노란 점: 경고 (DEGRADED)
- 빨간 점: 오류 (DOWN)
- 회색 점: 미확인 (UNKNOWN)

---

### 4-3. 최근 오류/경고 요약 섹션

최근 발생한 오류 및 경고를 목록 형태로 표시:

#### 항목 구성

- 발생 시각
- 오류 유형 (JOB_FAILED, INDEXING_ERROR 등)
- 대상 리소스
- 상태 배지

#### 기능

- 최근 N건 표시 (기본 10건)
- 각 항목 클릭 시 관련 상세 화면으로 이동
- "전체 보기" 링크 제공

---

### 4-4. 최근 감사 이벤트 요약 섹션

최근 발생한 감사 이벤트 요약 (특히 중요도 높은 이벤트 우선):

#### 항목 구성

- 발생 시각
- 중요도 배지
- 이벤트 타입
- 행위자
- 결과

#### 기능

- 최근 N건 표시 (기본 10건)
- "전체 보기" → `/admin/audit-logs` 이동

---

### 4-5. 대시보드 레이아웃 구조

```
┌─────────────────────────────────────────────────────┐
│ 핵심 지표 카드 (가로 배열, 6개)                       │
├──────────────────────────┬──────────────────────────┤
│ 시스템 컴포넌트 상태      │ 최근 오류/경고 요약        │
├──────────────────────────┴──────────────────────────┤
│ 최근 감사 이벤트 요약                                 │
└─────────────────────────────────────────────────────┘
```

#### 요구사항

- 페이지 진입 시 데이터 자동 로드
- 수동 새로고침 버튼 제공
- 자동 갱신 주기 옵션 (선택: 30초/1분/5분)

---

### 4-6. 이상 상태 우선 가시화 원칙

- 대시보드는 모든 것이 정상일 때가 아니라, **이상이 있을 때 운영자가 즉시 인지**할 수 있어야 한다
- 이상 카드 / 이상 컴포넌트 / 오류 이벤트는 페이지 내에서 시각적으로 우선 강조
- 아무 이상도 없을 때는 "전체 정상" 상태 메시지 표시

## 7. Codex 작업 지침

- 코드 작성 금지 (설계 중심)
- 운영자의 첫 화면이므로 가장 중요한 정보가 가장 먼저 보여야 한다
- 이상 상태 강조를 시각 설계의 핵심 원칙으로 삼는다
- 각 지표 카드에서 상세 화면으로의 이동 흐름을 명확히 정의한다
- 향후 검색/벡터/RAG 관련 지표 카드 추가가 가능한 확장 구조로 설계한다
