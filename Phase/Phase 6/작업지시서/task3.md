

# Task 6-3. 문서 목록 및 검색 UI 설계

## 1. 작업 목적

사용자가 문서를 빠르게 탐색하고 접근할 수 있도록 **문서 목록 및 검색 UI를 설계**한다.

이 작업의 핵심 목표는 다음과 같다.

- 대량의 문서 중 원하는 문서를 빠르게 찾을 수 있도록 한다
- 문서 상태, 유형, 작성자 등 주요 정보를 한눈에 파악 가능하게 한다
- 이후 상세 보기(Task 6-4)로 자연스럽게 연결되는 진입점 제공

---

## 2. 작업 범위

### 포함 범위

- 문서 목록 화면 구조 설계
- 검색 UI 설계
- 필터 및 정렬 UI 설계
- 문서 아이템 표시 방식 정의

### 제외 범위

- 검색 엔진 구현 (Phase 8)
- API 상세 구현

## 4. 주요 설계 대상

### 4-1. 문서 목록 화면 구조

문서 목록 화면의 기본 구조를 정의한다.

#### 구성 요소

- Page Header (제목: Documents)
- Action Bar (검색, 필터, 정렬)
- Document List 영역
- Pagination 또는 Infinite Scroll

---

### 4-2. 문서 아이템 표현 방식

문서를 리스트 형태로 표현한다.

#### 표시 항목

- 문서 제목
- Document Type
- 상태 (Draft / Review / Approved / Published)
- 작성자
- 마지막 수정일

#### 선택적 항목

- 즐겨찾기 여부
- 태그 / 카테고리

#### UI 요구사항

- 클릭 시 문서 상세 화면으로 이동
- 상태에 따른 시각적 구분 (배지 등)

---

### 4-3. 검색 UI 설계

#### 기본 검색

- 텍스트 기반 검색 입력창
- Enter 또는 버튼 클릭으로 검색 실행

#### 요구사항

- 제목 기반 검색 (기본)
- 향후 내용 검색 확장 가능 구조

---

### 4-4. 필터 UI 설계

다음 기준으로 필터링 가능하도록 설계한다.

#### 필터 항목

- 문서 상태
- Document Type
- 작성자
- 날짜 범위

#### UI 형태

- Dropdown / Multi-select
- 필터 초기화 기능 제공

---

### 4-5. 정렬 UI 설계

정렬 기준을 정의한다.

#### 정렬 항목

- 최신 수정순
- 생성일순
- 이름순

---

### 4-6. 권한 기반 노출 정책

문서 목록에서 다음을 고려한다.

- 접근 권한이 있는 문서만 노출
- 권한 부족 시 목록에서 제외 또는 제한 표시

---

### 4-7. 상태별 UI 처리

#### (1) Loading

- Skeleton 리스트 표시

#### (2) Empty

- “문서가 없습니다” 메시지
- 문서 생성 버튼 제공

#### (3) Error

- 데이터 로드 실패 메시지
- 재시도 버튼
