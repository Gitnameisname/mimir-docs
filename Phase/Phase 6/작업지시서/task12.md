

# Task 6-12. 구현 우선순위 및 MVP 범위 확정

## 1. 작업 목적

Phase 6에서 설계한 User UI를 실제 구현 단계로 전환하기 위해 **구현 우선순위와 MVP(Minimum Viable Product) 범위를 확정**한다.

이 작업의 핵심 목표는 다음과 같다.

- 필수 기능과 확장 기능을 명확히 분리
- 실제 개발 착수 시 어떤 순서로 구현할지 정의
- 초기 배포 가능한 최소 기능 세트(MVP)를 확정

---

## 2. 작업 범위

### 포함 범위

- User UI 기능 목록 정리
- MVP 포함 기능 정의
- 후속 확장 기능 분리
- 구현 우선순위 및 단계 정의
- API 의존성 및 선후 관계 정리

### 제외 범위

- 실제 개발 일정 산정 (세부 일정은 별도)
- 인력 배치 계획

---

## 3. 선행 조건

- Task 6-1 ~ 6-10 UI 설계 완료
- Phase 3 (API 기초 계층) 구현 가능 상태
- Phase 4~5 (문서/버전/워크플로) 백엔드 설계 확정

---

## 4. 주요 설계 대상

### 4-1. 기능 분류 (필수 vs 확장)

User UI 기능을 다음과 같이 분류한다.

#### (1) 필수 기능 (MVP 대상)

- 문서 목록 조회
- 기본 검색
- 문서 상세 보기
- 구조 탐색
- 문서 편집 (Draft 기준)
- 버전 목록 조회
- 검토 대상 목록
- 승인 / 반려 기본 기능
- 공통 피드백 UX

#### (2) 확장 기능 (후속 Phase)

- 고급 검색 (필터 확장, 전문 검색)
- 버전 Diff 시각화
- 실시간 협업 편집
- 고급 코멘트 시스템
- RAG 질의 UI 실제 기능
- DocumentType별 커스텀 에디터

---

### 4-2. MVP 사용자 시나리오 정의

MVP에서 반드시 지원해야 하는 최소 사용자 흐름을 정의한다.

#### 시나리오 1. 문서 열람

- 문서 목록 → 문서 선택 → 상세 보기 → 구조 탐색

#### 시나리오 2. 문서 작성

- 문서 생성 → 편집 → 저장 (Draft 유지)

#### 시나리오 3. 문서 검토

- Review 대상 확인 → 문서 열람 → 승인 또는 반려

---

### 4-3. 구현 단계 분할

구현을 단계적으로 나누어 정의한다.

#### Phase 6-1. 기본 구조 구축

- Layout 및 공통 컴포넌트 구현
- 라우팅 구조 적용

#### Phase 6-2. 열람 기능 구현

- 문서 목록
- 문서 상세
- 구조 탐색

#### Phase 6-3. 편집 기능 구현

- 문서 편집 UI
- 저장 기능

#### Phase 6-4. 버전 및 워크플로

- 버전 목록
- 승인 / 반려 UI

#### Phase 6-5. 공통 UX 및 안정화

- 피드백 UX 적용
- 권한 / 상태 가드 적용

---

### 4-4. API 의존성 정리

각 UI 기능이 의존하는 API를 정리한다.

#### 예시

- 문서 목록 API
- 문서 상세 API
- 문서 저장 API
- 버전 조회 API
- 워크플로 상태 전이 API

#### 요구사항

- API 준비 여부 확인
- UI 구현 순서와 API 의존성 일치

---

### 4-5. 리스크 및 우선 고려 사항

#### 주요 리스크

- 편집 UI 복잡도 증가
- 상태/권한 처리 누락
- API 응답 구조 변경

#### 대응 방향

- MVP 범위 내 단순화
- 공통 가드 체계 우선 적용
- API 계약 명확화

---

## 5. 산출물

1. MVP 기능 정의서
2. 기능 분류표 (필수 / 확장)
3. 구현 단계별 로드맵
4. API 의존성 매핑 문서

---

## 6. 완료 기준

- MVP 범위가 명확히 정의되어 있다
- 구현 우선순위가 단계별로 정리되어 있다
- 사용자 핵심 시나리오가 모두 커버된다
- 개발 착수가 가능한 수준의 계획이 수립되어 있다

---

## 7. Codex 작업 지침

- 코드 구현 금지
- 실제 개발 착수를 고려한 현실적인 범위 설정
- MVP는 "반드시 동작해야 하는 최소 기능" 기준으로 정의
- 기능 욕심보다 안정적인 흐름을 우선할 것
