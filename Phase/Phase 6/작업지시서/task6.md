


# Task 6-6. 버전 관리 UI 설계

## 1. 작업 목적

사용자가 문서의 변경 이력을 이해하고, 필요 시 특정 시점으로 되돌릴 수 있도록 **버전 관리 UI**를 설계한다.

이 작업의 핵심 목표는 다음과 같다.

- 문서의 시간 흐름(Version History)을 명확하게 표현
- 각 버전의 변경 맥락을 이해 가능하게 제공
- 안전하게 이전 버전으로 복원할 수 있는 UX 제공

---

## 2. 작업 범위

### 포함 범위

- 버전 목록 화면 설계
- 버전 메타데이터 표시 방식 정의
- 버전 상세 조회 UI 정의
- 버전 복원 UX 설계

### 제외 범위

- 버전 간 Diff 시각화 (Phase 9)
- 실제 복원 로직 구현

---

## 3. 선행 조건

- Task 6-4 (문서 상세 UI)
- Task 6-5 (문서 편집 UI)

---

## 4. 주요 설계 대상

### 4-1. 버전 목록 화면

#### 기본 구조

- Version List (Timeline 또는 리스트)
- 각 버전의 요약 정보 표시

#### 표시 항목

- Version 번호
- 작성자
- 생성 시각
- 변경 사유 (Change Reason)
- 상태 (Draft / Published 등)

---

### 4-2. 버전 표현 방식

#### UX 옵션

- 최신 → 과거 순 정렬
- Timeline 형태 (권장)

#### 요구사항

- 현재 버전 강조 표시
- Published 버전 표시

---

### 4-3. 버전 상세 보기

#### 기능

- 특정 버전의 문서 내용 조회
- 읽기 전용 모드

#### 요구사항

- 현재 버전과 구분 명확히
- 편집 불가 상태 표시

---

### 4-4. 버전 복원 UX

#### 주요 기능

- 특정 버전을 기준으로 복원

#### UX 흐름

- 버전 선택 → Restore 버튼 → Confirm Dialog → 복원 실행

#### 정책

- 복원은 새로운 버전을 생성하는 방식
- 기존 버전은 유지

---

### 4-5. 권한 기반 제어

#### 규칙

- 버전 조회: 대부분 사용자 허용
- 버전 복원: 제한된 역할만 가능

---

### 4-6. 상태별 UI 처리

#### Loading

- 버전 목록 로딩 표시

#### Empty

- 버전 없음 안내

#### Error

- 로딩 실패 메시지
