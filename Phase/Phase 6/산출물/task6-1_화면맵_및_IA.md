# Task 6-1 산출물: User UI 정보 구조 및 화면 맵

---

## 1. User UI 화면 맵 (Page Structure)

```
User UI
├── /                        → 대시보드 또는 /documents 리다이렉트
├── /documents               → 문서 목록 화면
│   ├── /documents/new       → 문서 생성 화면
│   └── /documents/[id]      → 문서 상세 보기 화면
│       ├── /documents/[id]/edit          → 문서 편집 화면
│       ├── /documents/[id]/versions      → 버전 목록 화면
│       └── /documents/[id]/versions/[vid] → 특정 버전 상세 보기
├── /reviews                 → 검토 대상 목록 화면 (My Tasks)
└── /profile                 → 사용자 프로필 (선택)
```

## 2. 라우팅 구조 정의서

### 2-1. 라우팅 테이블

| Route | 화면 명 | 진입 역할 | 설명 |
|-------|---------|-----------|------|
| `/documents` | 문서 목록 | 전체 | 문서 목록 조회, 검색, 필터 |
| `/documents/new` | 문서 생성 | Author 이상 | 새 문서 생성 진입점 |
| `/documents/[id]` | 문서 상세 | 권한 있는 모든 역할 | 문서 내용 열람, 메타데이터 확인, 액션 |
| `/documents/[id]/edit` | 문서 편집 | Author (Draft 상태) | 구조 기반 편집 |
| `/documents/[id]/versions` | 버전 목록 | 권한 있는 모든 역할 | 버전 이력 조회 |
| `/documents/[id]/versions/[vid]` | 버전 상세 | 권한 있는 모든 역할 | 특정 버전 내용 열람 |
| `/reviews` | 검토 대상 목록 | Reviewer / Approver | 검토 대기 문서 목록 |

### 2-2. 라우팅 원칙

- **문서 중심 계층 구조**: 모든 문서 관련 화면은 `/documents/[id]` 하위로 구성
- **RESTful URL**: 자원 식별자는 path parameter, 필터/검색은 query parameter
- **권한 가드**: 라우트 진입 시 역할 및 문서 상태 검사
- **확장 가능성**: 추후 `/documents/[id]/diff`, `/documents/[id]/rag` 등 추가 가능

### 2-3. Query Parameter 정의 (문서 목록)

| Parameter | 타입 | 설명 |
|-----------|------|------|
| `q` | string | 전문 검색 키워드 |
| `status` | enum | 문서 상태 필터 (draft, in_review, approved, published) |
| `type` | string | 문서 유형 필터 |
| `author` | string | 작성자 필터 |
| `page` | number | 페이지 번호 |
| `limit` | number | 페이지당 항목 수 (기본 20) |
| `sort` | string | 정렬 기준 (updated_at, created_at, title) |
| `order` | enum | 정렬 방향 (asc, desc) |

## 3. 주요 화면 정의서

### 3-1. 문서 목록 화면 (`/documents`)

**목적**: 사용자가 접근 가능한 문서를 탐색하고 원하는 문서에 빠르게 접근

**핵심 요소**:
- 검색 입력 + 필터 바 (상태, 유형, 작성자)
- 문서 목록 (테이블 또는 카드 뷰)
  - 표시 컬럼: 제목, 유형, 상태 배지, 작성자, 최종 수정일, 버전 번호
- 정렬 컨트롤
- 페이지네이션
- 새 문서 생성 버튼 (Author 이상만 노출)

---

### 3-2. 문서 상세 화면 (`/documents/[id]`)

**목적**: 문서 내용을 읽고 구조를 탐색하며 상태에 따른 액션 수행

**레이아웃**:
```
┌─────────────────────────────────────────────────────┐
│ Header: 문서 제목 / 상태 배지 / 버전 정보 / 액션 버튼 │
├──────────────┬──────────────────────────────────────┤
│  구조 트리   │  문서 본문 (Node 렌더링)               │
│  (좌측 패널) │                                       │
│              │  메타데이터 섹션                       │
└──────────────┴──────────────────────────────────────┘
```

**액션 버튼 (상태/역할별 가변)**:
- [편집] — Author + Draft 상태
- [검토 요청] — Author + Draft 상태
- [승인] — Approver + In Review 상태
- [반려] — Reviewer/Approver + In Review 상태
- [버전 목록] — 전체

---

### 3-3. 문서 편집 화면 (`/documents/[id]/edit`)

**목적**: Draft 상태 문서를 구조 기반으로 편집

**핵심 요소**:
- 문서 제목 편집
- 메타데이터 편집 영역
- 노드(섹션) 구조 편집기
  - 노드 추가/삭제/순서 변경
  - 인라인 텍스트 편집
- 자동 저장 표시기
- [저장] / [취소] 버튼

---

### 3-4. 버전 목록 화면 (`/documents/[id]/versions`)

**목적**: 문서 버전 이력 조회

**핵심 요소**:
- 버전 목록 (버전 번호, 생성일, 작성자, 워크플로 상태, 변경 사유)
- 현재 버전 / 발행 버전 강조 표시
- 버전 상세 진입 링크

---

### 3-5. 검토 대상 목록 화면 (`/reviews`)

**목적**: 검토자/승인자가 처리해야 할 문서 목록 확인

**핵심 요소**:
- In Review 상태 문서 목록
- 제목, 작성자, 검토 요청일, 문서 유형 표시
- 상세 보기로 바로 이동

## 4. 사용자 흐름 정의서

### 4-1. 열람자 흐름

```
[문서 목록] → 제목 클릭 → [문서 상세]
                              ↓
                         구조 트리 탐색
                              ↓
                         특정 섹션 이동
                              ↓
                    [버전 목록] → 특정 버전 열람
```

**주요 진입점**: `/documents`

---

### 4-2. 작성자 흐름

```
[문서 목록] → [새 문서 생성]
                    ↓
             문서 생성 폼 작성
                    ↓
             [문서 편집 화면]
                    ↓
         편집 → 저장 (반복) → 자동 저장
                    ↓
             [검토 요청 액션]
                    ↓
         상태: Draft → In Review
```

**주요 진입점**: `/documents` → "새 문서" 버튼

---

### 4-3. 검토자/승인자 흐름

```
[검토 대상 목록 /reviews]
          ↓
    문서 선택
          ↓
[문서 상세 화면] — 문서 내용 검토
          ↓
  [승인] 또는 [반려] 액션
          ↓
  반려 시: 사유 입력 → 상태: Rejected
  승인 시: 상태: Approved → (Approver만) [발행 가능]
```

**주요 진입점**: `/reviews`

---

### 4-4. 후반 확장 흐름 (Phase 11 연계)

```
[문서 상세 화면]
       ↓
 [RAG 질의 패널 열기]
       ↓
  질문 입력 → 답변 수신
       ↓
  Citation 클릭 → 해당 섹션 이동
```

## 5. 역할별 진입 시나리오

| 역할 | 기본 진입점 | 주요 행동 | 핵심 화면 |
|------|------------|----------|----------|
| Viewer (열람자) | `/documents` | 검색 → 상세 열람 → 구조 탐색 | 목록, 상세, 버전 상세 |
| Author (작성자) | `/documents` | 생성 → 편집 → 저장 → 검토 요청 | 목록, 편집, 버전 목록 |
| Reviewer (검토자) | `/reviews` | 검토 대상 확인 → 열람 → 반려/승인 의견 | 검토 목록, 상세, 승인 액션 |
| Approver (승인자) | `/reviews` | 검토 대상 확인 → 최종 승인 → 발행 | 검토 목록, 상세, 승인/발행 액션 |

### 역할 중복 시 우선 규칙

- 사용자가 복수 역할을 가질 경우 **가장 높은 권한 역할이 우선** 적용
- 단, 본인이 작성한 문서에 대한 승인자 역할은 **이해충돌 방지** 정책에 따라 제한 (서버 정책 따름)

## 6. 공통 레이아웃 구조 초안

```
┌─────────────────────────────────────────────────────────┐
│                        Header                            │
│  [Mimir 로고]  [네비게이션 메뉴]          [사용자 정보]  │
├──────────────┬──────────────────────────────────────────┤
│              │                                           │
│   Sidebar    │           Main Content Area              │
│  (Navigation │  (페이지별 콘텐츠)                        │
│   + Context) │                                           │
│              ├──────────────────────────────────────────┤
│              │      Optional Right Panel                 │
│              │  (구조 트리 / RAG 패널 / 버전 패널)       │
└──────────────┴──────────────────────────────────────────┘
```

### 레이아웃 구성 요소

| 영역 | 역할 | 비고 |
|------|------|------|
| Header | 전역 네비게이션, 사용자 정보, 알림 | 모든 페이지 고정 |
| Sidebar | 주 메뉴 (Documents, My Tasks, Favorites) | 접기 가능 |
| Main Content | 각 페이지 핵심 콘텐츠 | 스크롤 가능 |
| Right Panel | 문서 구조 트리 / RAG 패널 (상황별 노출) | 토글 가능 |

### Admin UI와의 분리

- User UI: `/` ~ `/documents`, `/reviews` 등
- Admin UI: `/admin/...` (Phase 7에서 별도 라우트로 분리)
- 공통 컴포넌트는 공유하되 레이아웃과 네비게이션 컨텍스트는 분리
