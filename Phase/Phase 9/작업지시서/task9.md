# Task 9-9. 버전 이력 타임라인 구현

## 1. 작업 목적

문서의 전체 버전 이력을 시각적 타임라인으로 제공하여, 사용자가 문서가 어떻게 변화해 왔는지 한눈에 파악하고 원하는 버전을 탐색할 수 있도록 한다.

이 작업의 목표는 다음과 같다.

- 버전 이력 타임라인 UI 설계 및 구현
- 버전별 메타데이터(작성자, 날짜, 변경 요약, 변경 사유) 표시
- 버전 간 비교 진입 흐름 구현
- Published 버전 등 주요 이정표 표시

---

## 2. 작업 범위

### 포함 범위

- 버전 이력 타임라인 컴포넌트 설계 및 구현

## 4. 주요 구현 대상

### 4-1. 버전 이력 타임라인 진입점

#### 진입 위치

- 문서 상세 화면 우측 패널 "버전 이력" 탭
- URL: `/documents/{doc_id}?tab=history`

```
버전 이력
```

### 4-3. 버전 항목 구성 요소

각 버전 항목에 포함할 정보:

| 항목 | 설명 |
|------|------|
| 버전 번호 | v1.0, v1.1 등 |
| 상태 배지 | Published / Draft / In Review / Approved |
| 이정표 배지 | ★Published, ★최초 작성 등 |
| 작성 일시 | 상대 시간 (예: "3일 전") + 절대 시간 툴팁 |
| 작성자 | 이름 |
| 변경 사유 | 버전 생성 시 입력한 메모 (없으면 "변경 사유 없음") |
| 변경 요약 | DiffSummary에서 생성된 description |
| 액션 버튼 | [이 버전 보기] [직전 버전과 비교] |

---

### 4-4. 이정표 버전 표시

특별한 의미를 가진 버전은 시각적으로 강조:

| 이정표 유형 | 표시 방식 |
|-----------|---------|
| Published 버전 | ★ 배지 + 강조 테두리 |
| 최초 버전 (v1.0) | "최초 작성" 라벨 |
| 현재 활성 버전 | "현재" 배지 |
| 검토 중 버전 | "검토 중" 배지 |

---

### 4-5. 버전 비교 진입 흐름

버전 항목에서의 비교 액션:

#### [직전 버전과 비교] 버튼

- 현재 버전과 바로 이전 버전 비교
- Task 9-7의 DiffViewer 페이지로 이동
- URL: `/documents/{doc_id}/versions/{current_v}/compare`

#### [버전 선택 비교] 기능

- 타임라인에서 두 버전 선택 (체크박스 또는 클릭+Shift)
- 선택된 두 버전 비교 버튼 활성화
- URL: `/documents/{doc_id}/versions/{v1}/compare/{v2}`

---

### 4-6. 버전 이력 API 활용

기존 버전 목록 API를 활용하여 타임라인 데이터 구성:

```
GET /documents/{doc_id}/versions
```

응답에 각 버전의 변경 요약을 포함하도록 API 확장 (또는 별도 enrichment):

```json
{
  "versions": [
    {
      "id": "ver_003",
      "number": "1.3",
      "status": "DRAFT",
      "created_at": "...",
      "created_by": { "id": "user_park", "name": "park" },
      "note": "3분기 정책 개정 반영",
      "diff_summary": {
        "description": "섹션 2개 수정됨",
        "total_modified": 2
      }
    }
  ]
}
```

`diff_summary`는 서버에서 사전 계산 또는 캐시된 값 사용.

### 4-7. 무한 스크롤 또는 페이지네이션

- Published 버전은 많은 경우에도 항상 표시

## 6. 완료 기준

- 문서 상세 화면에서 버전 이력 타임라인이 표시된다
- 각 버전 항목에 작성자, 날짜, 변경 요약이 표시된다
- Published 등 이정표 버전이 시각적으로 강조된다
- 버전 항목에서 직전 버전과의 비교 페이지로 이동 가능하다
- 버전 이력이 많은 경우 페이지네이션이 동작한다

## 7. Codex 작업 지침

- Phase 6 User UI의 우측 패널(Side Panel) 패턴을 활용한다
- 타임라인 UI는 버전 수가 많아도 성능이 유지될 수 있도록 가상화(virtualization)를 고려한다
- diff_summary 계산 비용을 고려하여 API 응답에 미리 포함하거나 lazy loading으로 처리한다
- 권한에 따라 접근 불가한 버전은 타임라인에서 흐릿하게 표시하거나 제외 처리한다
