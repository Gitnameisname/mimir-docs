# Task 9-8. 워크플로 검토 화면 diff 통합

## 1. 작업 목적

검토자가 검토 요청 화면에서 직전 버전 대비 변경 내용을 즉시 확인할 수 있도록 워크플로 검토 화면에 diff 기능을 통합한다.

이 작업의 목표는 다음과 같다.

- 검토 화면 상단에 변경 요약 배너 자동 표시
- 변경 내용 보기 인라인 토글 (별도 페이지 이동 없이)
- 직전 Published/Draft 버전 자동 감지 로직
- 검토자 관점의 UX 최적화 (변경 범위 파악 → 승인/반려 결정)

---

## 2. 작업 범위

### 포함 범위

- 검토 화면에 DiffSummary 배너 자동 로드 및 표시
- "변경 내용 보기" 토글 기능 구현
- 직전 비교 기준 버전 자동 감지 로직

## 4. 주요 구현 대상

### 4-1. 직전 비교 기준 버전 자동 감지

검토 화면에서 "이전 버전"을 자동으로 결정하는 로직:

#### 비교 기준 우선순위

1. 가장 최근의 PUBLISHED 버전 (안정된 기준점)
2. PUBLISHED 버전이 없으면 가장 최근의 APPROVED 버전
3. 그것도 없으면 직전 버전 (버전 번호 기준 -1)
4. 첫 버전이면 "이전 버전 없음" 처리

#### 자동 감지 결과 표시

"기준 버전: v1.2 (Published, 2025-03-01)" 형태로 검토자에게 안내.

---

### 4-2. 검토 화면 변경 요약 배너

검토 화면 상단에 DiffSummary 배너 자동 표시:

```
┌────────────────────────────────────────────────────────┐
│ 변경 내용 (v1.2 → 현재 검토 버전)                        │
│ 섹션 1개 수정, 단락 3개 수정됨                            │
│ 변경된 섹션: [2. 정책 본문]                              │
│                           [변경 내용 상세 보기 ▼]       │
└────────────────────────────────────────────────────────┘
```

- 검토 화면 진입 시 DiffSummary API 자동 호출
- 로딩 Skeleton 표시 후 결과 렌더링
- "변경 사항 없음" 시 배너 숨김 또는 회색 표시

---

### 4-3. 인라인 변경 내용 토글

"변경 내용 상세 보기" 클릭 시 검토 화면 내에서 DiffViewer 인라인 펼침:

```
┌────────────────────────────────────────────────────────┐
│ 변경 내용 (v1.2 → 현재 검토 버전)                        │
│ 섹션 1개 수정, 단락 3개 수정됨                            │
│                             [변경 내용 접기 ∧]          │
├────────────────────────────────────────────────────────┤
│ [DiffViewer - Unified 뷰]                              │
│ ~ 2. 정책 본문 [수정됨]                                  │
│   기존: ...                                             │
│   이후: ...                                             │
└────────────────────────────────────────────────────────┘

[검토 의견 입력 영역]
[승인]  [반려]
```

- DiffViewer는 Task 9-7의 컴포넌트를 재사용
- 검토 화면에서는 Unified 뷰 전용 (Side-by-side는 별도 페이지에서)
- 매우 많은 변경이 있을 경우 "전체 비교 페이지에서 보기" 링크 제공

---

### 4-4. 검토자 UX 흐름 개선

변경 확인 → 검토 의견 입력 → 승인/반려 흐름:

1. 검토 화면 진입
2. 상단 변경 요약 배너 확인
3. "변경 내용 상세 보기" 필요 시 펼침
4. 특정 노드에서 "이 섹션에 의견 추가" 기능 (선택, Phase 이후 확장)
5. 검토 의견 입력 후 승인/반려

---

### 4-5. 검토 화면 변경 요약 API 연계

검토 화면 로드 시 DiffSummary API 호출:

```
GET /documents/{doc_id}/versions/{current_v_id}/diff/summary
```

응답에서 DiffSummary만 사용하여 배너 렌더링 (전체 diff는 토글 시에만 요청).

## 6. 완료 기준

- 검토 화면 진입 시 변경 요약이 자동으로 표시된다
- 직전 기준 버전이 올바르게 자동 감지된다
- "변경 내용 상세 보기" 토글이 동작한다
- 첫 버전 (이전 버전 없음) 상황이 안전하게 처리된다
- 변경 요약 로딩 중 Skeleton이 표시된다

---

## 7. Codex 작업 지침

- Phase 5 워크플로 검토 화면의 기존 UX를 최대한 유지하면서 diff 기능을 추가한다
- 변경 요약 배너는 검토자의 집중을 방해하지 않는 수준으로 배치한다
- DiffViewer는 Task 9-7에서 구현된 컴포넌트를 그대로 재사용한다
- 검토 화면 내 인라인 DiffViewer는 높이 제한과 스크롤을 적용하여 검토 의견 입력 영역이 가려지지 않도록 한다
