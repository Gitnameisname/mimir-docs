## 4. 주요 구현 대상

### 4-1. diff 결과 캐싱 전략

#### 캐시 키 설계

```
diff:{min(v1_id, v2_id)}:{max(v1_id, v2_id)}
```

버전 ID를 정규화(작은 것이 앞)하여 A↔B, B↔A 요청이 동일한 캐시를 사용.

#### 캐싱 대상

| 응답 유형 | 캐싱 여부 | 이유 |
|----------|----------|------|
| DiffResult (전체) | O | 버전 불변 → 항상 동일한 결과 |
| DiffSummary (경량) | O | 버전 불변 → 항상 동일한 결과 |
| 인라인 diff 포함 | O | 동일 조건 → 동일 결과 |

#### TTL 설정

- 버전은 변경되지 않으므로 TTL을 매우 길게 설정 가능
- 권장: 7일 또는 무제한 (메모리 압박 시 LRU 퇴출)
- 문서 삭제/보존 정책에 따라 캐시 무효화 필요

#### 캐시 저장소 선택

| 옵션 | 장점 | 단점 |
|------|------|------|
| 인메모리 (애플리케이션 내) | 구현 단순, 매우 빠름 | 서버 재시작 시 소실, 다중 서버 시 불일치 |
| Redis | 다중 서버 공유, 영속 가능 | 별도 인프라 필요 |
| PostgreSQL (DB 저장) | 영속, 쿼리 가능 | 저장 공간, 조회 속도 |

→ **Phase 9 MVP: 인메모리 캐싱**으로 시작, 필요 시 Redis로 전환

### 4-2. 캐시 무효화 조건

버전은 불변이지만 다음 경우 캐시 무효화 필요:

| 상황 | 무효화 대상 |
|------|-----------|
| 문서 삭제 | 해당 문서의 모든 diff 캐시 |
| 버전 데이터 수정 (드문 경우) | 해당 버전을 포함하는 모든 diff 캐시 |
| 캐시 저장소 용량 초과 | LRU 방식으로 오래된 캐시 자동 퇴출 |

### 4-3. 대용량 문서 diff 비동기 처리

노드 수 기준 임계값(기본 10,000개) 초과 시:

#### 동기 처리 불가 응답

```json
{
  "status": "PROCESSING",
  "job_id": "diff_job_abc123",
  "message": "변경 사항이 매우 많아 처리 중입니다. 잠시 후 다시 조회하세요.",
  "estimated_seconds": 30
}
```

#### 비동기 처리 흐름

1. diff 요청 수신 → 즉시 `PROCESSING` 응답
2. 백그라운드 작업으로 diff 계산 (Phase 7 Background Job 연계)
3. 완료 시 캐시에 저장
4. 클라이언트가 `GET /diff/jobs/{job_id}` 폴링 또는 완료 콜백

### 4-4. 성능 목표 기준

| 측정 항목 | 목표 기준 |
|----------|----------|
| 소규모 문서 diff (< 100 노드) | < 100ms |
| 중규모 문서 diff (100~1000 노드) | < 500ms |
| 대규모 문서 diff (> 1000 노드) | < 2000ms 또는 비동기 처리 |
| 캐시 히트 응답 | < 20ms |
| DiffSummary 응답 (경량) | < 200ms |

### 4-5. 성능 측정 및 병목 식별

측정 대상:

| 단계 | 측정 항목 |
|------|---------|
| 노드 로딩 (DB) | 두 버전의 Node 트리 조회 시간 |
| ID 매칭 | 맵 생성 및 분류 시간 |
| 내용 정규화 | 정규화 함수 처리 시간 |
| 인라인 diff | diff-match-patch 처리 시간 |
| 요약 생성 | DiffSummaryGenerator 처리 시간 |
| 직렬화 | DiffResult → JSON 변환 시간 |
| 합계 | 전체 응답 시간 |

#### 일반적인 최적화 방안

| 병목 지점 | 최적화 방안 |
|----------|-----------|
| 노드 로딩 느림 | version_id 인덱스 최적화, 필요 컬럼만 SELECT |
| 정규화 비용 큼 | 정규화 결과 임시 캐싱 |
| 인라인 diff 느림 | 텍스트 길이 제한, 병렬 처리 |
| JSON 직렬화 느림 | 불필요한 필드 제거, 스트리밍 응답 고려 |
