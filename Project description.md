# Mimir 개발 계획 v2 (최종 상위 Phase)

---

## 0. 핵심 정의 (변경된 방향 요약)

### 프로젝트 정체성

> **Mimir = 범용 문서/지식 플랫폼 + API 기반 인프라**

---

### 핵심 설계 원칙 (반드시 유지)

#### 1. API-first
- 모든 기능은 API로 먼저 정의된다
- UI는 API의 클라이언트 중 하나일 뿐이다
- 외부 시스템/AI/에이전트 연계를 기본 전제로 한다

#### 2. Dual UI
- User UI (업무 사용자)
- Admin UI (운영 관리자)

> 처음부터 분리 설계

#### 3. Document = 구조, Type = 행동
- Document는 generic
- DocumentType이 동작을 정의
- 모든 확장은 Type 레이어에서 처리

#### 4. 권한은 모든 계층에 적용
- API
- 검색
- 벡터
- RAG

---
