# Phase 0 — 데이터 자산 분류 매트릭스

## 사용 목적

이 매트릭스는 Mimir 플랫폼이 다루는 데이터 자산을 유형별로 분류하고, 각 자산의 저장 특성과 적합한 저장 계층을 정리한다. Phase 1 도메인 모델 설계와 저장 구조 설계에서 참조한다.

### 저장 계층 약어

| 약어 | 저장 계층 |
|------|----------|
| **RDB** | 관계형 DB (정형 컬럼, 트랜잭션, 외래키) |
| **JSON** | RDB 내 JSON/JSONB 확장 필드 |
| **OBJ** | 오브젝트/파일 스토리지 |
| **IDX** | 검색 인덱스 계층 |
| **LOG** | 로그·감사·운영 저장소 |

### 변경 빈도 기준

- **낮음**: 생성 후 거의 변경 없음
- **중간**: 주기적으로 변경 발생
- **높음**: 빈번하게 변경 발생

### 무결성·이력·검색 요구 기준

- **필수**: 반드시 요구
- **권장**: 대부분의 경우 필요
- **선택**: 상황에 따라

## 저장 계층별 자산 요약

### 관계형 DB (RDB)

Document 기본 정보 / DocumentType 정의 / Version 메타 정보 / Node 메타 / Metadata 공통 / Status / Relationship / Attachment 메타 / External Resource Ref / Permission·Policy / Config·Settings

### JSON 확장 필드 (RDB 내)

DocumentType 스키마 정의 내부 / Metadata 유형별 확장 값 / Config 일부 보조 속성

### 오브젝트 스토리지 (OBJ)

Version 본문 원본 (대용량) / Node 본문 (대용량) / Attachment 파일 원본

### 검색 인덱스 (IDX)

전문 검색 토큰 / 메타데이터 필터 인덱스 / Node 단위 색인 / AI 벡터 임베딩 (향후)

### 로그·감사 저장소 (LOG)

상태 전이 이력 / Audit Log / Operational Log·Job Status
